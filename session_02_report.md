## Теоремы сессии 02

### T_DW4_FORMULA (подтверждена из сессии 01)
```
dW[4] = -2 · d0  mod 2^32
P = 1.000000  (10000/10000 образцов, любой W_normal)
```

### T_DW23_BINARY — новая
```
dW[2] и dW[3] принимают значения из малых целых Z:
  P(dW[2] = 0) = 33.3%
  P(|dW[2]| ≤ 4 как знаковое) = 59.4%
  P(dW[2] ≈ -1 mod 2^32) = 33.3%
  Уникальных значений: ~100–150 из 2^32 возможных

Распределение: геометрическое P(|x|=k) ≈ C · 2^{-k}
```

**Причина:** `dW[2] = -De[3]_nat` где `De[3]_nat = carry_xor(BASE, correction)`.
При малом d0=1 вся цепочка несёт "эхо" единицы через carry — это геометрический закон переносов.

### T_DE3_GEOMETRIC — новая
```
De[3]_nat при De[2]=0, d0=1:
  P(De3=0) = 33.3%
  P(De3=1) = 32.9%
  P(|De3|≤4) = 92.3%
  P(|De3|>4) = 7.7%

Это геометрическое распределение: P(|k|) ≈ 2^{-|k|}
```

### T_DW23_CORRELATED — новая
```
dW[2] и dW[3] ЗАВИСИМЫ (положительная корреляция):
  P(dW[2]=0) = 33.3%
  P(dW[3]=0) = 33.3%
  P(dW[2]=0 AND dW[3]=0) = 14.1%  >  0.33² = 10.9%
  Коэффициент корреляции: ~1.3× выше независимости
```

### T_CARRY_ACCELERATES — из сессии 01, подтверждена
```
В XOR-модели: E[HW(De2)] = 4.0
В реальном SHA: E[HW(De2)] = 14.9
Разница: +10.9 бит

Carry УСКОРЯЕТ диффузию в 3.7× на раунде 2.
Это защитная роль carry: ускорение лавинного эффекта.
```

## Структурная карта каскадного dW

```
d0 = 1

dW[0]  = 1           ← INPUT (HW=1)
dW[1]  = f(W[0])     ← СЛУЧАЙНЫЙ, зависит от W[0]
dW[2]  = g(W[0])     ← МАЛЫЙ ЦЕЛЫЙ, ~геометрич.
dW[3]  = h(W[0])     ← МАЛЫЙ ЦЕЛЫЙ, ~геометрич.
dW[4]  = 0xfffffffe  ← ДЕТЕРМИНИРОВАНО = -2
dW[5]  = q(W[0..4])  ← СЛУЧАЙНЫЙ
dW[6..15] = случайные
```

**Граница детерминизма:** только позиции 0 и 4.
**Граница "малости":** позиции 2 и 3.
**Всё остальное:** случайные 16-битные слова.

---

## Почему именно позиция 4?

Аналитически: De[5]_nat определяется состоянием после 4 раундов.
К раунду 5 система "поглощает" всю нелинейность из W[0..3] через Sigma, Ch, Maj.
Но Da[1] = carry_xor(BASE_A1, d0) имеет структуру зависящую только от d0.
Через 3-раундовый сдвиговый регистр: Da[4] = Da[1].
Поэтому De[5] = Da[4] + Sigma1(De[4]) + DCh[4] = детерминировано при De[2..4]=0.

**Формально:**
```
De[5]_nat = Da[1] + 0 + 0 = Da[1] mod 2^32
Da[1] = carry_xor(BASE_A1, d0)
При d0=1: Da[1] = 2  (геом. распределение trailing_ones)
Ожидание: E[Da1] ≈ 2 (геометрическое среднее + 1)
```

Но Da[1] = 2 точно при W[0..15] = 0 и d0=1 — это совпадение с нулевым IV.

## Что НЕ работает (отрицательные результаты)

1. **Барьер r=17 не снижается** для пар с dW[2]=dW[3]=0
   - E[HW(De17)] = 15.96 (свободные) vs 15.92 (обычные) — неотличимо
   - P(De17=0) = 0 в обоих случаях

2. **Нейтральных бит в dW нет** — каждый бит влияет на нули

3. **Sol_Wang равномерно распределено** — нет кластеров
