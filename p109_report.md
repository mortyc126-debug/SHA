## КОНТЕКСТ

Открытый вопрос из П-108: почему SHA-256 G5=21.3 — аномально высокий результат,
в 3.5× выше Keccak и в 4.5× выше Blake2? Четыре направления параллельно:
A — откуда берётся r*=3, B — анатомия G5, C — теория оптимального хеша, D — связь с SM.

## П-109A: ОТКУДА r*=3?

### Измерение r* при разных δW[0]

```
DEP(r) без Wang (raw дифференциал):

r=1:   0.00%
r=2:  89.42%  ← r* при δW[0]=1 (v2=0)
r=3:  99.81%
r=4+: 100.00%

r*(δW[0]=1)  = 2  (v2=0)
r*(δW[0]=4)  = 3  (v2=2)
r*(δW[0]=0x8000) = 3  (v2=15)

r* зависит от v2(δW[0]):
  v2=0: r*=2  v2≥2: r*=3
```

### Аналитика r*=3

```
SHA-256 специфика: S1(e) = ROTR(6) ⊕ ROTR(11) ⊕ ROTR(25)
Три вращения на очень разные позиции (6, 11, 25) размазывают
однобитовый дифференциал по всему 32-битному слову за 2-3 раунда.

r*=3 — прямое следствие τ=4:
  τ=4: HW-насыщение за 4 раунда
  r*=3: DEP-насыщение за 3 раунда (чуть раньше)
```

## П-109B: АНАТОМИЯ G5

### Формула

```
G5(f) = Rounds_total / r*(f)

SHA-256: 64 / 3 = 21.3  ✓
```

### Ключевая таблица r*/τ

```
Хеш      τ    r*   r*/τ   G5
SHA-256   4     3   0.75  21.3  ← r* < τ (уникально!)
Blake2b   2     2   1.00   6.0
Blake2s   2     2   1.00   5.0
Keccak    2     2   1.00   9.0
SHA-1     9    21   2.33   3.8
MD5       5    21   4.20   3.0
```

### Главный инсайт: SHA-256 единственный с r* < τ

```
Для всех других хешей: r* ≥ τ
Для SHA-256:           r* = τ-1 = 3

Интерпретация: SHA-256 достигает оракульной случайности
ДО полного насыщения HW дифференциала.

Причина: S1(e) = ROTR(6)⊕ROTR(11)⊕ROTR(25)
быстро распространяет дифференциал ПОБИТОВО
даже до полного HW-насыщения.

В SHA-1: F(b,c,d)=(b&c)|(~b&d) медленнее — нужно
21 раунд для DEP=100% (при τ=9).
```

## П-109C: ТЕОРИЯ ОПТИМАЛЬНОГО G5

### Закон компромисса

```
G5 = Rounds / r*  ≈  Rounds / τ

Максимизируется при:
  1. max(Rounds) — много раундов
  2. min(r*) — быстрый оракул
  3. r* < τ — оракул до насыщения HW (уникально для SHA-256)

Теоретический максимум: G5 = Rounds при r*=1
Достижимо в AES-based хешах (τ≈1), но они медленнее.
```

### Новая метрика: Security Density

```
SD(f) = G5(f) × w(f) = (Rounds/r*) × w

SHA-256:  21.3 × 32 =  681.6
SHA-512:  21.3 × 64 = 1363.2  (прогноз)
Blake2b:   6.0 × 64 =  384.0
Keccak:    9.0 × 8  =   72.0  (w=8 для f[200])
SHA-1:     3.8 × 32 =  121.6
MD5:       3.0 × 32 =   96.0

SD учитывает диффузионную скорость И барьерную размерность Λ=w.
SHA-256 и SHA-512 доминируют по SD среди всех измеренных хешей.
```

## П-109D: G5 И SECURITY MARGIN

### Корреляция

```
Хеш      G5    Атакованных раундов  SM(%)
MD5       3.0   64/64 (взломан)      0%
SHA-1     3.8   80/80 (взломан)      0%
Blake2s   5.0   2.5/10              75%
SHA-256  21.3   31/64               51%
Keccak   ~12    4/24                83%

Наблюдение: G5 и SM НЕ монотонно связаны.
```

### Гипотеза G7

```
G5 измеряет ДИФФЕРЕНЦИАЛЬНУЮ устойчивость (Wang-стиль).
SM измеряет ОБЩУЮ практическую устойчивость (все атаки).

Они независимы:
  Keccak: SM=83% (высокий), G5≈12 (умеренный)
  SHA-256: SM=51% (умеренный), G5=21.3 (высокий)

G7: взломанные хеши (MD5, SHA-1) имеют G5 < 4.
    Непоколебленные хеши имеют G5 ≥ 5.
    G5 < 4 может служить предупреждающим сигналом.
```

## ГЛАВНЫЙ ВЫВОД П-109

```
Три причины высокого G5(SHA-256)=21.3:

  1. Rounds=64 — большое число раундов (главная причина)
     При тех же r*=3: G5=64/3=21.3 vs Blake2s 10/2=5

  2. r*=3 — быстрый оракул благодаря S1=ROTR(6,11,25)
     Три разноудалённых вращения → быстрая побитовая диффузия

  3. r*/τ = 0.75 < 1 — уникальное свойство
     Единственный хеш где оракул наступает до HW-насыщения.
     Следствие специфики S1-нелинейности SHA-256.

Формула: G5(SHA-256) = 64 / (τ-1) = 64 / 3 = 21.3
```
