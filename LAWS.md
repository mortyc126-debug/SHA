## СЛОЙ 1: ВТОРИЧНЫЕ УРАВНЕНИЯ (комбинации Z1..Z11)

### E1. Уравнение максимального раунда атаки [Z2 + Z10]
```
N(d) = C(R_max)
→ R_max(d) = 17 + √(N(d) / 7)
```
Атака возможна пока нейтральных бит хватает покрыть условия.

| d  | N(d) | R_max |
|----|------|-------|
| 0  | 78   | 20.3  |
| 9  | 153  | 21.7  |
| 12 | 90   | 20.6  |

**Оптимальный дифференциал: W[9], даёт R_max ≈ 22.**

### E2. Уравнение стоимости атаки [Z5 + Z6 + Z10]
```
Cost(R) = 2^{C(R) - Δ_GA × min(R-17, D_casc)}
        = 2^{7(R-17)² - 9 × min(R-17, 2)}
```

| R  | Cost        |
|----|-------------|
| 17 | 2^0 (free)  |
| 18 | 2^(-2) → 1  |
| 19 | 2^10        |
| 20 | 2^45        |
| 22 | 2^157       |

### E3. Закон фазового порога [Z1 + Z10]
Фазовый переход (Z1) наступает когда C(R) ≈ 16:
```
C(R*) = 7(R* - 17)² = 16
→ R* = 17 + √(16/7) = 18.5
```
**Фазовый переход = момент когда unsatisfied conditions заполняют половину регистра (16 бит).**

---

### E4. Нейтрально-барьерный трейдофф [Z2 + Z3]
```
Эффективные DOF(d) = N(d) - 32 × B(d)
```

| d  | N(d) | 32×B(d) | Eff.DOF |
|----|------|---------|---------|
| 0  | 78   | 470     | -392    |
| 9  | 153  | 544     | -391    |
| 12 | 90   | 42      | +48     |

**Только d=12 даёт положительные effective DOF!**
При d=12: 48 свободных бит после покрытия барьера.

---

### E5. Закон структурной энтропии [Z4 + Z8]
```
S(R) = 256 - D(R) = 256 × exp(-(R/4.5)²)
```
S(R) = "структурная информация" — биты, несущие управляемую структуру.

S(R=4) ≈ 209 бит структуры (80% сохранена)
S(R=8) ≈ 128 бит (50%)
S(R=16) ≈ 0 бит (0% — вся структура уничтожена)

**Скорость потери структуры: dS/dR ≈ -2R/20.25 × 256 × exp(-(R/4.5)²)**
Максимальная при R ≈ 4.5 раунда (≈ -40 бит/раунд).

---

### E6. Закон GA-потолка [Z5 + Z7]
```
Δ_total = Δ_GA × R_active / √R_active = 9√R_active
```
Поскольку барьеры независимы (Z7), GA оптимизирует каждый независимо.
Суммарная экономия за R_active раундов: 9 × R_active бит.
Но это НЕ компаундится: каждый барьер 32-9=23 бита.

Для 48 пост-барьерных раундов: cost = 48 × 23 = 1104 бит.
Это sequential — birthday (128) всегда лучше.

---

### E7. Закон каскадного бюджета [Z2 + Z6]
```
N_per_cascade = N(d) / D_casc
```
82 бит / 2 раунда = 41 бит на раунд каскада.
153 бит / 2 = 76.5 бит на раунд.

Стоимость барьера через каскад: 32 - 41 = отрицательная → барьер бесплатен!
Но только для 2 раундов (Z6).

---

### E8. Принцип ортогональной блокировки [Z11 + Z9]
```
deg_total(R) = deg_Ch(R) × deg_carry(R) → растёт как ≈ 2 × R²
```
Но измеренное: deg ≈ R (линейно, Z9).
Разница: ортогональность (Z11) означает что степени НЕ перемножаются,
а складываются: deg ≈ max(deg_Ch, deg_carry) ≈ R.

**Следствие**: нелинейность SHA-256 растёт ЛИНЕЙНО, не квадратично.
Это слабее чем казалось, но всё равно достаточно для R ≥ 19.

## СЛОЙ 2: ТРЕТИЧНЫЕ ЗАКОНЫ (из E1..E8)

### T1. Уравнение оптимального дифференциала [E1 + E4]
E1 говорит: лучший d максимизирует R_max = 17 + √(N(d)/7).
E4 говорит: d=12 единственный с Eff.DOF > 0.

Противоречие! d=9 даёт R_max=22, но Eff.DOF < 0.
d=12 даёт R_max=20.6, но Eff.DOF = +48.

**Разрешение**: существуют ДВА режима атаки:
```
Режим A (поисковый): используй d=9, ищи среди 2^153 вариантов.
  R_max = 22, но стоимость ≈ 2^{C(22) - 153} = 2^{7×25 - 153} = 2^{22}

Режим B (конструктивный): используй d=12, строй решение.
  R_max = 20.6, стоимость ≈ 2^{C(20) - 48} = 2^{63-48} = 2^{15}
```

**T1 предсказывает: SHA-256/20 коллизия за 2^15 операций (режим B).**
**SHA-256/22 коллизия за 2^22 операций (режим A).**

---

### T2. Закон структурного окна [E3 + E5]
E3: фазовый переход при C = 16 бит.
E5: структурная энтропия S(R) = 256 exp(-(R/4.5)²).

Атака работает пока S(R) > C(R):
```
256 exp(-(R/4.5)²) > 7(R-17)²
```
Это уравнение пересечения двух кривых.
При R = 18: S = 256×exp(-16) ≈ 0 vs C = 7. S < C.

**Проблема**: S(R) уже ≈ 0 при R=16, а C(R) начинает расти от R=17.
Окно перекрытия ПУСТОЕ для полного state-дифференциала.

Но для De-only: S_e(R) = 32 × metric(R) из Z1.
Wang chain поддерживает S_e ≈ 32 до R=16.
S_e(17) ≈ 32×(1-0.11) = 28, S_e(18) ≈ 32×0.57 = 18.

Окно S_e > C: 28 > 0 при R=17, 18 > 7 при R=18, 10 > 28 при R=19? Нет: S_e(19) ≈ 6, C=28. S_e < C.

**Структурное окно = раунды 17-18.** Ровно то что наблюдаем.

### T3. Принцип двойной оптимизации [E2 + E7]
E2: Cost(R) = 2^{7(R-17)² - 18} (с каскадом).
E7: каскадный бюджет = N(d)/2 бит на раунд.

Комбинируем: если каскад покрывает 2 раунда полностью,
то реальная стоимость начинается с R=19:
```
Cost_real(R) = 2^{7(R-19)²}    для R ≥ 19
```

| R  | Cost_real   |
|----|-------------|
| 19 | 2^0 (free!) |
| 20 | 2^7         |
| 21 | 2^28        |
| 22 | 2^63        |
| 23 | 2^112       |
| 24 | 2^175       |

**T3 сдвигает «нулевую стоимость» с R=17 на R=19!**
Каскад даёт 2 бесплатных раунда поверх базовой атаки.

---

### T4. Формула абсолютного предела [E1 + E6 + T3]
E1: R_max(9) = 22 (из нейтральных бит).
E6: GA даёт 9 бит/раунд но sequential.
T3: каскад сдвигает базу на +2 раунда.

Sequential стоимость с ВСЕМИ оптимизациями:
```
Cost_opt(R) = 2^{max(0, 7(R-19)² - 9×(R-19))}
```

| R  | 7(R-19)² | 9(R-19) | Cost_opt |
|----|----------|---------|----------|
| 19 | 0        | 0       | 2^0      |
| 20 | 7        | 9       | 2^0      |
| 21 | 28       | 18      | 2^10     |
| 22 | 63       | 27      | 2^36     |
| 23 | 112      | 36      | 2^76     |
| 24 | 175      | 45      | 2^130    |

**Пересечение с 2^128: R ≈ 24.**
Все наши оптимизации вместе дают атаку до ~24 раундов.

### T5. Принцип информационного каскада [E5 + E7 + Z6]
Структурная энтропия S(R) уничтожается со скоростью ~40 бит/раунд (E5).
Каскад восстанавливает ~41 бит/раунд (E7, для d=9).

40 ≈ 41 — **каскад ТОЧНО КОМПЕНСИРУЕТ потерю структуры!**

Но только на 2 раунда (Z6). Почему?
Потому что нейтральные биты КОНЕЧНЫ: 153 бит / 41 бит·раунд ≈ 3.7 раунда.
На практике: 2 раунда (интерференция между группами снижает до 2).

**Если бы N(d) = 300+: каскад продлился бы до 7 раундов.**
**Если бы N(d) = 600+: структурное окно сдвинулось бы до R ≈ 25.**

### T6. Уравнение необходимого числа нейтральных бит [T4 + T5]
Для атаки на R раундов полной SHA-256 (стоимость < 2^128):
```
N_required(R) ≈ 41 × (R - 17) + 128
```
R=17: N = 128 (покрыть стоимость барьеров)
R=24: N = 128 + 287 = 415 бит
R=64: N = 128 + 1927 = 2055 бит

**Для полной SHA-256: нужно 2055 нейтральных бит.**
Имеем: 153 максимум. Дефицит: 1902 бита.

### T7. Принцип нелинейного замедления [E8 + Z9]
Степень растёт как deg(R) ≈ R (линейно).
Это значит: на каждом раунде добавляется ОДНА степень нелинейности.

Количество мономов степени d в n переменных: C(n, d).
При n=512, d=20: C(512, 20) ≈ 2^100.
При d=64: C(512, 64) ≈ 2^300.

**Число мономов растёт как 2^{5×R} — ЭКСПОНЕНЦИАЛЬНО.**
Линейный рост степени создаёт ЭКСПОНЕНЦИАЛЬНЫЙ рост сложности.

### T8. Закон двух масштабов [T2 + T3]
В SHA-256 действуют ДВА временных масштаба:

```
τ_diff = 4.5 раунда   (диффузия, Z4)
τ_struct = 2 раунда    (каскадная оптимизация, Z6)
```

Отношение τ_diff / τ_struct = 2.25.
Это значит: диффузия уничтожает структуру в 2.25 раза быстрее,
чем оптимизация может её восстанавливать.

**Для прорыва нужно τ_struct ≥ τ_diff, т.е. каскад на 5+ раундов.**
Требует N ≥ 41 × 5 = 205 нейтральных бит (vs 153 имеющихся).

```
  Cost(25) = 2^{7×36 - 9×6} = 2^{198} (всё ещё > 128)

Если D_casc = 5:
  R_attack = 17 + 153/41 + 5 = 25.7
  Нужен N ≈ 205 для τ_struct = τ_diff.
  С двойным дифференциалом (P5): N ≈ 200 ≈ N_crit!
```

**КОНКРЕТНАЯ ЦЕЛЬ: доказать D_casc ≥ 4 с лучшей оптимизацией нейтральных бит + протестировать двойной дифференциал W[9]+W[12].**
