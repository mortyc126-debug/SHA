# SHA-256 РЕНТГЕН — Этап 3: Потоки данных через 64 раунда

## 1. ВРЕМЕННАЯ КАРТА (что происходит на каждом раунде)

```
ФАЗА 0: РОЖДЕНИЕ (r=0)
  Вход: IV + W[0]
  δstate = 4 бита
  δa = 1 бит (детерминировано)
  δe = 1 бит (детерминировано)
  Степень: 1 (линейно)
  Кривизна: 0 (аффинно)

ФАЗА 1: ВЗРЫВ (r=1-3)
  δstate: 4 → 25 → 55 → 87
  α = 6.3 → 2.2 → 1.6
  δa mod 8: только {1,7} на r=1
  Створочне χ²: 5000 → 369 → 4.5
  Word-level структура: БОГАТАЯ
  Bit-level структура: БОГАТАЯ
  Маячок: 248 → 192 → 122 det.бит

ФАЗА 2: НАСЫЩЕНИЕ (r=4-7)
  δstate: 87 → 115 → 126 → 128
  α → 1.00
  mod 8: UNIFORM (χ² < 2)
  Кривизна: 0 → 0.50 (фазовый переход на r=4!)
  Маячок: 56 → 7 → 0 det.бит (МЁРТВ к r=6)
  Word-level: UNIFORM
  Степень: → 32 (максимальная)

ФАЗА 3: WANG ZONE (r=1-16, при Wang cascade)
  δe = 0 (гарантировано, P=1.0)
  δa = 16 бит (насыщен с r=5)
  δf = δg = δh = 0 (из δe=0 через shift)
  δb, δc, δd = копии δa (shift)
  HW(δstate) = 64 (половина: δe-ветвь = 0)
  GPK match: 3/7 операций = 100%
  Каждый раунд: 1 слово W «потрачено» на δe=0

ФАЗА 4: БАРЬЕР (r=17)
  W[16] из schedule (не свободный!)
  δe[17] = δa[13] + δW[16] mod 2³²
  P(δe[17]=0) = 2⁻³² (uniform)
  δstate: 64 → 96 (δe≠0 впервые)
  Birthday стоимость: O(2³²) для δe[17]=0

ФАЗА 5: ЗОНА 2 (r=17-59)
  δstate = 128 ± 8 (термостат)
  Все метрики = random:
    HW = 128, mod k = uniform
    Корреляция между раундами = 0
    P-count = 16/32 на каждом раунде
    GPK-diff = 112/224 = 50%
  Shield potential = 0.50 (случайная монетка)
  43 раунда × 16 P-бит = 688 бит угадывания

ФАЗА 6: ПОЗДНИЕ РАУНДЫ (r=60-63)
  Backward divergence: 128 → 64 → 48 → 32 → 16
  При carry[63]=0:
    Ch[62][b30,b31] = 0 (детерминировано)
    g[62][b30,b31] = 0 (через Ch → SS)
    e[60][b30,b31] = 0 (горлышко)
  Bridge: carry → H[7][b29] с phi=+0.129

ФАЗА 7: ФИНАЛИЗАЦИЯ (r=64)
  H[i] = state[64][i] + IV[i]
  Feedforward: одно сложение, 1 carry-событие
  H = 256 бит, попарно независимые
  κ = 0 (невидимое звено a[60] ∉ H)
```

## 2. КАРТА ЗАВИСИМОСТЕЙ (что от чего зависит)

```
           Раунды 0-15         Раунд 16        Раунды 17-63
         ┌────────────┐      ┌──────────┐     ┌─────────────┐
W[0..15] → schedule ─────────→ W[16..63]──────→ (фиксировано)
    │     ├ σ₀, σ₁           │                │
    ↓     │ линейно/GF(2)    │                │
  IV ───→ state[0] ──rounds──→ state[16] ─────→ state[64] → H

         └────────────┘      └──────────┘     └─────────────┘
              ↑                    ↑                ↑
         κ=0 (L=4)          barrier 2³²     thermostat HW*=128
```

## 3. CARRY FLOW (как carry течёт внутри раунда)

```
Раунд r: 7 сложений, каждое порождает 32-битный carry-вектор

       h ─────┐
    Σ₁(e) ────┼─(+)─→ sum1 ─┐
    Ch(e,f,g)─┘             ├─(+)─→ sum2 ─┐
                            │              ├─(+)─→ sum3 ─┐
                     K[r] ──┘              │              ├─(+)─→ T1
                                    W[r] ──┘              │
                                                          │
    Σ₀(a) ────┐                                          │
    Maj(a,b,c)┼─(+)─→ T2                                │
              │                                          │
              │         d ─────┐                         │
              │                ├─(+)─→ e' = d + T1       │
              │         T1 ───┘                          │
              │                                          │
              └──→ T1 ─┐                                 │
                       ├─(+)─→ a' = T1 + T2              │
              T2 ─────┘

Итого: 4 carry в T1-цепочке, 1 в T2, 1 в e', 1 в a' = 7 carry × 32 бит = 224 бит
Из них P-бит: 112 (50%) — carry неопределён без знания операндов
```

## 4. SHIFT REGISTER FLOW (как данные копируются)

```
Раунд r:      a[r]  b[r]  c[r]  d[r]  e[r]  f[r]  g[r]  h[r]
                ↓     ↓     ↓                ↓     ↓     ↓
Раунд r+1:   a[r+1] a[r]  b[r]  c[r]  e[r+1] e[r]  f[r]  g[r]
              (новый)(копия)(коп.)(коп.) (новый)(копия)(коп.)(коп.)

6 из 8 регистров = побитовые копии предыдущего раунда.
2 из 8 (a,e) = нелинейные функции всех 8 + W[r] + K[r].

Через 4 раунда: a[r] проходит a→b→c→d и «падает» за край.
a[r-4] = d[r] → e[r+1] (через d+T1). Но a[r-4] не в выходе.
```

## 5. ИНФОРМАЦИОННЫЙ ПОТОК (M → H)

```
Раунд 0-4:    сигнал «открытый» — один бит state = M[0] bit
Раунд 5:      сигнал «закодирован» — ~128 бит state зависят от M
Раунд 8+:     Oracle Distance ≈ 5.3 — неотличим от random

Два канала передачи:
  КАНАЛ 1 (копирование):  192 бит/раунд, точная копия, 3 раунда жизни
  КАНАЛ 2 (пересоздание): 64 бит/раунд (a,e), нелинейное преобразование

Цикл: τ★ = 4 раунда
  Каждый бит проходит 3 раунда копирования + 1 пересоздание = 4 раунда.

Ляпунов: λ ≈ 0 для r ≥ 7 (нейтральный, не хаотический).
  Не растёт, не затухает. Осциллятор в равновесии HW=128.
```

## 6. COLLISION EQUATION (уравнение коллизии)

```
SHA(M₁) = SHA(M₂)
⟺ state[64](M₁) = state[64](M₂)
⟺ δstate[64] = 0
⟺ δa[61..64] = 0 AND δe[61..64] = 0     (через shift register)
⟺ 256 бит условий на 512 бит входа

Wang: обнуляет δe[2..16] бесплатно (16 условий).
Барьер: δe[17] = δa[13] + δW[16] = 2⁻³² (1 условие).
Зона 2: δe[18..63] = 46 × 2⁻³² каждый (46 условий).

Если все условия независимы:
  P(collision) = 2⁻(32×48) = 2⁻¹⁵³⁶
  Но birthday на 256-бит выходе: 2⁻²⁵⁶
  Birthday стоит: 2¹²⁸

Термостат: escape cost = 2¹⁸⁶ > 2¹²⁸ (birthday).
→ Birthday ОПТИМАЛЕН. Прямое преодоление термостата ДОРОЖЕ.
```
