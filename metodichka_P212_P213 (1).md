# П-212..П-213: Три направления + Birthday МИТМ

## П-212: Три ключевых результата

### A: Структура δstate[17] — уточнение

Сдвиговый регистр даёт:
```
δb[17] = δc[17] = δd[17] = 0   (доказано аналитически)
δf[17] = δg[17] = δh[17] = 0   (из δe[14..16]=0)
```
**192 always-zero бит. Ненулевые: только δa и δe.**

Ранг (δa,δe)[17] = **64 / 64** → МИТМ O(2^32).

### B: Фиксация W0 снижает ранг вдвое

При W0=0xc97624c6 (из П-97):
```
Ранг(δa,δe)[17] = 32 / 64  → МИТМ O(2^16)!
```
При случайном W0: ранг=64. Фиксация W0 добавляет структуру.

### C: Truncated SHA-256

SHA-256/k collision стоит O(2^(k/2)):
- SHA-256/32: O(2^16) ← подтверждено birthday
- SHA-256/64: O(2^32)
- Full SHA-256: O(2^32) по (δa,δe)[17]

## Концептуальное прояснение

**Birthday collision (δa,δe)[17]** означает:
- Два разных трейла (W₁,W₁') и (W₂,W₂') имеют одинаковый δstate[17]
- Это NOT означает что δstate[17]=0
- Для атаки нужна КОНКРЕТНАЯ целевая δstate

**Что это дает:**
Если построить таблицу из 2^32 пар с разными δstate[17],
среди них будет ~1 пара с δa[17]=0 (и отдельно ~1 с δe[17]=0).
Пара с δa[17]=δe[17]=0: нужно 2^64 — недостижимо напрямую.

**Но:** при W0=фикс ранг=32 → можно найти δa[17]=0 за 2^32 пар,
и среди них δe[17]=0 за ещё 2^32 → итого 2^32 (не 2^64!).

## Дорожная карта: P(δa[17]=δe[17]=0) при W0=фикс

При фиксированном W0:
- δe[17] определяется цепочкой δe[1]=d0=1 → T1[1] → ... → δe[17]
- При W0=фикс δe[1]=d0=const → δe[17] имеет фиксированную структуру
- Ранг(δe[17]|W0=фикс) = 32/32 (случайный, но зависит от W[1..15])

**P(δa[17]=0 И δe[17]=0 | W0=фикс) = 2^{-32} × 2^{-0?}**

Нужно проверить: при W0=фикс — независимы ли δa[17] и δe[17]?
