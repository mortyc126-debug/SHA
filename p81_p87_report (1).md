## КОНТЕКСТ

До П-81 было установлено (методичка П-1..П-80):
- **T_CASCADE_17**: 15 нулей De3..De17 за 2^32 через адаптивный каскад
- **T_BARRIER17_EXACT**: Da13 + DW16 ≡ 0 mod 2^32 (984/984 верифицировано)

- **T_INFINITE_TOWER**: slope=1.000 до k=24, height_2(SHA-256) ≥ 32
- **T_RANK5_INVARIANT**: rank_GF2(J_{5×15}) = 5, |Sol_1| = 1024
- Пара верификации: W0=e82222c7, W[1]=516cfb41, DW0=1, De3..De17=0

## П-81..П-83: ДИАГНОСТИКА — КОРНЕВАЯ ПРИЧИНА

### Проблема
Все эксперименты П-81..П-83 давали de_zeros=0 для известной пары.
При 5M случайных пар максимум нулей De3..De17 = 0.

### Корневая причина (П-83)
**W[2..15] ≠ 0 в реальных парах.** Пара (e82222c7, 516cfb41) получена
через T_CASCADE_17 с адаптивными DW[1..15], не с W[2..15]=0.

При W[2..15]=0:
```
DW16 = W_a[16] - W_b[16]
     = (σ0(w1) + W0) - (σ0(w1) + W0 + 1)
     = -1 = 0xFFFFFFFF  (всегда!)
```
Это делает De3..De17=0 недостижимым при W[2..15]=0.

### Верификация B5
```
Da13(e82222c7, 516cfb41) = a9bca814  (B5 утверждает 7711498a — разные!)
DW16 = 0xffffffff
Da13 + DW16 = a9bca813 ≠ 0
```
Пара из B5 требует АДАПТИВНЫХ W[2..15], которые были вычислены каскадом.

## П-84: ВОССТАНОВЛЕНИЕ CASCADE_17

### Правильная модель
Пара: (W_a, W_b) где **W_b = W_a + DW** (аддитивный дифференциал).
- DW[0] = 1 (фиксировано)
- DW[1..15] адаптируются последовательно: для каждого r=1..15
  вычисляется DW[r] = -De_r(Wa,Wb) / (dDe_r/dWb[r]) через ньютоновский шаг

### Алгоритм
```c
// Для r=1..15:
uint32_t cur = De_r(Wa, Wb, r);       // текущая разность
if (cur == 0) continue;
Wb[r] += 1;
uint32_t dDe = De_r(Wa,Wb,r) - cur;  // чувствительность
Wb[r] -= 1;
// Ньютон (если dDe нечётное):
uint32_t inv = mod_inverse(dDe);
Wb[r] = Wa[r] + (0u - cur) * inv;
```

### Результат П-84
- 14/15 нулей (De[2..16]=0) за **O(1)** для любого w1
- 100% случайных w1 → 14 нулей (верифицировано на 10 000 парах)
- Единственный барьер: **De17 = 0** требует birthday O(2^32)

### DW для пары (W0=e82222c7, w1=516cfb41)
```
DW[0]  = 00000001  (фиксировано)
DW[1]  = 0be0017f  → De[2]=0
DW[2]  = 00000001  → De[3]=0
DW[4]  = fffffffe  → De[5]=0
DW[5]  = 3ff7fc00  → De[6]=0
DW[6]  = 4ff3fe3f  → De[7]=0
...
DW[15] = 9baaf2b3  → De[16]=0
De[2..16] = 00000000000000X  (14/15 ✓, De17 = 18541585 ≠ 0)
```

## П-85: ОПТИМИЗАЦИЯ + НЕЙТРАЛЬНЫЕ БИТЫ

### Скорость
- Базовый каскад: ~130k/s (1 CPU)
- ETA birthday 2^32: ~9 часов (1 CPU), ~1.1 ч (8 потоков)

### GF(2) якобиан в DW-пространстве
```
J[15×16]: строки = De[3..17], столбцы = DW[0..15]
rank_GF2 = 14 / 15
ядро dim = 1 (одна степень свободы)
```
Структура **нижнетреугольная** — строки De[16] и De[17] совпадают,
что является прямым подтверждением **T_DE17_LINEAR**: De17 линейно
зависима от De[3..16] в GF(2).

### Нейтральные биты
Нейтральных битов w1 не обнаружено (0/640 одиночных флипов → 15 нулей).
Это ожидаемо: нейтральные биты применимы только после нахождения пары
с De17=0, когда флип бита w1 сопровождается пересчётом всего каскада.

## П-86: HENSEL + НЕЙТРАЛЬНЫЕ БИТЫ ПОЛНЫЙ АНАЛИЗ

### De17 распределение (T_STATE17 подтверждение)
```
v2(dDe17/dw1):  0→50%,  1→24%,  2→13%,  3→6%...
```
Геометрическое убывание — классическое p-адическое распределение.
De17 ~ Uniform(Z/2^32Z) ✓

### Hensel tower (исправленный)
Сид: все w1 ∈ {0..255} с De17 чётным (127 из 256):
```
k=1:  127 сидов
k=2:  120
k=3:  106
k=4:   84
k=5:   42
k=6:   24
k=7:    0  ← башня рухнула
```
**T_HENSEL_INAPPLICABLE подтверждена** для DW-cascade пространства.
Нелинейность SHA-256 (Ch, Maj, carries) блокирует 2-адический подъём
на k=7 — так же как в П-43 для обычного w1-пространства.

### Новая теорема
```
T_HENSEL_CASCADE_INAPPLICABLE (П-86):
  Hensel lifting De17 ≡ 0 mod 2^k в DW-cascade пространстве
  рушится на k=7. Подтверждает T_HENSEL_INAPPLICABLE (П-43)
  в расширенном пространстве.
```

## П-87: 64-BIT CASCADE

### Гипотеза
W_a = [W0, w1, w2, 0, ...] — добавление w2 как второго birthday
параметра может снизить стоимость через структуру De17(w1,w2).

### Результаты

**1. Нелинейность De17(w1,w2)**
```
Линейная модель base+A*w1+B*w2:   0/2000 OK
Аддитивное f(w1)+g(w2):           0/2000 OK
Cross-term h(w1,w2)=0:            0/1000
```
De17(w1,w2) сильно нелинейна. MITM неприменим.

**2. GF(2) ранг**
```
rank_GF2(J[15×17]) = 14  (без изменений vs J[15×16]=14)
```
w2 не добавляет нового ранга — линейно зависима от остальных столбцов.

**3. Структурная причина**
W[2] = W_a[2] входит в расписание начиная с W[18]:
```
W[18] = σ1(W[16]) + W[11] + σ0(W[3]) + W[2]
```
Это раунд 19 — за пределами окна De3..De17. Внутри 17 раундов W[2]
уже адаптирован каскадом (DW[2] задан для зануления De[2]).

**4. Новая теорема**
```
T_64BIT_CASCADE_NEGATIVE (П-87):
  Добавление w2=W_a[2] как второго свободного параметра не снижает
  стоимость birthday search:
  - De17(w1,w2) сильно нелинейна (аддитивность 0/2000)
  - rank_GF2(J[15×17]) = 14 = rank_GF2(J[15×16])
  - P(De17=0 | rand w1,w2) ≈ 2^{-32} (без улучшения)
  - ETA birthday: ~10.3 ч при 116k/s
  Причина: W[2] адаптирован каскадом, второй параметр не независим.
```
