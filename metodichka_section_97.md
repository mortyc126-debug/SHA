## T_DELTA_CARRY_DETERMINISM — главный результат

### Формулировка

```
При условии δstate[0..r] = 0 и δW[r] = 1 (однобитовый):

P(δcarry[r] = 0) = 1.0
```

Для всех r = 0..15. Верифицировано N=50 000 пар каждый раунд.

### Аналитическое доказательство

Раунд r при δstate=0:
```
T2[r] одинаков для M и M'  (зависит только от state)
T1[r]  = (fixed) + W[r]
T1'[r] = (fixed) + W[r]⊕1
```

Carry определяется знаком T1+T2:
```
carry  = 1  ⟺  T1  + T2 > 2^32
carry' = 1  ⟺  T1' + T2 > 2^32
```

T1 и T1' отличаются ровно на 1 в младшем бите.
Случай δcarry ≠ 0 требует что T1+T2 = 2^32 точно.
P(случайное 32-битное число = конкретному значению) = 2^{-32} ≈ 0.

Поэтому δcarry = 0 с вероятностью 1 - 2^{-32} ≈ 1.0. ✓

### Связь с T_HALFSTATE

T_HALFSTATE (δe=δf=δg=δh=0 для r=4..16) теперь имеет
аналитическое основание:

```
δstate[0..r] = 0  →  T_DELTA_CARRY_DETERMINISM
             →  δcarry[r] = 0
             →  δa[r+1] = δW[r] ⊕ δ(linear terms) = детерминировано
             →  δe[r+1] = δd[r] + δT1[r] = детерминировано
```

T_HALFSTATE — следствие T_DELTA_CARRY_DETERMINISM.

## T_K0_SPECIAL — структура начальных констант

### Результат

```
T1_base[0] + T2[0] = 0xFC0888844 = 0.9845 × 2^32
```

Фиксированные компоненты первого раунда занимают 98.5% диапазона.
Carry возможен только при W[0] > 0x03F777B2 (3.5% входов).

При случайных H0 и K: P(carry[0]=0) = 0.467.
При реальных H0 и K: P(carry[0]=0) = 0.967.

**Раунд 0 — структурно особенный.** Конструкторы SHA-256
выбрали H0 через квадратные корни простых чисел —
это создаёт специфическое распределение carry в раунде 0.

## Новые принципы

**П-24:** T_HALFSTATE — следствие T_DELTA_CARRY_DETERMINISM.
Барьер существует потому что δcarry=0 детерминированно
пока δstate=0.

любой раунд с нулевым δstate — независимо от P(carry).

P(carry[0]=0)=0.967 vs 0.5 для случайных констант.
Это заложено в выборе H0 через √простых чисел.
