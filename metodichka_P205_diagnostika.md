# П-205: Диагностика результатов MILP — важное исправление

## Контекст

В П-204 MILP (PuLP + CBC) выдал `HW(ΔW[17]) = 0` с решением:
```
ΔW[0] = 0x00000000  HW=0
ΔW[1] = 0x00000000  HW=0
ΔW[2] = 0x00000000  HW=0
ΔW[3] = 0xffffffff  HW=32
```

## Диагностика: тривиальное решение

**Проблема в MILP constraint:**
```python
# Неправильно:
sum(dw[r,b] for r in range(4) for b in range(32)) >= 1
# Это допускает ΔW[0]=0 при ΔW[3]≠0

# Правильно:
sum(dw[(0,b)] for b in range(32)) >= 1
# Нужно именно ΔW[0]≠0 — это начальное возмущение Wang chain
```

**Почему ΔW[3] не помогает:**

Формула: `ΔW[17] = sig0(ΔW[2]) ⊕ ΔW[1]`

ΔW[3] не входит в эту формулу. При ΔW[1]=ΔW[2]=0:
```
ΔW[17] = sig0(0) ⊕ 0 = 0 ⊕ 0 = 0
```
Это тождественно ноль независимо от ΔW[3].

**Физический смысл:** ΔW[0]=0 означает отсутствие начального возмущения → (M, M) одинаковые → не дифференциальная атака.

## Правильные результаты

| Метод | Результат | Статус |
|-------|-----------|--------|
| MILP (неправильный constraint) | HW=0 | ❌ ТРИВИАЛЬНО |
| Числовой поиск 700K пар | HW=3 | ✓ РЕАЛЬНЫЙ |
| Числовой поиск 500K пар | HW=4 | ✓ РЕАЛЬНЫЙ |
| Gurobi compact MILP (193 перем.) | HW=0 | ❌ ТА ЖЕ ПРОБЛЕМА |

## Реальный минимум HW(ΔW[17]) при Wang chain

```
Из 700K пар за 145 сек (Ryzen 5 3600):
  Лучший HW(ΔW[17]) = 3
  d0 = 0x59e4c20a
  DW[1]=..., DW[2]=...
```

Это **реальный результат** — сравнимо с Li et al. (3..5 бит/раунд).

## Исправленная формулировка

Правильная задача MILP:

```python
# Переменные: биты ΔW[0..3]
# ПРАВИЛЬНЫЙ constraint:
prob += sum(dw[(0,b)] for b in range(32)) >= 1  # ΔW[0] ≠ 0

# Дополнительно — Wang chain нелинейность:
# ΔW[1] зависит от δΣ1(e[1]) которое нелинейно по ΔW[0]
# Поэтому ΔW[1,2] не свободны при фиксированном ΔW[0]
# Нужен полный 16-раундовый MILP с нелинейными ограничениями
```

## Аналитический результат (верифицирован)

**Что верно:** аналитика XOR-расписания корректна:
```
ΔW[17]_XOR = sig0(ΔW[2])_XOR ⊕ ΔW[1]_XOR
```

**Где ошибка:** Wang chain работает в **аддитивной** алгебре Z_{2^32}, не XOR.
Аддитивное ΔW[17] ≠ XOR ΔW[17] из-за переносов.

**Поэтому:** формула `ΔW[17] = sig0(ΔW[2]) + ΔW[1]` (аддитивно) верна приближённо, но коэффициенты зависят от конкретных значений W[1], W[2] через переносы.

## Вывод для методички

**НЕ записывать:** "MILP доказал HW(ΔW[17])=0" — это тривиальное решение.

**Записывать:**
1. Числовой поиск: min HW(ΔW[17])=3 за 700K пар — **реальный результат**
2. XOR-формула приближённо верна: `ΔW[17] ≈ sig0(ΔW[2]) ⊕ ΔW[1]`
3. Для точного MILP нужен правильный constraint: `ΔW[0] ≥ 1`
4. Полный 16-раундовый MILP требует академической лицензии Gurobi

## Следующий шаг (П-206)

Исправленный компактный MILP:
```python
# Constraint: ΔW[0] ≠ 0 (не сумма всех ΔW)
prob += sum(dw[(0,b)] for b in range(32)) >= 1

# Минимизировать: HW(ΔW[17]) = HW(sig0(ΔW[2]) + ΔW[1])
# с аддитивным сложением (carry bits)
# при условии: ΔW[1] совместим с Wang chain δe[1]=0
```

Это задача с ~300 переменными — в лимите бесплатной лицензии.
