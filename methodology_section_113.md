### T_CARRY_PATTERN (П-470) ★★★★

**Утверждение:** Паттерн смещений (−, +, +, −) в битах b28-b31 слова H[7]
является подписью carry-propagation при финальном сложении.

**Механизм:**
```
carry[62]=0 → T1[62] специфическое значение
            → h[63] смещён (малое значение)
            → H[7] = h[63] + IV[7]
            → carry распространяется через биты 28-31:
              b28: отрицательный (carry погашен)
              b29: положительный (сильнейший сигнал)
              b30: положительный (второй сигнал)
              b31: отрицательный (carry вышел)
```

**Это классическая carry-signature в модульном сложении.**

## ФОРМАЛИЗАЦИЯ АТАКИ

```
CHOSEN-PREFIX DISTINGUISHER SHA-256 v1.0

Вход:   оракул O(·) — либо SHA-256, либо случайная функция
Выход:  "SHA-256" или "случайная"
Параметр: N (число запросов)

Алгоритм:
  count_11 = 0
  for i = 1..N:
    W[0] ← HC_optimize(150 шагов)
    H ← O(W[0])
    if H[7][bit29]=1 AND H[7][bit30]=1:
      count_11 += 1

  P_observed = count_11 / N
  if P_observed > 0.42:  # threshold (random: 0.25)
    return "SHA-256"
  else:
    return "случайная"

Сложность: O(N × 55,000 SHA-256 вычислений)
Advantage: ~0.14 (один бит) до ~0.20 (комбинация)

Доказательство корректности:
  P(b29=1 AND b30=1 | SHA-256, HC) ≈ 0.64 × 0.63 ≈ 0.40
  P(b29=1 AND b30=1 | случайная)   = 0.5 × 0.5 = 0.25
  Advantage = 0.40 - 0.25 = 0.15
```
