## T_SCHED_FULL_RANK — главный результат

### Формулировка

```
Матрица S_GF2: GF(2)^512 → GF(2)^1536

rank(S_GF2) = 512  (полный ранг по столбцам)

Следствия:
  dim(Ker S_GF2)  = 0
  dim(Im  S_GF2)  = 512
  Число независимых уравнений: 512
  Число избыточных уравнений: 1536 - 512 = 1024
```

### Верификация

Гауссово исключение над GF(2) на матрице 512×1536.
Результат: ранг = 512. Нулевых опорных элементов не найдено.

Дополнительная проверка: ранг каждого из 48 выходных слов = 32/32.
Все 48 слов полного ранга — нет слов с линейной зависимостью внутри.

## T_SCHED_CONSTRAINT_PROB — мощность schedule constraint

### Формулировка

```
Пусть v ∈ GF(2)^1536 — случайный равномерный вектор.

P(v ∈ Im S_GF2) = 2^{dim(Im) - dim(пространства)}
                = 2^{512 - 1536}
                = 2^{-1024}
```

### Интерпретация для KSIA

```
В KSIA атаке мы получаем W_rec[16..63] из обратного прохода.
Для случайного неправильного carry-профиля:

P(W_rec[16..63] ∈ Im S_GF2) = 2^{-1024}

Это означает: из 2^18 carry-профилей только ~2^18 × 2^{-1024}
удовлетворяют schedule constraint в XOR-модели.

Так как 2^18 << 2^1024, вероятность = 0 на практике.
Правильный профиль — единственный который проходит.
```

### Замечание о нелинейной поправке

В реальном schedule (над Z/2^32) добавляется carry_sched:

```
W_rec[16..63] = S_GF2(W_rec[0..15]) + carry_sched

Условие совместности меняется: нужно чтобы
W_rec[16..63] - carry_sched ∈ Im S_GF2
```

carry_sched известен при известном W_rec[0..15].
Но W_rec[0..15] — это то что мы ищем.

## T_SCHED_MIN_CHECK — минимальная проверка

### Формулировка

```
Для проверки schedule constraint достаточно любых
512 линейно независимых уравнений из 1536.

Например: W_rec[16..31] (первые 16 слов × 32 бит = 512 уравнений)
при условии что соответствующая подматрица имеет ранг 512.
```

### Проверка

Ранг подматрицы W[16..31]: все 16 слов имеют ранг 32/32.
Суммарный ранг 16 слов = min(512, 512) = 512. ✓

Достаточно проверить W_rec[16..31] = schedule(W_rec[0..15])[16..31].
Остальные 1024 уравнений выполнятся автоматически.

## Геометрия schedule constraint

```
Пространство W[16..63]:           GF(2)^1536
Образ schedule Im S_GF2:          512-мерное подпространство
Коразмерность (codim):            1024

Аналогия: прямая (1D) в трёхмерном пространстве (3D)
Schedule constraint — это попадание в очень тонкое подпространство.

Вероятность случайного попадания: 2^{-1024}
Число точек в образе: 2^{512}
Число точек вне образа: 2^{1536} - 2^{512} ≈ 2^{1536}
```

```
T_PROFILE_SPACE (раздел 99):
  ~2^18 достижимых carry-профилей из 2^64

T_SCHED_CONSTRAINT_PROB (раздел 102):
  P(случайный W_rec ∈ Im S) = 2^{-1024}

Совместно:
  KSIA перебирает 2^18 профилей
  Каждый даёт W_rec не в Im S с вероятностью 1 - 2^{-1024}

Сложность нахождения правильного профиля
определяется не 2^18, а структурой Im S.
```

## Новые принципы

**П-44:** 1024 из 1536 уравнений schedule constraint избыточны.
Для проверки достаточно 512 — первые 16 выходных слов.

**П-45:** Образ schedule — 512-мерное подпространство в 1536-мерном.
P(случайное W_rec ∈ Im S) = 2^{-1024}.

**П-46:** Schedule constraint в KSIA непреодолим не из-за сложности
вычислений, а из-за геометрии: образ schedule — тонкое подпространство.
