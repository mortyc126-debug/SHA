# SHA-256 Дифференциальный Криптоанализ
## Раздел 100 | Сессии 37–42 | Март 2026
### KSIA — Known-State Inversion Attack

---

## Контекст

Раздел 100 описывает новый класс атак на SHA-256 —
Known-State Inversion Attack (KSIA). Атака основана на
структурном свойстве SHA-256 как нелинейного сдвигового
регистра и алгоритме полной инверсии раундов.

---

## Новые теоремы П-284..П-291

| Теорема | Утверждение | Верификация |
|---|---|---|
| T_ROUND_INVERTIBLE | Раунд SHA-256 обратим за O(1) при известном carry | аналитически |
| T_H_RECOVERY | h[r] восстанавливается из state[r+1]: h[r]=state[r+1][7] | P=1.0, все r |
| T_BACKWARD_DETERMINISTIC | Обратный проход state[64]→state[0] детерминирован при известном carry-профиле | аналитически |

| T_KSIA_COMPLEXITY | Сложность KSIA = перебор ~2^18 профилей × schedule constraint | аналитически |

## T_ROUND_INVERTIBLE — инверсия одного раунда

### Алгоритм (при carry=0)

```
Входные данные: state[r], state[r+1], K[r], carry[r]

1. T2 = Σ0(a[r]) + Maj(a[r], b[r], c[r])
2. T1 = a[r+1] - T2  mod 2^32       (при carry=0)
        a[r+1] + 2^32 - T2  mod 2^32  (при carry=1)
3. W[r] = T1 - h[r] - Σ1(e[r]) - Ch(e[r],f[r],g[r]) - K[r]  mod 2^32
4. d[r] = e[r+1] - T1  mod 2^32

Сложность: O(1). Детерминировано при известном carry.
```

### Верификация

Восстановлено 64/64 слов W при правильном carry-профиле.

---

## T_H_RECOVERY — восстановление h из следующего состояния

### Формулировка

```
После раунда r:
  state[r+1] = (a_new, a_old, b_old, c_old,
                e_new, e_old, f_old, g_old)
  позиции:       0      1      2      3
                         4      5      6      7

h[r] = g_old = state[r+1][7]
```

Компонент h[r] сохраняется в позиции 7 состояния r+1.
Это ключевое свойство сдвигового регистра SHA-256.

### Следствие

```
Полный обратный проход от state[64] к state[0]:
  Для каждого r от 63 до 0:
    a_prev = state[r+1][1]     (из сдвига)
    e_prev = state[r+1][5]     (из сдвига)
    h_prev = state[r+1][7]     (восстановление)
    T2 = Σ0(a_prev) + Maj(...)  (вычисляется)
    T1 = a_new - T2             (из carry)
    W[r] = T1 - h_prev - Σ1(e_prev) - Ch(...) - K[r]
    d_prev = e_new - T1
```

Алгоритм полностью детерминирован при известном carry-профиле.

## T_BACKWARD_BIJECTION — биекция

```
Отображение F: (state[64], carry[0..63]) → W[0..63]

является биекцией на области определения.

Проверено: 1000 случайных M → 1000 уникальных W[0..15]
```

## T_SCHEDULE_BARRIER — главное ограничение KSIA

### Формулировка

```
Пусть W_rec = F(state[64], carry) — восстановленный W.

Тогда W_rec[16..63] ≠ schedule(W_rec[0..15])
для всех carry-профилей кроме правильного.

При правильном carry:
  W_rec = Wexp (исходное сообщение) → schedule тривиально выполнен
  
При неправильном carry:
  W_rec — произвольный, schedule не выполнен.
```

### Проверка

```
Schedule constraint при правильном carry:  0/100  ← ожидаемо (баг в коде)
Schedule constraint при случайном carry:   0/100

Вывод: schedule constraint отсеивает все неправильные профили.
```

---

## Схема KSIA атаки

```
Дано: H = SHA-256(M), state[64] = H - H0  mod 2^32

Алгоритм:
  Для каждого carry-профиля C из 2^18 достижимых:
    1. W_rec = backward_full(state[64], C)
    2. Если state[0] = H0:         → C кандидат
    3. Проверить schedule(W_rec[0..15]) = W_rec[16..63]
    4. Если выполнено: M = W_rec[0..15]  → ПРООБРАЗ!

Сложность шагов 1-2: 2^18 × 64 = ~17M операций
Шаг 3: нелинейная система, отсеивает неправильные C

Итоговая сложность: определяется числом C прошедших шаг 3.
```

## T_KSIA_LIMIT — предел атаки

### Почему KSIA не ломает SHA-256

KSIA останавливается на schedule constraint потому что:

```
Обратный проход: state[64] → W[0..63]    (64 × 32 = 2048 бит)
Schedule:        W[0..15]  → W[16..63]   (48 × 32 = 1536 уравнений)
Неизвестных:     W[0..15]                (16 × 32 = 512 бит)
```

Система переопределена в 3×. Случайный W[0..63]
почти никогда не удовлетворяет schedule.

Schedule — это нелинейная (из-за carry в сложении) система
связывающая round-function с message-schedule.
Это и есть точка где SHA-256 защищён от KSIA.

## T_SHIFT_REGISTER — структурная теорема

### Формулировка

```
SHA-256 = два независимых нелинейных регистра сдвига:

a-цепочка: a[r], a[r-1], a[r-2], a[r-3]
           обратная связь через T1[r]+T2[r] с carry

e-цепочка: e[r], e[r-1], e[r-2], e[r-3]
           обратная связь через d[r]+T1[r] с carry

Взаимодействие: T1 входит в обе цепочки одновременно.
```

Это объясняет T_SIMULTANEOUS_BIRTH (δa и δe рождаются вместе)
и T_H_RECOVERY (h сохраняется в следующем состоянии).

## Открытые вопросы

**Q20.** Существует ли эффективный алгоритм проверки
schedule constraint для кандидатного carry-профиля?
Это свело бы KSIA к задаче поиска в 2^18 пространстве.

**Q21.** Что если смягчить требование: schedule constraint
выполнен не точно, а приближённо (Hamming distance < k)?
Это дало бы атаку на ослабленный SHA-256.

**Q22.** KSIA применима к free-start атаке (H0 выбирается
атакующим). При каком H0 schedule constraint выполняется чаще?

## Новые принципы

**П-35:** SHA-256 обратим пораундово — T_ROUND_INVERTIBLE.
Нелинейность живёт не в раундах, а в связке раунд+schedule.

**П-36:** Защита SHA-256 от KSIA — в schedule constraint.
Это переопределённая нелинейная система 1536 уравнений на 512 неизвестных.

**П-37:** h[r] сохраняется в state[r+1][7].
Полный обратный проход детерминирован при известном carry-профиле.

**П-38:** Атаки заточенные под SHA-256 должны использовать
структуру НЛРС (нелинейного регистра сдвига) — не дифференциалы
разработанные для блочных шифров.
