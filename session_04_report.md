## Новые теоремы

### T_CARRY_IMAGE_FORMULA
```
carry(0xFFFFFFFF, w) = ~(w XOR (w-1))
                     = маска всех бит выше LSB(w)
```
Верификация: 99999/99999 ✓ (аналитически доказуема)

**Смысл:** при t=0xFFFFFFFF carry полностью определяется позицией
младшего единичного бита w. Это линейная функция от lsb-позиции.

### T_CARRY_IMAGE_SIZE
```
|image(carry(t,·))| зависит от HW(t):
  HW(t) = 0:  |image| = 1    (детерминировано)
  HW(t) = 32: |image| = 17   (почти детерминировано)
  HW(t) = 16: |image| ≈ 2^16 (максимум)
```
Монотонная зависимость: больше единиц → меньше возможных carry.

**Смысл:** нули в t — это "стены" для переноса. При HW(t)=32
нет стен, перенос всегда проходит насквозь, образ carry мал.

### T_HALFSTATE_STRUCTURE (подтверждена, P=1.0)
```
δstate[r] = (δa[r], δa[r-1], δa[r-2], δa[r-3], 0,0,0,0)
для r=4..16 при каскаде δe[2..16]=0
```

### T_DW4_FORMULA (подтверждена)
```
dW[4] = -2·d0 mod 2^32, P=1.0
```

## Отрицательные результаты

- Симметрии SHA-256 (комплементарная, ротационная, перестановочная) — нет структуры
- Группа биекций SHA-256 — некоммутативна, циклов нет за 10K итераций
- Дифференциальный MITM — тривиален (δstate[64]=0 → δstate[r]=0 для всех r)
- Carry-паттерн каскадных пар в раундах 16+ — неотличим от случайного
- Экстремальные t_before_W — недостижимы через выбор W (разница 1.13×)

## Где находится нерешённый вопрос

Carry-пространство в раундах 17-64 при каскадных парах имеет
~2^27-2^29 значений. SHA-256 диффундирует это равномерно.

**Открытый вопрос:** существует ли класс сообщений W при которых
carry-паттерн в раундах 17-64 структурирован (мал)?

Это единственное направление которое не было проверено систематически.
Теоремы T_CARRY_IMAGE_SIZE показывают что такие W существуют
(при HW(t)=32), но найти цепочку раундов с одновременно экстремальным t
пока не удалось.
