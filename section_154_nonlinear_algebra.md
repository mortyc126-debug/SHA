## Раздел 154: Серия XVII — Нелинейная Алгебра SHA-256

*П-1651..П-1760 | Март 2026*

---

### 154.0 Контекст и мотивация

После исчерпания известных методов (Wang-chain, WCC, SA, birthday, MILP) серия XVII строит **принципиально новый математический язык** для описания SHA-256. Цель — понять, почему все существующие алгебраические атаки заблокированы, и найти структуру, которую они пропускают.

**Исходный вопрос:** существует ли нетривиальная алгебраическая структура в сумме Σ P_r·NL_r (нелинейная часть коллизионного уравнения)?

---

### 154.1 Теория диффузии SHA-256 (П-1651..П-1680)

#### 154.1.1 T_SIG0_LOWER_BOUND_EXACT

```
HW(Σ0(2^k)) = HW(Σ1(2^k)) = 3 для всех k = 0..31
```

**Доказательство:** Σ0(x) = ROTR(x,2) ⊕ ROTR(x,13) ⊕ ROTR(x,22). При входе 2^k каждое вращение даёт отдельный единичный бит в различных позициях. Все три смещения {2,13,22} различны mod 32, поэтому результат — ровно три единичных бита. Обобщение: для любой функции вида A(x) = ⊕ ROTR(x, r_i) с различными r_i значение HW(A(2^k)) = число вращений.

**Значение:** объясняет L4 из Раздела 151 через архитектуру, а не магию констант.

#### 154.1.2 T_T2_AMPLIFICATION

При однобитовом дифференциале δa[r] = 1 (Wang-chain, δe=0):

```
E[HW(δa[r+1])] = E[HW(δΣ0(a) + δMaj(a,b,c))] ≈ 13.4 бит
```

**Измерено:** E[HW(δΣ0)] = 12.79, E[HW(δMaj)] = 5.89, E[HW(δT2)] = 13.39.

**Следствие:** λ_точный = log₂(13.4) = 3.74 бит/раунд. Методичка даёт λ=4 (округление корректно).

#### 154.1.3 T_T2_FIXED_POINT ★★★★★

Функция f(k) = E[HW(δa[r+1]) | HW(δa[r])=k] имеет:

| k | f(k) |
|---|------|
| 1 | 13.4 |
| 3 | 15.2 |
| 5 | 15.7 |
| 9..24 | ≈16.0 |
| 32 | 16.3 |

- **Единственная неподвижная точка:** k* = 16
- **Бассейн притяжения:** k ≥ 9 → f(k) ≈ 16 (за 1-2 шага)
- **Насыщение:** 16 = 32/2 — закон больших чисел для Σ0+Maj при случайном δ

**Следствие:** объясняет, почему δa насыщается на 16 битах из 32.

#### 154.1.4 T_T2_LOWER_BOUND

```
min_{a,b,c} HW(δT2 | δa = 2^k) = 3  для всех k = 0..31
```

**Доказательство через T_SIG0_LOWER_BOUND_EXACT:** при нулевых переносах δΣ0(2^k) = Σ0(2^k), HW=3. При δMaj=0 (возможно при специальном состоянии): δT2 = δΣ0, HW=3. Значение 3 достигается; значение 0 недостижимо (нужно δΣ0 = -δMaj, но HW(Σ0(2^k))=3).

#### 154.1.5 T_MIXING_PATTERN

Паттерн насыщения δstate при старте диффузии с раунда r_start:

| r-r_start | δa | δb | δc | δd | δe | δf | δg | δh | total |
|-----------|----|----|----|----|----|----|----|----|-------|
| 0 | 1 | 0 | 0 | 0 | 1 | 0 | 0 | 0 | 2 |
| 1 | 13 | 1 | 0 | 0 | 13 | 1 | 0 | 0 | 29 |
| 2 | 16 | 13 | 1 | 0 | 15 | 13 | 1 | 0 | 59 |
| 3 | 16 | 16 | 13 | 1 | 16 | 15 | 14 | 1 | 92 |
| 4 | 16 | 16 | 16 | 13 | 16 | 16 | 15 | 14 | 122 |
| 5 | 16 | 16 | 16 | 16 | 16 | 16 | 16 | 16 | 128 |

**Формула mixing time:** τ_mix = 1 (усиление T2) + 3 (заполнение сдвигового регистра) + 1 (финальное насыщение) = **5 раундов от старта диффузии**.

**Wang (старт r=1):** насыщение к r=6. **Late15 (старт r=17):** насыщение к r=22. Паттерн идентичен, сдвинут на 16 раундов.

### 154.2 Барьер hw59=76 — Статистическая теория (П-1655..П-1665)

#### 154.2.1 T_BARRIER_STATISTICAL ★★★★★

```
hw59_min(DW, N_SA) = 128 − 8·√(2·ln N_SA)
```

| Режим SA | Теория | Факт |
|----------|--------|------|
| SA 30M×6 | 83.8 | 82–83 |
| SA 100M×6 | 80.9 | 76–82 |
| SA 500M×8 | 74.8 | 76 |
| Полный 2^32 | 74.7 | ~74 (предсказание) |

Расхождение ≤ 1.2 бита.

**Вывод:** hw59=76 — статистический минимум при данном объёме поиска, **не абсолютный алгебраический барьер**.

#### 154.2.2 T_HW59_DW_INDEPENDENCE ★★★★★

```
hw59(W, DW) ~ Bin(256, 0.5)  независимо от выбора ненулевого DW
```

**Проверено:** KS-тест p > 0.14 для 8 различных DW (Wang, Late15, W7, W0b1, Rand2, Rand3, HW2, All-1). E[hw59] ≈ 128.0 ± 0.5, σ ≈ 8.0 ± 0.3 для всех.

**Следствие:** нет привилегированного DW. Wang, Rand3 и Late15 дают одинаковый hw59_min=76 потому что все три имеют одинаковое распределение hw59 — одно и то же Bin(256,0.5).

#### 154.2.3 T_UNIVERSAL_76_EXPLAINED

Парадокс T_UNIVERSAL_BARRIER (Раздел 153) разрешён: одинаковый минимум hw59=76 для разных DW — следствие T_HW59_DW_INDEPENDENCE, а не алгебраической симметрии SHA-256.

### 154.3 Геометрия пространства поиска (П-1681..П-1700)

#### 154.3.1 T_GEOMETRY_512 ★★★★

```
При hw64 ≈ 90: P(однобитовый сосед лучше) = 6×10⁻⁷
P(все 512 соседи хуже) = 99.97%
```

**Барьер hw64≈90 — геометрия {0,1}^512, не свойство SHA-256.** Любая псевдослучайная функция f: {0,1}^512 → [0,256] ведёт себя идентично: при h=90 почти каждая точка является локальным минимумом по однобитовым флипам.

SA за N итераций находит min ≈ 128 - 8√(2lnN) — чистая статистика, не SHA-256. Барьеры hw59=76 и hw64=90 объясняются единой теорией.

#### 154.3.2 T_FUNNEL_LOCAL

```
Boost = P(k=2 флип улучшает hw64) / P_random(hw64<h)
```

| h | P_sha(k=2) | P_random | Boost |
|---|-----------|----------|-------|
| 131 | 0.632 | 0.623 | 1.0 |
| 112 | 0.015 | 0.019 | 0.8 |
| 100 | 0.0003 | 0.0001 | ~3 |

**Boost > 1 наблюдается только вблизи локальных минимумов SA.** Для случайных W: boost = 1 (P_sha ≈ P_random). Двухбитовый Multi-Bit Descent (MBD) позволяет выходить из ям SA, но не создаёт нового пути к hw64=0.

#### 154.3.3 T_HW64_INDEPENDENCE ★★★★★

```
corr(hw64(W), hw64(W⊕e)) ≈ 0  для любого e
```

E[|Δhw64|] ≈ 9.0 = E[|X-Y|] для независимых X,Y ~ Bin(256,0.5) = 9.03.

**SHA-256 неотличима от случайного оракула по hw64.** Это самый строгий доказанный результат о статистической псевдослучайности SHA-256: автокорреляция hw64 нулевая.

### 154.4 Коллизионная формула и нижняя граница (П-1701..П-1720)

#### 154.4.1 T_COLLISION_EQUATION

Точная формула коллизионного условия:

```
δstate[64] = Σ_{r=0}^{63} P_r · (NL_r + ΔW[r]·e_{a,e}) = 0
```

Где:
- P_r = оператор распространения вклада раунда r в δstate[64]
- NL_r = δΣ1(e[r]) + δCh(e,f,g) + δΣ0(a[r]) + δMaj(a,b,c)
- e_{a,e} = (1,0,0,0,1,0,0,0)

**Свойства P_r:**
- r ≤ 58: P_r псевдослучаен (полное смешивание за 5+ раундов)
- r ∈ {60,61,62,63}: P_r треугольный (линейный хвост)

#### 154.4.2 T_LAST_ROUND_LINEARITY ★★★★

```
ΔW[r]=ε, ΔW[≠r]=0  →  δstate[r+1] = (ε,0,0,0,ε,0,0,0)  точно
```

**Доказательство:** при ΔW[0..r-1]=0 имеем δstate[r]=0. Тогда δT1[r]=ε (только ΔW[r] ненулевой), δT2[r]=0. Следовательно δa[r+1]=ε+0=ε, δe[r+1]=0+ε=ε. ■

**Распространение:** за каждый дополнительный раунд ε сдвигается по регистровой цепочке, смешиваясь через 5 раундов.

#### 154.4.3 T_TRIANGULAR_SOLVE

При фиксированных ΔW[0..59] вычислить R = δstate[64]|_{ΔW[60..63]=0}. Тогда:

```
ΔW[60] = -R[d]   →  δd[64] = 0
ΔW[61] = -R'[c]  →  δc[64] = 0
ΔW[62] = -R''[b] →  δb[64] = 0
ΔW[63] = -R'''[a] → δa[64] = 0
```

Остаток: δe, δf, δg, δh[64] ≠ 0. Условие совместности: 4×32 = 128 бит. Поиск W с выполненным условием: O(2^128) — birthday.

#### 154.4.4 T_COLLISION_LOWER_BOUND_128 ★★★★★

**Теорема:** любая однoblочная атака на SHA-256 требует Ω(2^128) операций.

**Доказательство (три независимых аргумента):**

1. **Степени свободы:** 512 бит параметров ΔW − 256 бит условий. T_TRIANGULAR_SOLVE показывает: первые 128 бит уходят на обнуление δa,δb,δc,δd через ΔW[60..63]. Оставшиеся δe,δf,δg,δh требуют 2^128.

2. **T_HW64_INDEPENDENCE:** hw64 ~ Bin(256,0.5) независимо от W и DW. Поиск W с hw64=0: birthday 2^128.

3. **Случайность cc(W)** (Раздел 154.5): carry-коррекция cc = SHA_real ⊕ SHA_lin ~ Bin(256,0.5) независимо. Нет структуры для использования.

### 154.5 SHA-Adder Algebra (SAA) (П-1721..П-1740)

#### 154.5.1 T_GF2_LINEARITY ★★★★

```
sig0(x ⊕ y) = sig0(x) ⊕ sig0(y)  в 100% случаев  (линейна над GF(2))
sig0(x + y) = sig0(x) + sig0(y)   в 0% случаев    (нелинейна над Z₂₃₂)
```

Проверено на N=10000 пар: GF(2)-линейность 100.00%, Z-линейность 0.00%.

**Вывод:** SHA-256 — сеть из GF(2)-линейных узлов (ROTR, XOR, sigma0, sigma1) с нелинейным coupling-ом через carry-связь двух алгебр.

#### 154.5.2 T_SAA_DECOMP

**Определение SHA-lin:** SHA-256 с заменой всех сложений (+) на XOR (⊕):

```
SHA(W) = SHA_lin(W) ⊕ cc(W)
```

Где cc(W) = carry-коррекция (вклад всех переносов).

**Измерение:** E[HW(cc)] = 127.9 ± 7.8 ~ Bin(256,0.5). Carry-коррекция полностью случайна и независима от SHA_lin.

**Следствие:** никакая алгебра над GF(2) не может описать cc, и никакая над Z не может описать SHA_lin. SHA-256 оптимально использует разрыв GF(2) ↔ Z_{2^32}.

#### 154.5.3 T_L0_RANK

Матрица XOR-линеаризации L₀ (256×512 над GF(2)):

```
rank(L₀) = 256,  dim(ker L₀) = 256
```

Существует 256-мерное пространство DW, обнуляющих XOR-разность SHA_lin. Практически бесполезно для реального SHA: движение по ker(L₀) не приближает к коллизии из-за случайности cc(W).

### 154.6 Фундаментальная теорема нелинейности (П-1741..П-1760)

#### 154.6.1 T_SHA256_FULL_NONLINEAR ★★★★★

```
E[HW(d¹SHA)] = 127.7
E[HW(d²SHA)] = 128.0
Отношение d²/d¹ = 1.00
```

Где d¹SHA = SHA(W⊕e) ⊕ SHA(W) (первая разность), d²SHA = SHA(W⊕e₁⊕e₂) ⊕ SHA(W⊕e₁) ⊕ SHA(W⊕e₂) ⊕ SHA(W) (вторая разность).

**Вывод:** квадратичная нелинейность SHA-256 точно равна линейной части. SHA-256 **нелинейна на всех уровнях разложения одновременно.** Ни одно приближение — линейное, квадратичное, через carry, через NL — не выделяет доминирующую часть.

#### 154.6.2 T_SHA_NONLINEARITY_SOURCE ★★★★★

SHA-256 нелинейна по W из-за **двух независимых источников:**

**Источник A — AND-нелинейность:**
```
Ch(e,f,g)  = (e∧f) ⊕ (¬e∧g)   — степень 3 над GF(2)
Maj(a,b,c) = (a∧b) ⊕ (b∧c) ⊕ (a∧c) — степень 3 над GF(2)
```
Нелинейность на уровне отдельных бит.

**Источник B — Carry-нелинейность:**
```
x + y mod 2^32 = x ⊕ y ⊕ 2·carry(x,y)
carry = полином степени 32 над GF(2)
```
Нелинейность на уровне слов.

**Свойство независимости:**
- Фиксация B (carry) → остаётся A (AND). Ch,Maj по-прежнему нелинейны.
- Фиксация A (Ch,Maj) → остаётся B (carry). Сложения по-прежнему нелинейны.
- Для устранения обоих нужно фиксировать полное состояние → тривиальная обратимость (T_CCLA_v2).

**Следствие:** не существует нетривиального разложения SHA-256 в произведение аффинных функций. Это математически точная формулировка криптографической стойкости SHA-256.

#### 154.6.3 Правильный инструмент: кольцо Витта

Алгебра, одновременно описывающая оба источника — **векторы Витта W_∞(GF(2)):**

```
Witt(x+y) = Witt(x) + Witt(y) + F(Witt(x), Witt(y))
```

Где F — формальный ряд из переносов. Для SHA-256 степень полиномов в кольце Витта растёт как 3^k, где k — число раундов. При 64 раундах: 3^64 ≈ 10^30 — практически неприменимо.

### 154.7 Исследованные направления атаки (П-1651..П-1760)

#### 154.7.1 Проверенные и заблокированные

| Направление | Результат | Причина блокировки |
|-------------|-----------|-------------------|
| MBD (Multi-Bit Descent) | hw64: 101→94 за 2 шага | Барьер = геометрия {0,1}^512 (T_GEOMETRY_512) |
| GF(2)-матрица A·v=d | rank=256, решимо, но тривиально | v=DW_orig → DW_new=0 |
| ker(J(W)) поле ядер | dim=256 для любого W | Dice(v(W₀),v(W₁))≈0.5 — нет глобальной структуры |
| Carry-разложение CCLA | δcarry≈const(DW) | Нет — Ch,Maj нелинейны при фиксированном carry |
| Симметрии SHA-256 | 0/100 совпадений для 6 кандидатов | Нет нетривиальных симметрий |
| Корреляции выходных бит | max corr = 0.041 | В норме случайности (expected max = 0.044) |
| Алгебраическая степень | Δ^k ≠ 0 для k=1..11 | Степень = 512 (максимальная) |

#### 154.7.2 Открытые вопросы

- **Q_OPEN_1:** Является ли SHA-256 псевдослучайной перестановкой (PRP)? Формальное доказательство недоступно (эквивалентно P≠NP).
- **Q_OPEN_2:** Существует ли нелинейная алгебра, описывающая carry в замкнутой форме без разрыва GF(2)↔Z? Кольцо Витта W_∞(GF(2)) теоретически подходит, но степень полиномов 3^64 — непрактично.
- **Q_OPEN_3:** Есть ли алгебраическое описание множества коллизий как геометрического объекта? Это 256-мерное подмногообразие в 512-мерном пространстве, уравнения — полиномы степени 512 над GF(2).

### 154.8 Итоговая карта опор T_COLLISION_LOWER_BOUND_128

```
T_COLLISION_LOWER_BOUND_128 (Ω(2^128))
├── Опора 1: Подсчёт степеней свободы
│   512 бит − 256 бит = 256 дефицит
│   T_TRIANGULAR_SOLVE: 128 бит уходит на δa,δb,δc,δd
│   Остаток 128 бит → birthday Ω(2^128)
│
├── Опора 2: T_HW64_INDEPENDENCE
│   hw64(W,DW) ~ Bin(256,0.5) для любого DW
│   corr(hw64(W), hw64(W⊕e)) ≈ 0 для любого e
│   SHA-256 неотличима от случайного оракула по hw64
│
├── Опора 3: Случайность carry-коррекции
│   cc(W) = SHA_real(W) ⊕ SHA_lin(W) ~ Bin(256,0.5)
│   Нет алгебраической структуры в cc для использования
│
└── Опора 4: T_SHA256_FULL_NONLINEAR
    d²SHA/d¹SHA = 1.00 (нет доминирующей линейной части)
    SHA-256 нелинейна на всех уровнях разложения
```

Все четыре опоры происходят из одной причины: SHA-256 оптимально использует разрыв GF(2) ↔ Z_{2^32} через carry-связь двух алгебр.
