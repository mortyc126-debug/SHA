# П-226: Итеративный multi-block

## Схема

```
IV₀ = SHA IV
IV₁ = best_M₁ argmin eval(compress(IV₀, M₁))
IV₂ = best_M₂ argmin eval(compress(IV₁, M₂))
...
IVₙ → аттрактор?
```

## Результаты

### Эксперимент 1: 20 шагов × 500 кандидатов × 1K проб

```
Шаг  0 (SHA IV): min_HW=48
Шаг  1: min_HW=37
Шаг  6: min_HW=35  ← локальный минимум
Шаг 16: min_HW=40  ← откат вверх
...
mean=37.9, std=2.7
```

### Эксперимент 2: 10 шагов × 2K кандидатов × 10K проб

```
min=39, mean=43.0
Нет монотонного убывания
```

## Вывод: T_NO_ATTRACTOR

**SHA-256 compress не имеет слабых аттракторов.**

Итерация не сходится — каждый шаг это независимая случайная выборка из одного и того же распределения. `compress(IV, M)` при случайном M ведёт себя как псевдослучайная перестановка: IV' равномерно распределена на `{0,1}^256` вне зависимости от IV.

Математически: `P(eval(compress(IV', M)) < eval(IV'))` ≈ const независимо от IV'.

### Верифицированные теоремы

| # | Теорема | Содержание |
|---|---|---|
| 1 | T_ADAPTIVE_P1 | Адаптивный трейл P=1, 64 раунда |

| 4 | T_NDR_CH | d0 & (f⊕g)=0 → δCh=0, +15% |
| 5 | T_GF2_SCHED | Расписание линейно над GF(2), E[HW]=128 |

| 7 | T_NO_ATTRACTOR | SHA-256 compress без слабых аттракторов |

### Числовые рекорды

```
min HW(δstate[64]) = 30   (адаптивный, d0=1, 200M пар)
min HW(ΔW[17])     = 1    (адаптивный, d0=8, 45M пар)
min HW (real SHA)  = 86   (однобитное M vs M', 10M пар)
min HW (weak IV)   = 40   (два блока, 10K пар)
```

### Фундаментальный барьер

```
Адаптивный (W свободны):     min=30  ← наш потолок
Multi-block слабый IV:        min=40
Реальный SHA-256 хеш:         min=86  ← практический предел
```

Разрыв **30 vs 86** — это "цена расписания". Устранить его без принципиально нового метода (MILP, SAT, структурный алгебраический) невозможно.
