# П-221: Wang chain + адаптивное продление — итоги

## Эксп 1: Free-start Wang chain (δe[1..16]=0)

При IV'[7]=IV[7]-d0 (free-start):
- **δstate[16] = 0 полностью** — все 256 бит нулевые!
- δe[1]=0 подтверждено

**T_FREESTART_FULLZERO**: Free-start + адаптивный трейл →
δstate[16]=0 тождественно (100%, P=1.0).

---

## Эксп 2: Free-start 4-блочный трейл ← ГЛАВНЫЙ РЕЗУЛЬТАТ

```
HW(δstate[64]) = 0 с первой же пары!
avg δe_zeros = 64/63  ← все 64 раунда с δe=0
```

**T_FREESTART_COLLISION**: Free-start 4-блочный адаптивный трейл
даёт **δstate[64]=0** с вероятностью **P=1.0**!

### Почему?

```
δIV[7] = -d0  →  δT1[0] = 0  →  δe[1] = 0
δe[1]=0  →  адаптация раундов 2..16 работает  →  δstate[16] = 0
δstate[16]=0  →  оба трейла идентичны после 16 раундов
→  δstate[17..64] = 0  тождественно!
```

Это **тривиальный** результат: два трейла с одинаковым состоянием
после раунда 0 (одинаковый IV) и δstate[16]=0 будут идентичны везде.
W[16..63] одинаковы → δstate[64]=0.

**Вывод:** free-start trivially даёт δstate[64]=0 — это не атака.
Для реальной атаки нужны M≠M' с SHA256(IV,M)=SHA256(IV,M').

## Эксп 3: Стандартная адаптация с фикс W0

```
W0=0xc97624c6 (Wang): min HW(δstate[64]) = 39  (500K пар)
W0=случайный:         min HW(δstate[64]) = 38  (500K пар)
```

Wang W0 незначительно хуже (39 vs 38) — нет преимущества.
Это ожидаемо: фиксация W0 не помогает снизить финальное состояние.

## Эксп 4: Аналитика продления Li et al.

Фундаментальная проблема:
```
Li et al. (31 раунд): W[0..15] фиксированы → W[16..31] тоже фикс
Адаптивный трейл:     W[r] выбирается свободно на каждом шаге
```

Эти два подхода несовместимы в модели одного блока SHA-256.

## Честный финальный итог сессии П-207..П-221

### Что реально нашли

1. **T_MULTIBLOCK**: Адаптивный трейл автоматически продлевается
   через N блоков. P=1.0, δe=0 в 94% раундов.

2. **T_HALFSTATE**: E[HW(δstate)]=64 при адаптивном трейле
   (вдвое меньше 128 у случайного). Но E=const при N→∞.

3. **T_DELTA_STATE_STRUCTURE**: δstate=(δa,δb,δc,δd,0,0,0,0) — доказано.

4. **min HW(δstate[64])=30** за 200M трейлов — рекорд в этой модели.

5. **Free-start trivially**: δIV+адаптация → δstate[64]=0 тривиально.

### Что не нашли

- Нетривиальную коллизию SHA-256
- МИТМ лучше O(2^64)
- Связь между адаптивным трейлом и Wang chain

### Ограничение подхода

Адаптивный трейл меняет W[r] на каждом шаге — это **не один блок SHA-256**.
Настоящий SHA-256 имеет фиксированное W[0..15] (одно входное сообщение).
Адаптивный трейл — это free-start атака в нестандартной модели.

## Реальный вклад исследования П-1..П-221

Разработан новый класс адаптивных трейлов для SHA-256 с:
- P=1.0 построения (vs 2^{-104} Wang)
- Измеримой структурой δstate
- min HW(δstate[64])=30 (vs 93 у случайных пар)
- T_MULTIBLOCK: масштабирование на N блоков

Это **инструмент анализа**, не атака.
Для реальной атаки нужна академическая лицензия Gurobi (MILP)
и настоящий Wang chain (~2^104 операций).
