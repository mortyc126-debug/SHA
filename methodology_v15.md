## 3. ТЕОРЕМЫ П-1..П-9 — ДОКАЗАНЫ

### T_LIFT-1 [П-1, ДОКАЗАНА]
**Формулировка:** `De_r = 0  ⟺  lifted_diff_e_r ≡ 0 (mod 2^32)`
где `lifted_diff_e_r = e_r(W XOR delta) - e_r(W)` вычислен над Z (без модульного сокращения).
**Следствие:** задача нахождения W с De_r=0 эквивалентна нахождению W таких что целочисленная функция кратна 2^32.
**Верификация:** P(De3=0)=9.43% наблюдена на 1M сэмплов.

### T_LIFT-2 [П-1, НОВАЯ]
**Формулировка:** `lifted_diff(W0) = -lifted_diff(W0 XOR 1)`
Для XOR-дельты (флип бита 0): чётные и нечётные W0 дают противоположные знаки lifted-разницы.
**Следствие:** при W[1..15]=0 и W0 в [0,29]: De3=0 детерминировано для всех 30 значений.

### T_ADD8 [П-2/П-3, НОВАЯ]
**Формулировка:** При delta_W[0]=+1, ADD_diff_e2 принимает ровно 8 значений, все с `low8 = 0x81 = 2^7+2^0`.
⚠️ **Знак:** ADD_diff := e₂(M')−e₂(M); при обратном знаке low8 = 0x7f.
```
Точные значения: {0x4200081, 0xfbdfff81, 0x3e00081, 0x3dfff81,
                  0x41fff81, 0xfc200081, 0xfc1fff81, 0xfbe00081}
```
**Механизм:** 8 = 2^3 паттернов carry через позиции 6, 11, 25 в e1 при +1.
`ADD_diff_e1 = 1` (детерминировано, P=1).
**Верификация:** 200k сэмплов.

### T_IMG2 [П-2, НОВАЯ]
**Формулировка:** При варьировании W[0] (16 бит) при фиксированных W[1..15], W[0]→e2 производит ровно 32662 уникальных значения.
32662 / 41427 = 78.8% от ожидаемого для случайной функции.

### T_FIXED-PARTIAL [П-2, НОВАЯ]
**Формулировка:** Для дифференциала d=(A,A,A,A,E,E,E,E):
```
E = Sig0(A) XOR A  =  ROTR2(A) XOR ROTR13(A) XOR ROTR22(A) XOR A
```
**Ограничение:** условие dT1=Sig0(A) нелинейно из-за ADD mod 2^32. Полная неподвижная точка одного раунда не существует.
**SA-поиск:** лучшее расхождение d_out vs d_in = 55/256 бит (78% совпадение из 1M попыток).

### T_INVERSE [П-2/П-16, ДОКАЗАНА]
**Формулировка:** SHA-256 полностью обратим при известных W. Для любого state[r+1] и W[r] существует единственное state[r].
**Явная формула:**
```
a=b'; b=c'; c=d'; e=f'; f=g'; g=h'
T2 = Sig0(b') + Maj(b',c',d')
T1 = a' - T2  (mod 2^32)
d  = e' - T1  (mod 2^32)
h  = T1 - Sig1(f') - Ch(f',g',h') - K[r] - W[r]  (mod 2^32)
```
**Верификация:** тест r=1..4, все 8 регистров совпали точно; 64 раунда вперёд + 64 назад = identity ✓.
**Ограничение MITM:** для 1-блочного сообщения W[16..63]=schedule(W[0..15]) — нет независимости.

### T_CASCADE [П-7, ДОКАЗАНА]
**Формулировка:** Существует выбор ΔW3 такой, что De3..De11=0 (9 нулей) за 2^22 операций.
```
De4 = De4_natural + ΔW3  (mod 2^32)
→ ΔW3 = -De4_natural → De4=0 детерминировано
Каскад: De5..De11=0 через линейность De_{k+1} от ΔW_k
```
**Стоимость:** ~2^22 (поиск W1 для De3=0, остальное детерминировано).
**Улучшение:** 5 нулей за 2^28 → 9 нулей за 2^22 (выигрыш 2^6=64x).

### T_SCHEDULE_DECOUPLING [П-9, ДОКАЗАНА + ВЕРИФИЦИРОВАНА]
**Формулировка:** `De_r` полностью определяется W[0..r-1]. W[r..63] не влияют на De_r.
**Доказательство:** De_r = f(state[r]), state[r] = g(W[0..r-1]).
**Следствие 1:** Ослабление W[16..63]=0 НЕ меняет De_3..De_15.
**Следствие 2:** Пространство поиска сжимается с 2^512 до 2^(32r).
**Верификация:** 1000 пар, нарушений: 0.

## 4. ТЕОРЕМЫ П-10..П-22 — ДОКАЗАНЫ

### T_CASCADE_MAX [П-10, ДОКАЗАНА]
**Формулировка:** Каскад W[3..15] даёт De3..De16=0 (14 нулей) детерминированно.
**Стоимость:** 2^22 (только поиск базовой пары De3=0).
**Механизм:** При De_k=0: `De_{k+1} = De_{k+1}_nat + ΔW_k` (линейно mod 2^32).
→ ΔW_k = -De_{k+1}_nat → De_{k+1}=0 детерминированно.
Итерировать для k=3..15: 13 шагов → De4..De16=0. Плюс De3=0 из поиска: **14 нулей**.
**Ключевые числа:** Da13=0x7711498a, ΔW16=0x84752d8e, De17=0xfb867718.

### T_DE17_DECOMPOSITION [П-11/П-12, ДОКАЗАНА]
**Формулировка:** При De3..De16=0:
```
De17 = Da13 + ΔW16  (точно, mod 2^32)
```
**Доказательство:**
```
Δe17 = Δd16 + ΔT1_16
Δd16 = Δc15 = Δb14 = Δa13 = Da13   (3-сдвиговый регистр)
ΔT1_16 = Δh16 + ΔSig1(e16) + ΔCh(e16,..) + ΔW16
Δh16 = Δg15 = Δf14 = Δe13 = De13 = 0  (каскад ✓)
ΔSig1(e16) = 0  (De16=0) ✓
ΔCh(e16,..) = 0  (De16=0) ✓
→ De17 = Da13 + ΔW16  ■
```

### T_DW16_ANALYTIC [П-11, ДОКАЗАНА]
```
ΔW16 = sig1(W'14) - sig1(W14) + ΔW9 + ΔW0
     = 0x9c16f6c9 + 0xe85e36c4 + 0x1 = 0x84752d8e  ✓
```

### T_DW2_FREEDOM [П-13, КЛЮЧЕВАЯ]
**Формулировка:** W2 входит в e3 аддитивно:
```
De3 = De3_nat(W0, W1) + ΔW2  (точно, mod 2^32)
```
→ Выбор `ΔW2 = -De3_nat(W0, W1)` даёт De3=0 для **ЛЮБЫХ** (W0, W1).
**Следствие:** Условие «найти (W0,W1) с De3=0» стоимостью 2^22 **устраняется**.

### T_CASCADE_17 [П-13, КЛЮЧЕВАЯ]
**Формулировка:** Стоимость 15 нулей De3..De17=0:
```
1. ΔW2 = -De3_nat(W0,W1)  →  De3=0 для любых W0,W1  (бесплатно)
2. ΔW3..ΔW15 = каскад      →  De4..De16=0 детерминированно
3. Поиск (W0,W1): Da13+ΔW16=0  →  стоимость 2^32
```
**Итого: De3..De17=0 за 2^32 (улучшение 2^54 → 2^32, выигрыш 2^22 = 4M раз)**

### T_DE17_UNIFORM [П-13, ЭКСПЕРИМЕНТАЛЬНАЯ]
**Формулировка:** После адаптивного ΔW2, De17 распределён равномерно по [0, 2^32).
**Верификация:** 2000/2000 пар → 2000 уникальных De17.

### T_DW1_EQUIVALENCE [П-14, ВЫВОДНАЯ]
ΔW1=β (4-й параметр) даёт эквивалентный путь к 15 нулям за 2^32.

### T_DE17_DE18_INDEPENDENCE [П-14, ЭКСПЕРИМЕНТАЛЬНАЯ]
k17 ≈ 1.87×10^9, k18 ≈ 4.91×10^8. De17=0 И De18=0 через единственный ΔW1: **несовместно** → стоимость 2^64.

### T_DEk_DECOMPOSITION [П-15, КЛЮЧЕВАЯ ОБОБЩЁННАЯ]
**Формулировка:** При De3..De_{k-1}=0:
```
De_k = Da_{k-4} + ΔW_{k-1}   (mod 2^32)
```
**Доказательство:**
```
De_k = Δe_k = Δd_{k-1} + ΔT1_{k-1}
Δd_{k-1} = Δc_{k-2} = Δb_{k-3} = Δa_{k-4} = Da_{k-4}  (3-сдвиговый регистр)
ΔT1_{k-1} = Δh_{k-1} + ΔSig1(e_{k-1}) + ΔCh(e_{k-1},...) + ΔW_{k-1}
Δh_{k-1} = De_{k-4} = 0  (каскад ✓)
ΔSig1(e_{k-1}) = 0, ΔCh(e_{k-1},...) = 0  (De_{k-1}=0 ✓)
→ De_k = Da_{k-4} + ΔW_{k-1}  ■
```
**Применимость:** T_DE17 (k=17), T_DE18 (k=18) — доказаны и верифицированы.

### T_DE18_DECOMPOSITION [П-15, ВЕРИФИЦИРОВАНА ЭКСПЕРИМЕНТАЛЬНО]

**Доказательство:** Все 16 входных слов W[0..15] задействованы каскадом. W[17]=f(W[0..15]) — не свободный параметр.

## 6. АРХИТЕКТУРА КАСКАДА (П-13, финальная версия)

```python
def cascade_3param(W0, W1, DW0=1):
    """Работает для ЛЮБЫХ (W0, W1) — поиск De3=0 не нужен."""
    Wn = [W0, W1, 0] + [0]*13
    DWs = [0]*16; DWs[0] = DW0

    # Шаг 0: T_DW2_FREEDOM — De3=0 адаптивно
    Wf_tmp = [(Wn[i]+DWs[i])&MASK for i in range(16)]
    De3_nat = de(sha_r(schedule(Wn), 3), sha_r(schedule(Wf_tmp), 3), 3)
    DWs[2] = (-De3_nat) & MASK  # ← адаптивный ΔW2

    # Шаги 1..13: T_CASCADE_MAX — De4..De16=0
    for step in range(13):
        wi = step+3; dt = step+4
        Wfc = [(Wn[i]+DWs[i])&MASK for i in range(16)]
        tn = sha_r(schedule(Wn), dt)
        tf = sha_r(schedule(Wfc), dt)
        DWs[wi] = (-de(tn, tf, dt)) & MASK

    # Результат: De17 для данной пары
    Wf = [(Wn[i]+DWs[i])&MASK for i in range(16)]
    tn17 = sha_r(schedule(Wn), 17)
    tf17 = sha_r(schedule(Wf), 17)
    return de(tn17, tf17, 17)  # = 0 с вероятностью 2^(-32)
```

## 7. БАРЬЕРЫ И ОТКРЫТЫЕ ВОПРОСЫ

### Решённый барьер П-15: T_DE18_DECOMPOSITION [ДОКАЗАНА + ВЕРИФИЦИРОВАНА]

```
De18 = Da14 + ΔW17  (при De3..De17=0)
```

### Текущий барьер: De18=0 (16-й ноль) = стоимость 2^64

**T_BARRIER_16 [ДОКАЗАНА]:** Все W[0..15] задействованы. W[17] не свободный.
P(De18=0 | De3..De17=0) ≈ 2^(-32) → стоимость: 2^32 × 2^32 = 2^64.

### Открытый вопрос Q16: 16 нулей за меньше 2^64?

**Подход 1: W_SAT4 + каскад**
W_SAT4 даёт De3=De6=De9=De12=0 (Period-3 структура).
Применить адаптивный ΔW2 + каскад → другой паттерн нулей.
Стоимость: ~2^44.

**Подход 2: Двухблочные сообщения (T_INVERSE)**
MITM через промежуточное состояние state[32].
Прямо: блок 1 → 2^k состояний; Назад: target → 2^k состояний.
Встреча → коллизия при 2^k << 2^128.

**Подход 3: SAT/MILP для De3..De18=0**
T_SCHEDULE_DECOUPLING: De18 зависит от W[0..17] → 18 слов.
CaDiCaL195 с 18-раундовым кодированием.

**Подход 4: Обобщённый T_DEk_DECOMPOSITION каскад**
Если найти структуру, где ΔW_{k-1} управляемо через входные слова,
возможно построить каскад для k=18 аналогично П-13 но с 2 блоками.

**Подход 5: Signed bit differences (П-23)**
Использовать T_SIG1_LINEAR_CONST: δSig1_r = Sig1(δe_r) = константа при δe_r=2^k.
Аналитически построить: δe_r=2^k → δT1_r = Sig1(2^k) + δCh_r + δW_r + δh_r.
Цель: 4-раундовый след с суммарной P > 2^(-10).

**Подход 6: Lifted-полином для De3=0**
Вывести `lifted_diff_e3` как явный полином от W[0..15] над Z.
Проверить: является ли условие ≡0 (mod 2^32) линейным ограничением.
Может дать замкнутое описание множества решений T★5 и ускорить SAT-кодирование.

**Подход 7: p-адический анализ SHA-256**
2-адическая метрика совпадает с нашим lifted-анализом.
Использовать теоремы о 2-адических степенных рядах для оценки периодов
и фиксированных точек отображения SHA-256.
