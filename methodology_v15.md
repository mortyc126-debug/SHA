## 2. АДДИТИВНЫЕ ТЕОРЕМЫ (П-1..П-19) — ДОКАЗАНЫ

### T_ADD8 [П-2, ДОКАЗАНА]
8 структур `lifted_diff_e2` — полное описание De3=0 через аналитические пути.

### T_CASCADE_MAX [П-10, ДОКАЗАНА]
**Формулировка:** Каскад W[3..15] даёт De3..De16=0 (14 нулей) детерминированно.
**Стоимость:** 2^22 (поиск базовой пары De3=0).
**Механизм:** При De_k=0: `De_{k+1} = De_{k+1}_nat + ΔW_k` (линейно mod 2^32).
→ `ΔW_k = -De_{k+1}_nat` → De_{k+1}=0 детерминированно.

### T_DE17_DECOMPOSITION [П-11/П-12, ДОКАЗАНА]
При De3..De16=0:
```
De17 = Da13 + ΔW16  (точно, mod 2^32)
Da13 = 0x7711498a, ΔW16 = 0x84752d8e, De17 = 0xfb867718  ✓
```

### **T_DW2_FREEDOM [П-13, КЛЮЧЕВАЯ ТЕОРЕМА]**
```
De3 = De3_nat(W0, W1) + ΔW2  (точно, mod 2^32)
→ ΔW2 = -De3_nat → De3=0 для ЛЮБЫХ (W0, W1)
```
Устраняет поиск 2^22 базовой пары.

### **T_CASCADE_17 [П-13, КЛЮЧЕВАЯ ТЕОРЕМА]**
```
1. ΔW2 = -De3_nat(W0,W1)  →  De3=0 (бесплатно)
2. ΔW3..ΔW15 = каскад     →  De4..De16=0 (детерминировано)
3. Поиск (W0,W1): Da13+ΔW16=0  →  стоимость 2^32
Итого: De3..De17=0 за 2^32
```

### T_DE17_UNIFORM [П-13, ЭКСПЕРИМЕНТАЛЬНАЯ]
После адаптивного ΔW2, De17 распределён равномерно по [0, 2^32). Верификация: 2000 уникальных De17 из 2000 пар.

### **T_DEk_DECOMPOSITION [П-15, КЛЮЧЕВАЯ ОБОБЩЁННАЯ ТЕОРЕМА]**
При De3..De_{k-1}=0:
```
De_k = Da_{k-4} + ΔW_{k-1}   (mod 2^32)
```
**Доказательство:**
```
De_k = Δd_{k-1} + ΔT1_{k-1}
Δd_{k-1} = Da_{k-4}  (3-сдвиговый регистр)
ΔT1_{k-1} = Δh_{k-1} + ΔSig1(e_{k-1}) + ΔCh(...) + ΔW_{k-1}
Δh_{k-1} = De_{k-4} = 0  (каскад ✓)
ΔSig1(e_{k-1}) = 0, ΔCh = 0  (De_{k-1}=0 ✓)
→ De_k = Da_{k-4} + ΔW_{k-1}  ■
```

### **T_DE18_DECOMPOSITION [П-15, ВЕРИФИЦИРОВАНА]**
При De3..De17=0: `De18 = Da14 + ΔW17`
```
Найденная пара: W0=0xe82222c7, W1=0x516cfb41
Da14=0x6ac9d38b, ΔW17=0x6c286d53, De18=0xd6f240de ✓
```

### **T_BARRIER_16 [П-15, ДОКАЗАНА]**
**Стоимость De3..De18=0 = 2^64.**
Все W[0..15] задействованы каскадом; W[17]=f(W[0..15]) не свободен.
P(De18=0 | De3..De17=0) ≈ 2^(-32) → стоимость 2^32 × 2^32 = **2^64**.

### T_INVERSE [П-16, ДОКАЗАНА]
Обратный SHA-256 раунд существует, O(1), биективен. 64 раунда вперёд + 64 назад = identity ✓.

### T_DE18_INDEPENDENCE [П-16, ДОКАЗАНА]
P(De18=0 | De17=0) ≈ 2^(-32) аналитически (независимые события). MITM 2-блочный не снижает барьер ниже 2^64.

### T_SAT_CASCADE [П-17, ВЕРИФИЦИРОВАНА]
SAT (Z3 BitVec): k≤16 за <1с; k=17 timeout. Подтверждает T_BARRIER_16.

### T_GENERALIZED_CASCADE [П-19, ДОКАЗАНА]
Каскад De3..De16=0 работает для **любого ΔW0 ≠ 0** (не только ΔW0=1). T_DW2_FREEDOM и T_DEk_DECOMPOSITION применимы при любом ΔW0.

### T_3D_BARRIER [П-19, СТАТИСТИЧЕСКАЯ]
2^64 подтверждён статистически в 3D-пространстве (W0, W1, ΔW0): N=50000, корреляция ≈ 0.

## 3. XOR-ДИФФЕРЕНЦИАЛЬНЫЕ ТЕОРЕМЫ (П-20..П-22) — ДОКАЗАНЫ

### **T_MIXED_DIFF [П-20, КЛЮЧЕВАЯ]**
sig0 и sig1 **линейны над XOR**:
```python
sig0(W ^ δ) = sig0(W) ^ sig0(δ)   # P=1.0, всегда
sig1(W ^ δ) = sig1(W) ^ sig1(δ)   # P=1.0, всегда
```
→ Расписание (schedule) линейно над XOR, P=1.

### T_SCHEDULE_XOR [П-20, ЭКСПЕРИМЕНТАЛЬНАЯ]
δW0=0x80000000 → 12/32 ненулевых δW_i в расписании. Нет нулевого дифференциала.

### **T_IV_BIT0 [П-21, ДОКАЗАНА АНАЛИТИЧЕСКИ]**
C = d_iv + S = 0x98c7e2a2, **bit0(C) = 0** → δe1=1 для ЛЮБОГО W0 при δW0=1.
```
Доказательство:
  W0[bit0]=0: e1_n[bit0]=0 → (e1_n+1) XOR e1_n = 1 ✓
  W0[bit0]=1: e1_n[bit0]=1 → (e1_n-1) XOR e1_n = 1 ✓
```

### **T_DOM_DIFF [П-21, КЛЮЧЕВАЯ]**
**δW0=0x8000 → δe1=0x8000 с P=0.773 (доминирующий XOR-дифференциал).**
Это лучший δW0 по критерию минимального веса δW[16..63] (sum_wt=466).

### T_CARRY_ANALYTIC [П-22, ДОКАЗАНА]
P(δe1=2^k) при δW0=2^k:
```
Если C[bit k]=1: P = C[0..k-1] / 2^k
Если C[bit k]=0: P = (2^k - C[0..k-1]) / 2^k
где C = d_iv + T1_0_base = 0x98c7e2a2
```
Верифицирована для всех чётных k=0..30 ✓.

### **T_SUFFICIENT_R1 [П-22, ДОКАЗАНА]**
**W0[0..14] ≥ 7518 AND W0[15]=0 → δe1=0x8000 с P=1.0.**
Верификация: 38554/38554 ✓.
```
Объяснение: C_e = 0x98c7e2a2, C_e[0..14]=25250
Условие: carry_15=1 ↔ W0[0..14] ≥ 7518. При W0[15]=0: bit15(e1_n)=0 → δe1=0x8000.
```

### **T_SIG1_LINEAR_CONST [П-22, КЛЮЧЕВАЯ]**
**Sig1 линейна над XOR:**
```
Sig1(e XOR δ) XOR Sig1(e) = Sig1(δ)   (точно, P=1.0)
```
При δe1=0x8000: δSig1(e1) = Sig1(0x8000) = **0x00400210** для ВСЕХ e1_n (5000/5000 ✓).
Аналогично: Sig0 линейна (T_SIG0_LINEAR), Maj линейна над XOR в первом аргументе.

### T_DCH_EXACT [П-22/П-23, ДОКАЗАНА]
**Точная формула без приближений:**
```
Ch(e XOR δ, f, g) XOR Ch(e, f, g) = δ AND (f XOR g)
```
Доказательство: Ch(x,y,z) = (x&y)^(~x&z) → разница = δ&y^δ&z = δ&(y^z) ■

## 4. ТЕОРЕМЫ ПОСЛЕДНИХ ИССЛЕДОВАНИЙ (П-23..П-25)

### **T_SCHEDULE_FULL_RANK [П-23, ДОКАЗАНА]**
Матрица XOR-расписания L: GF(2)^512 → GF(2)^1536 имеет **ранг 512/512** (полный ранг).
```
ker(L) = {0}: нет ненулевого ΔW[0..15] с ΔW[16..63] = 0
```
**Следствие:** Нельзя выбрать ΔW[0..15] ≠ 0 так, чтобы δW[16..63] = 0.

### **T_PERIOD3_CASCADE [П-23, ЭКСПЕРИМЕНТАЛЬНАЯ]**
**Адаптивный выбор ΔW2, ΔW5, ΔW8, ΔW11 даёт De3=De6=De9=De12=0 одновременно (1000/1000 ✓).**
Период-3 структура: De_{3k}=0 при k=1,2,3,4. В отличие от стандартного каскада (нули на [4..16]).
Основа: T_DW2_FREEDOM применимо к раундам 3, 6, 9, 12.

### **T_ANALYTIC_TRAIL [П-23, ВЕРИФИЦИРОВАНА]**
Аналитический XOR-след при δW0=0x8000, δW1=Sig1(0x8000), δW2...=0:
```
Round 1: δe1=0x8000  (P=1.0 при SC: W0[0..14]≥7518, W0[15]=0)
Round 2: δe2=0       (P=0.1256)
Round 3: δe3=0       (P=0.0636)
Round 4: δe4=0       (P=0.0338)
Round 5+: δa1≠0 входит через d-регистр → след разрушается
```
Ключевые формулы для следа:
- δT1_r = δSig1(e_r) XOR δCh(e_r,f_r,g_r) XOR δh_r  (линейно!)
- δSig1(e1) = Sig1(0x8000) = 0x00400210  (константа!)
- δCh(e1,f1,g1) = 0x8000 & (e_iv XOR f_iv) = 0  (бит15 нулевой!)

### **T_SC_A1 [П-24, ДОКАЗАНА + ВЕРИФИЦИРОВАНА 100000/100000]**
**δa1=0x8000 ⟺ bit15(a1_n)=0**
```
a1_n = (T1_0_base + T2_0 + W0) mod 2^32 = S_base + W0
S_base = 0xfc08884d

Точное условие: W0[15] = S_base[15] XOR carry_14
  carry_14 = 1 iff W0[0..14] ≥ 30643
```
Распределение δa1 (геометрическое):
```
P(δa1=0x8000)  = 0.5  (≈ 1/2)
P(δa1=0x18000) = 0.25 (≈ 1/4)
P(δa1=0x38000) = 0.125 (≈ 1/8)
...
```
Объяснение: каскад переносов при добавлении 0x8000 к a1_n.

### **T_JOINT_SC [П-24, ДОКАЗАНА]**
**Совместное SC для δe1=0x8000 AND δa1=0x8000 (оба P=1):**
```
SC_e:  W0[0..14] ≥ 7518  AND  W0[15] = 0       (carry_15=1 гарантирован)
SC_a1: W0[15] = S_base[15] XOR carry_14
     = 0 при W0[0..14] ≥ 30643                  (carry_14=1)
     = 1 при W0[0..14] < 30643                  (carry_14=0)

СОВМЕСТНАЯ SC: W0[0..14] ≥ 30643  AND  W0[15] = 0
P(совм. SC) = 2125/65536 ≈ 3.2% (из W0[0..15] пространства)
```
**P(δa1=0x8000 | SC_e) ≈ 0.50 (W0[15] независим от SC_e).**

### **T_WANG_ADAPTIVE [П-25, КЛЮЧЕВАЯ, ДОКАЗАНА + ВЕРИФИЦИРОВАНА 50000/50000]**
**Для ЛЮБОГО W0 (без SC!): адаптивный выбор δW1 → δe2=0 с P=1.0.**
```python
# Вычисление из W0:
e1_f = e1_n + dW0   (аддитивно, всегда)
dW1 = -(Sig1(e1_f) - Sig1(e1_n)) - (Ch(e1_f,f1,g1) - Ch(e1_n,f1,g1))
# Результат: T1_1_f = T1_1_n → e2_f = e2_n → δe2=0  (P=1.0!)
```
**Обоснование:** e1_f = e1_n + 0x8000 аддитивно, независимо от SC. Коррекция δW1 полностью аннулирует аддитивный сдвиг.
- 9206 уникальных значений δW1 (каждый W0 требует своего δW1).
- P=1.0 проверена для 50000 случайных W0 ✓.

### **T_ONE_CONSTRAINT [П-25, ТЕОРЕТИЧЕСКАЯ]**
**Каждое сообщение W_r даёт ровно 1 степень свободы (δW_r):**
```
Для δe_{r+1}=0: δW_r = -(δd_r + δh_r + δSig1(e_r) + δCh(...))   [одно уравнение]
Для δa_{r+1}=0: δW_r = -(δSig1(e_r) + δCh(...) + δSig0(a_r) + δMaj(...))  [другое!]
→ Нельзя одновременно δe=0 и δa=0 при одном δW_r.
```
**Следствие:** δa_r остаётся неконтролируемым при Wang-коррекции e-регистра.

### **T_SCHEDULE_PROPAGATION [П-25, ЭКСПЕРИМЕНТАЛЬНАЯ]**
При адаптивных δW[0..15]: δW[16..63] ≈ случайные (avg 16.1 бит из 32 ненулевые).
→ Раунды 17-64 получают хаотические δW (следствие T_SCHEDULE_FULL_RANK).
**Следствие:** Адаптивный Wang-след ограничен 16 раундами из 64.

## 6. АРХИТЕКТУРА КАСКАДА (П-13, финальная версия)

```python
def cascade_3param(W0, W1, DW0=1):
    """Работает для ЛЮБЫХ (W0, W1, DW0≠0) — поиск De3=0 не нужен."""
    Wn = [W0, W1] + [0]*14
    DWs = [0]*16; DWs[0] = DW0

    # T_DW2_FREEDOM: De3=0 адаптивно
    De3_nat = de(sha_r(schedule(Wn), 3), sha_r(schedule([(Wn[i]+DWs[i])&M for i in range(16)]), 3), 3)
    DWs[2] = (-De3_nat) & MASK

    # T_CASCADE_MAX: De4..De16=0 каскадом
    for step in range(13):
        wi = step+3; dt = step+4
        Wfc = [(Wn[i]+DWs[i])&MASK for i in range(16)]
        DWs[wi] = (-de(sha_r(schedule(Wn), dt), sha_r(schedule(Wfc), dt), dt)) & MASK

    # De17=0 при Da13+ΔW16=0 (стоит 2^32)
    return de17  # = 0 с P=2^(-32)
```

```python
def wang_adaptive_trail(W0, dW0=0x8000, n_rounds=16):
    """T_WANG_ADAPTIVE: δe1=...=δen=0 для любого W0 (без SC)."""
    Wn = [W0] + [random.randint(0,MASK) for _ in range(15)]
    Wf = [Wn[0] + dW0] + list(Wn[1:])
    state_n, state_f = IV_state, IV_state

    for r in range(n_rounds):
        a_n,b_n,c_n,d_n,e_n,f_n,g_n,h_n = state_n
        a_f,b_f,c_f,d_f,e_f,f_f,g_f,h_f = state_f
        # Вычислить δW_r для δe_{r+1}=0:
        dW_r = (-(d_f-d_n)-(h_f-h_n)-(Sig1(e_f)-Sig1(e_n))-(Ch(e_f,f_f,g_f)-Ch(e_n,f_n,g_n))) & MASK
        Wn[r] = random.randint(0,MASK)
        Wf[r] = (Wn[r] + dW_r) & MASK
        state_n = one_round(state_n, Wn[r], K[r])
        state_f = one_round(state_f, Wf[r], K[r])
    # Результат: δe1=...=δe_{n_rounds}=0 (P=1.0!)
    # НО: δa[1..n_rounds] ≠ 0 (неконтролируемо)
    # И: δW[16..63] ≠ 0 (schedule propagation)
```

```
T_BARRIER_16 [ФИНАЛЬНО]: De3..De18=0 стоит 2^64.
Причина: 2 независимых 32-битных условия (Da13+ΔW16=0 и Da14+ΔW17=0).
```

### XOR путь (Wang-adaptive)
```
T_ONE_CONSTRAINT: 1 δW_r → только 1 регистр. Нельзя одновременно δe=0, δa=0.
T_SCHEDULE_FULL_RANK: δW[16..63]≠0 при любых ненулевых δW[0..15].
→ Максимальная глубина Wang-следа: 16 раундов из 64.
```

### Общая позиция
```
Для КОЛЛИЗИИ SHA-256 нужно:
  - δe=0 И δa=0 (и все остальные 6 регистров) после 64 раундов
  - После сложения с IV: все 8 выходных слов должны совпасть

Текущий результат:
  - 15 нулей аддитивного De (из ~64 нужных) за 2^32
  - 16 нулей XOR De (только e-регистр) за ~1 (Wang adaptive)

Оба результата НЕ ведут к коллизии без прорыва в управлении a-регистром.
```

## 8. ОТКРЫТЫЕ ВОПРОСЫ И НАПРАВЛЕНИЯ

### Q1: Одновременное управление δe И δa (ПРИОРИТЕТ ВЫСОКИЙ)
```
Нужно: 2 уравнения → 2 параметра на раунд.
Идеи:
  (а) Двухблочные сообщения: 2 слова на раунд (M1[r] и M2[r]) → 2 степени свободы
  (б) Нелинейные пары: найти (W_r, W_r') такие что и δe=0, и δa=0
      Стоимость: для каждого раунда независимо → суммарно 2^{64} или хуже
  (в) Структурные условия на IV: изменить атаку на другую хэш-функцию с лучшими IV
```

### Q2: Продолжение Wang-следа за раунд 16
```
После раунда 16: W[16] определяется расписанием (T_SCHEDULE_FULL_RANK).
Идеи:
  (а) Статистический: собрать 2^32 (Wn, Wf) пар и искать δe17=0 случайно
  (б) Meet-in-the-middle: прямо от IV до r=32; назад от target до r=32
  (в) Аналитически: вычислить ΔW[16] = sig1(ΔW14) ⊕ ΔW9 ⊕ sig0(ΔW1) ⊕ ΔW0
      и проверить, создаёт ли он δe17=0 при некоторых W0..W15.
```

### Q3: Period-3 cascade + Wang combination
```
T_PERIOD3_CASCADE даёт De3=De6=De9=De12=0 (аддитивный).
Идея: применить Wang к XOR дифференциалу между двумя Period-3 парами.
```

### Q4: SAT для 17+ раундов с ослабленными условиями
```
Вместо De3..De18=0 рассмотреть: De3..De17=0 AND δe1=0x8000 (XOR).
Это смешанный аддитивный+XOR след → может быть решаем SAT.
```

## 10. ИНСТРУМЕНТЫ И БАЗОВЫЙ КОД

```python
MASK = 0xFFFFFFFF
K = [0x428a2f98, 0x71374491, ...]  # 64 константы SHA-256
IV = (0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a,
      0x510e527f, 0x9b05688c, 0x1f83d9ab, 0x5be0cd19)

def rotr(x, n):  return ((x >> n) | (x << (32-n))) & MASK
def sig0(x):     return rotr(x,7) ^ rotr(x,18) ^ (x>>3)
def sig1(x):     return rotr(x,17) ^ rotr(x,19) ^ (x>>10)
def Sig0(x):     return rotr(x,2) ^ rotr(x,13) ^ rotr(x,22)
def Sig1(x):     return rotr(x,6) ^ rotr(x,11) ^ rotr(x,25)
def Ch(e,f,g):   return ((e&f) ^ (~e&g)) & MASK
def Maj(a,b,c):  return (a&b) ^ (a&c) ^ (b&c)
def schedule(W16):
    W = list(W16) + [0]*48
    for i in range(16,64):
        W[i] = (sig1(W[i-2])+W[i-7]+sig0(W[i-15])+W[i-16]) & MASK
    return W
def sha_rounds(W16, n):  # возвращает state после каждого раунда
    W = schedule(W16); state = IV; states = [state]
    for i in range(n):
        T1 = (state[7]+Sig1(state[4])+Ch(*state[4:7])+K[i]+W[i]) & MASK
        T2 = (Sig0(state[0])+Maj(*state[:3])) & MASK
        state = ((T1+T2)&M, state[0], state[1], state[2],
                 (state[3]+T1)&M, state[4], state[5], state[6])
        states.append(state)
    return states

# Ключевые функции:
de(s1, s2, r) = (s2[r][4] - s1[r][4]) & MASK   # аддитивная De_r
da(s1, s2, r) = (s2[r][0] - s1[r][0]) & MASK   # аддитивная Da_r
xde(s1,s2,r)  = s2[r][4] ^ s1[r][4]            # XOR δe_r
```
