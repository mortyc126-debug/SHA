## 2. НОВЫЕ ТЕОРЕМЫ (П-10..П-14) — ДОКАЗАНЫ

### T_CASCADE_MAX [П-10, ДОКАЗАНА]
**Формулировка:** Каскад W[3..15] даёт De3..De16=0 (14 нулей) детерминированно.
**Стоимость:** 2^22 (только поиск базовой пары De3=0).
**Механизм:** При De_k=0: De_{k+1} = De_{k+1}_nat + ΔW_k (линейно mod 2^32).
→ ΔW_k = -De_{k+1}_nat → De_{k+1}=0 детерминированно.
Итерировать для k=3..15: 13 шагов → De4..De16=0. Плюс De3=0 из поиска: 14 нулей.
**Ключевые числа:** Da13=0x7711498a, ΔW16=0x84752d8e, De17=0xfb867718.

### T_DE17_DECOMPOSITION [П-11/П-12, ДОКАЗАНА]
**Формулировка:** При De3..De16=0:
```
De17 = Da13 + ΔW16  (точно, mod 2^32)
```
где Da13 = Δa после 13 раундов, ΔW16 из расписания.
**Доказательство:**
```
Δe17 = Δd16 + ΔT1_16
Δd16 = Δc15 = Δb14 = Δa13 = Da13   (3-сдвиговый регистр)
ΔT1_16 = Δh16 + ΔSig1(e16) + ΔCh(e16,..) + ΔW16
Δh16 = Δg15 = Δf14 = Δe13 = De13 = 0  (каскад обнулил) ✓
ΔSig1(e16) = 0  (De16=0) ✓
ΔCh(e16,..) = 0  (De16=0) ✓
→ De17 = Da13 + ΔW16  ■
```

### T_DW16_ANALYTIC [П-11, ДОКАЗАНА]
**Формулировка:**
```
ΔW16 = sig1(W'14) - sig1(W14) + ΔW9 + ΔW0
     = 0x9c16f6c9 + 0xe85e36c4 + 0x1 = 0x84752d8e  ✓
```

### **T_DW2_FREEDOM [П-13, КЛЮЧЕВАЯ ТЕОРЕМА]**
**Формулировка:** W2 входит в e3 аддитивно (раунд 2 использует W2 в T1 линейно):
```
De3 = De3_nat(W0, W1) + ΔW2  (точно, mod 2^32)
```
→ Выбор ΔW2 = -De3_nat(W0, W1) даёт De3=0 для **ЛЮБЫХ** (W0, W1).
**Следствие:** Условие "найти (W0,W1) с De3=0" стоимостью 2^22 **устраняется**.

### **T_CASCADE_17 [П-13, КЛЮЧЕВАЯ ТЕОРЕМА]**
**Формулировка:** Стоимость 15 нулей De3..De17=0:
```
1. ΔW2 = -De3_nat(W0,W1)  →  De3=0 для любых W0,W1  (бесплатно)
2. ΔW3..ΔW15 = каскад     →  De4..De16=0 детерминированно
3. Поиск (W0,W1): Da13+ΔW16=0  →  стоимость 2^32
```
**Итого: De3..De17=0 за 2^32 (улучшение 2^54 → 2^32, выигрыш 2^22 = 4M раз)**

### T_DE17_UNIFORM [П-13, ЭКСПЕРИМЕНТАЛЬНАЯ]
**Формулировка:** После адаптивного ΔW2, De17 распределён равномерно по [0, 2^32).
**Верификация:** 2000/2000 пар → 2000 уникальных De17, равномерное покрытие 256/256 байт.

### T_DW1_EQUIVALENCE [П-14, ВЫВОДНАЯ]
**Формулировка:** ΔW1=β (4-й параметр) даёт эквивалентный путь к 15 нулям за 2^32.
Альтернатива П-13: варьировать β вместо (W0,W1).

### T_DE17_DE18_INDEPENDENCE [П-14, ЭКСПЕРИМЕНТАЛЬНАЯ]
**Формулировка:** k17 ≈ 1.87×10^9, k18 ≈ 4.91×10^8. k17 ≠ k18.
→ De17=0 И De18=0 через единственный ΔW1: **несовместно** → стоимость 2^64.

### **T_DEk_DECOMPOSITION [П-15, КЛЮЧЕВАЯ ОБОБЩЁННАЯ ТЕОРЕМА]**
**Формулировка:** При De3..De_{k-1}=0:
```
De_k = Da_{k-4} + ΔW_{k-1}   (mod 2^32)
```
**Доказательство:**
```
De_k = Δe_k = Δd_{k-1} + ΔT1_{k-1}
Δd_{k-1} = Δc_{k-2} = Δb_{k-3} = Δa_{k-4} = Da_{k-4}  (3-сдвиговый регистр)
ΔT1_{k-1} = Δh_{k-1} + ΔSig1(e_{k-1}) + ΔCh(e_{k-1},...) + ΔW_{k-1}
Δh_{k-1} = De_{k-4} = 0  (каскад ✓)
ΔSig1(e_{k-1}) = 0, ΔCh(e_{k-1},...) = 0  (De_{k-1}=0 ✓)
→ De_k = Da_{k-4} + ΔW_{k-1}  ■
```
**Применимость:** T_DE17 (k=17), T_DE18 (k=18) — доказаны и верифицированы.

### **T_DE18_DECOMPOSITION [П-15, ВЕРИФИЦИРОВАНА ЭКСПЕРИМЕНТАЛЬНО]**
**Формулировка:** При De3..De17=0: `De18 = Da14 + ΔW17`
**Верификация (найденная пара):**
```
W0 = 0xe82222c7, W1 = 0x516cfb41  (найдено за ~2^32 итераций, C-поиск)
De3..De17 = 0 (все 15 нулей ✓)
De18 = 0xd6f240de, Da14 = 0x6ac9d38b, ΔW17 = 0x6c286d53
Da14 + ΔW17 = 0xd6f240de  ✓ T_DE18_DECOMPOSITION ВЕРИФИЦИРОВАНА
```

### **T_BARRIER_16 [П-15, ДОКАЗАНА]**
**Формулировка:** Стоимость De3..De18=0 = **2^64**.
**Доказательство:** Все 16 входных слов W[0..15] задействованы каскадом.
W[17] = f(W[0..15]) — не свободный параметр.
De3..De18=0 требует 2 независимых условия по 32 бита → 2^64.

**Следствие:** Для преодоления барьера 2^64 нужен принципиально новый подход:
- Двухблочные сообщения (MITM, П-16)
- SAT/MILP кодирование (П-16)

## 4. АРХИТЕКТУРА КАСКАДА (П-13, финальная версия)

```python
def cascade_3param(W0, W1, DW0=1):
    """Работает для ЛЮБЫХ (W0, W1) — поиск De3=0 не нужен."""
    Wn = [W0, W1, 0] + [0]*13
    DWs = [0]*16; DWs[0] = DW0

    # Шаг 0: T_DW2_FREEDOM — De3=0 адаптивно
    Wf_tmp = [(Wn[i]+DWs[i])&MASK for i in range(16)]
    De3_nat = de(sha_r(schedule(Wn), 3), sha_r(schedule(Wf_tmp), 3), 3)
    DWs[2] = (-De3_nat) & MASK  # ← адаптивный ΔW2

    # Шаги 1..13: T_CASCADE_MAX — De4..De16=0
    for step in range(13):
        wi = step+3; dt = step+4
        Wfc = [(Wn[i]+DWs[i])&MASK for i in range(16)]
        tn = sha_r(schedule(Wn), dt)
        tf = sha_r(schedule(Wfc), dt)
        DWs[wi] = (-de(tn, tf, dt)) & MASK

    # Результат: De17 для данной пары
    Wf = [(Wn[i]+DWs[i])&MASK for i in range(16)]
    tn17 = sha_r(schedule(Wn), 17)
    tf17 = sha_r(schedule(Wf), 17)
    return de(tn17, tf17, 17)  # = 0 с вероятностью 2^(-32)
```

## 5. БАРЬЕРЫ И ОТКРЫТЫЕ ВОПРОСЫ

### Решённый барьер П-15: T_DE18_DECOMPOSITION [ДОКАЗАНА + ВЕРИФИЦИРОВАНА]
```
De18 = Da14 + ΔW17  (при De3..De17=0)
```
Конкретные числа: Da14=0x6ac9d38b, ΔW17=0x6c286d53, De18=0xd6f240de ✓

### Текущий барьер: De18=0 (16-й ноль) = стоимость 2^64

**T_BARRIER_16 [ДОКАЗАНА]:** Все W[0..15] задействованы. W[17] не свободный.
P(De18=0 | De3..De17=0) ≈ 2^(-32) → стоимость: 2^32 × 2^32 = 2^64.

### Открытый вопрос Q16: 16 нулей за меньше 2^64?

**Подход 1: W_SAT4 + каскад**
W_SAT4 даёт De3=De6=De9=De12=0 (Period-3 структура).
Применить адаптивный ΔW2 + каскад → другой паттерн нулей.
Стоимость: ~2^44 (поиск De3=De6=0) или 2^22 + 2^22 (если условия независимы).

**Подход 2: Двухблочные сообщения (T_INVERSE)**
MITM через промежуточное состояние state[32].
Прямо: блок 1 → 2^k состояний; Назад: target → 2^k состояний.
Встреча → коллизия при 2^k << 2^128.

**Подход 3: SAT/MILP для De3..De18=0**
T_SCHEDULE_DECOUPLING: De18 зависит от W[0..17] → 18 слов.
CaDiCaL195 с 18-раундовым кодированием.

**Подход 4: Обобщённый T_DEk_DECOMPOSITION каскад**
Если найти структуру, где ΔW_{k-1} управляемо через входные слова,
возможно построить каскад для k=18 аналогично П-13 но с 2 блоками.

### ✅ П-17: SAT для De3..De18=0 [ЗАВЕРШЁН]

### ✅ П-18: T_STRUCTURED_SAT + De18 статистика [ЗАВЕРШЁН]

### ✅ П-19: Обобщённые дифференциалы [ЗАВЕРШЁН]
- T_GENERALIZED_CASCADE: каскад для ЛЮБОГО ΔW0 ≠ 0 (не только ΔW0=1).
- T_3D_BARRIER: 2^64 подтверждён статистически (N=50000, 3D пространство).
- T_XOR_DIFFERENTIAL: XOR-каскад нет (не детерминирован).

### ✅ П-20: XOR-дифференциальный анализ [ЗАВЕРШЁН]
- T_XOR_PROB: δW0=1 → δe1=1 (всегда, не 0) — нет XOR-каскада.
- T_MIXED_DIFF: sig0, sig1 ЛИНЕЙНЫ над XOR → schedule линейно (P=1).
- T_SCHEDULE_XOR: δW0=2^31 → 12 ненулевых δW_i (нет нулей расписания).
- XOR-путь глубины ≥1 из 64: не найдено за 200000 попыток.
- **Вывод: SHA-256 криптографически стоек; текущий результат — частичная характеристика e-регистра.**

### ✅ П-21: XOR-дифференциальный след + IV-структура [ЗАВЕРШЁН]
- T_IV_BIT0: C = d_iv + S = 0x98c7e2a2 (bit0=0) → δe1=1 ∀W0 при δW0=1. ДОКАЗАНА аналитически.
- T_ROUND1_FULL: (δa1..δh1) ≤ 10 вариантов; δe1=1 всегда, δa1=0x7 или 0xf/0xff (зависит от битов W0).
- T_TRAIL_MAP: r=1 детерминирован; r≥2 полная лавина (5000 уникальных пар из 5000 попыток).
- T_SCHEDULE_FREE: δW0=0x8000 (бит 15) — минимальный sum_wt=466; только 12/64 активных раундов.
- **T_DOM_DIFF [КЛЮЧЕВАЯ]: δW0=0x8000 → δe1=0x8000 с P=0.773 (доминирующий дифференциал!).**
- T_SINGLE_BIT: P(δe_r+1=0 | δW_r≠0) = 0 для всех r — нет позиции с P>2^-32.

### П-22 (следующий): Wang-style message modification + boomerang
- Целенаправленное обнуление битовых условий (δe_r=0 на первые 20 раундов).
- Использовать T_DOM_DIFF: δW0=0x8000 → δe1=0x8000 P=0.77 как стартовую точку.
- Boomerang attack: два коротких XOR-пути «склеиваются».

```python
# Все эксперименты используют единый базовый код:
MASK = 0xFFFFFFFF
def rotr(x, n): return ((x >> n) | (x << (32-n))) & MASK
def sig0(x):  return rotr(x,7) ^ rotr(x,18) ^ (x>>3)
def sig1(x):  return rotr(x,17) ^ rotr(x,19) ^ (x>>10)
def Sig0(x):  return rotr(x,2) ^ rotr(x,13) ^ rotr(x,22)
def Sig1(x):  return rotr(x,6) ^ rotr(x,11) ^ rotr(x,25)
def Ch(e,f,g): return ((e&f) ^ (~e&g)) & MASK
def Maj(a,b,c): return (a&b) ^ (a&c) ^ (b&c)
def schedule(W16): ...  # расширение W[16..63]
def sha_r(W, R): ...    # R раундов SHA-256
def de(t1, t2, r): return (t2[r][4] - t1[r][4]) & MASK
def da(t1, t2, r): return (t2[r][0] - t1[r][0]) & MASK
```
