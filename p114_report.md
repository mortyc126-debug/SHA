## EXP B: Карта мёртвых раундов

```
Для SHA-256, SHA-1, MD5: first_use(W[k]) = k (r=0..15)
DRI = 15 одинаков для всех трёх!

Это потому что W[k]=W64[k] для k<16 — прямое использование
в k-м раунде. Расписание у всех начинается одинаково.
```

---

## EXP C: DRI-классификатор

```
schedule_barrier = первый r где все W[0..15] использованы

MD5:    schedule_barrier = 16  из  64 раундов
SHA-1:  schedule_barrier = 16  из  80 раундов
SHA-256: schedule_barrier = 16  из  64 раундов

Все три одинаковы! Но SHA-256 устойчив, остальные нет.
```

**Уточнение:** классификатор через DRI и schedule_barrier
одинаков для всех трёх. Различие — в том что происходит
ПОСЛЕ schedule_barrier:

```
MD5/SHA-1: после r=16 нет никакого барьера.
           Wang продолжает 48/64 раундов свободно.

SHA-256:   после r=16 наступает Wang-барьер r=17.
           S1 + информационный барьер P(δe=0)=2^-32
           блокируют Wang ровно здесь.
```

## EXP D: T_BARRIER_EQUALS_SCHEDULE — финальная теорема

```
T_WANG_BARRIER_SCHEDULE:

  Wang-атака строит sufficient conditions через мёртвые раунды.
  После schedule_barrier мёртвые раунды исчезают.

  Хеш уязвим к Wang тогда и только тогда, когда:
  schedule_barrier(H) >= total_rounds(H)
  ИЛИ нет S1-аналога блокирующего Wang сразу после barrier.

T_BARRIER_EQUALS_SCHEDULE (для SHA-256):
  Wang-барьер (r=17) = schedule_barrier(SHA-256) + 1 = 17
  Это НЕ совпадение — одно и то же явление.
```

## ФИНАЛЬНЫЙ СИНТЕЗ П-102..П-114

```
Устойчивость SHA-256 к Wang-атакам = три взаимосвязанных фактора:

1. T_DEAD_ROUNDS:
   Каждый W[k] невидим для функции сжатия до раунда k.
   DP_eff(r) = 1.0 детерминированно для r < first_use(W[k]).
   Это одинаково для MD5, SHA-1, SHA-256.

2. T_S1_NECESSITY:
   S1(e)=ROTR(6,11,25) разрушает структуру δe за 1 применение.
   DP_eff падает от 0.5 до ≈0 через propagation цепочку.
   В MD5 нет S1-аналога → структура δ сохраняется.

3. T_BARRIER_EQUALS_SCHEDULE:
   В SHA-256: schedule_barrier = 16 = Wang-барьер - 1.
   Расписание исчерпывает мёртвые раунды ровно там,
   где S1 блокирует Wang.
   В MD5/SHA-1: после schedule_barrier нет S1-блокировки.

ИТОГ:
  SHA-256 устойчив не из-за одного свойства,
  а из-за идеального совпадения трёх:
  расписание + нелинейность S1 + информационный барьер
  работают вместе и взаимно усиливают друг друга.
```
