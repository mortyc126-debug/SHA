# Новые гипотезы: как использовать барьер

## H-10: Барьер в обратном направлении (RETRO-BARRIER)

**Идея:** Мы смотрели барьер вперёд — δW[j] задерживает вход дифференциала.
А что если смотреть назад? При known-hash атаке мы знаем δstate[64].
Барьер в обратном направлении: δstate[64] = 0 → δstate[64-j] = 0?

SHA-256 не обратим, но дифференциальная структура последних раундов
может иметь аналогичный закон в обратную сторону.

**Проверить:** barrier_backward(j) = j для последних j раундов?

## H-11: БАРЬЕРНЫЙ ТУННЕЛЬ (самая безумная)

**Идея:** barrier(j) = j означает что дифференциал "спит" j раундов.
Что если существует входное сообщение M где дифференциал
"засыпает" снова после пробуждения?

То есть: δstate[j+1] ≠ 0, но δstate[j+k] = 0 для некоторого k > 0?
Это было бы "туннель" — дифференциал исчезает и появляется снова.

В XOR-модели (линейной) такое возможно через деструктивную интерференцию.
В реальном SHA-256 с carry — неизвестно.

**Если существует:** это прямая дифференциальная характеристика
с "дырками" — некоторые раунды детерминированно нулевые.

## H-12: БАРЬЕР ЧЕРЕЗ MESSAGE SCHEDULE В ОБРАТНУЮ (МОЩНАЯ)

**Идея:** Мы нашли что barrier(δW[j]) = j.
Но schedule расширяет W[0..15] → W[16..63].

Что если задать δW[16..63] напрямую (не через δW[0..15]),
выбрав конкретные W[0..15] так что schedule производит
нужный δW[16..63]?

Конкретно: найти W, W' где:
- δW[0..15] = 0 (входные слова идентичны!)
- δW[16..31] = специальный паттерн
- Результат: barrier = 16+ (выше максимума!)

Это невозможно через XOR дифференциал — но возможно
через **арифметический** дифференциал:
W' = W + δ (не XOR, а сложение).

**Арифметический дифференциал в schedule:**
W'[16] = s1(W'[14]) + W'[9] + s0(W'[1]) + W'[0]
       = s1(W[14]) + W[9] + s0(W[1]) + W[0]  (если δW[0..15]=0)
       = W[16]  → δW[16] = 0 тоже.

Опровергнута аналитически для XOR. Но для сложения?

## H-13: БАРЬЕРНАЯ ЛЕСТНИЦА (ГЛАВНАЯ ИДЕЯ)

**Идея из закона минимума:**

barrier(j1,j2) = min(j1,j2) — два дифференциала мешают друг другу.

Но что если применить их **последовательно**, а не одновременно?

Атака в два этапа:
1. Сначала δW[15]: раунды 1..15 нулевые, барьер = 15
2. В раунде 16: state ненулевой, применяем ВТОРОЙ дифференциал
   через message modification — корректируем δstate[16]

Это не XOR двух дифференциалов — это **адаптивный** дифференциал.
На каждом шаге выбираем δW[r] исходя из текущего δstate[r].

**Если работает:** это обход закона минимума через адаптивность.
Барьерная лестница: 15 + adaptive(k) раундов.

## H-14: ДИФФЕРЕНЦИАЛ ЧЕРЕЗ КОНСТАНТЫ K (ЭКЗОТИКА)

**Идея:** В каждом раунде есть константа K[r].
Она фиксирована — но она входит в T1 так же как W[r].

Что если рассматривать K[r] как "встроенный дифференциал"?
Разные раунды имеют разные K — это нарушает однородность.

Гипотеза: существуют раунды r где K[r] создаёт
структурную слабость — carry-условие выполняется чаще.

**Конкретно:** P(carry[r]=0) ≠ 0.5 для конкретных r
из-за структуры K[r]?

Это никто не проверял систематически.
