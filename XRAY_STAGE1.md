# SHA-256 РЕНТГЕН — Полная карта с уравнениями
# Этап 1: Структурный скелет (один раунд)

## 1. РАУНДОВАЯ ФУНКЦИЯ

Вход: state = (a, b, c, d, e, f, g, h) ∈ (Z/2³²)⁸, W[r], K[r]
Выход: state' = (a', b', c', d', e', f', g', h')

```
T1 = h + Σ₁(e) + Ch(e,f,g) + K[r] + W[r]     mod 2³²
T2 = Σ₀(a) + Maj(a,b,c)                        mod 2³²
a' = T1 + T2                                     mod 2³²
e' = d + T1                                      mod 2³²
b' = a,  c' = b,  d' = c                         (сдвиг, точная копия)
f' = e,  g' = f,  h' = g                         (сдвиг, точная копия)
```

## 2. КОМПОНЕНТЫ (формулы)

```
Σ₁(x) = ROTR(x,6) ⊕ ROTR(x,11) ⊕ ROTR(x,25)    GF(2)-линейная
Σ₀(x) = ROTR(x,2) ⊕ ROTR(x,13) ⊕ ROTR(x,22)    GF(2)-линейная
Ch(e,f,g) = (e ∧ f) ⊕ (¬e ∧ g)                    степень 2 над GF(2)
Maj(a,b,c) = (a ∧ b) ⊕ (a ∧ c) ⊕ (b ∧ c)         степень 2 над GF(2)
```

ANF-формы:
```
Ch[i]  = e[i]·f[i] ⊕ e[i]·g[i] ⊕ g[i]           (3 монома, степень 2)
Maj[i] = a[i]·b[i] ⊕ a[i]·c[i] ⊕ b[i]·c[i]      (3 монома, степень 2)
```

## 3. РАСПИСАНИЕ (schedule)

```
W[0..15] = M (вход, свободные 512 бит)
W[r] = σ₁(W[r-2]) + W[r-7] + σ₀(W[r-15]) + W[r-16]   для r = 16..63

σ₀(x) = ROTR(x,7) ⊕ ROTR(x,18) ⊕ SHR(x,3)      GF(2)-линейная
σ₁(x) = ROTR(x,17) ⊕ ROTR(x,19) ⊕ SHR(x,10)     GF(2)-линейная
```

Над GF(2): schedule ЛИНЕЕН (σ₀, σ₁ — XOR ротаций).
Над Z/2³²: schedule НЕЛИНЕЕН (+ порождает carry).

## 4. ФИНАЛИЗАЦИЯ (feedforward)

```
H[i] = state_final[i] + IV[i]   mod 2³²   для i = 0..7

IV = (0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a,
      0x510e527f, 0x9b05688c, 0x1f83d9ab, 0x5be0cd19)

K[0..63] = дробные части кубических корней первых 64 простых чисел × 2³²
```

## 5. СДВИГОВЫЙ РЕГИСТР (shift register)

```
state[r] = (a[r], a[r-1], a[r-2], a[r-3], e[r], e[r-1], e[r-2], e[r-3])

Полная эквивалентность:
  b[r] = a[r-1]      f[r] = e[r-1]
  c[r] = a[r-2]      g[r] = e[r-2]
  d[r] = a[r-3]      h[r] = e[r-3]

Длина регистра L = 4 (a→b→c→d, e→f→g→h)
Из 256 бит state: 64 свежих (a,e) + 192 копии = 75% избыточны
```

## 6. КЛЮЧЕВЫЕ ТОЖДЕСТВА (доказанные)

Створочне число:
```
a[r] - e[r] = Σ₀(a[r-1]) + Maj(a[r-1],a[r-2],a[r-3]) - a[r-4]   mod 2³²
```
Верификация: 600000/600000 = 100% (раздел 202 методички)

Труба (pipe conservation):
```
(a + e)[r] = (b + f)[r+1] = (c + g)[r+2] = (d + h)[r+3]   mod 2³²
```
Время жизни: 4 раунда = L = τ★

Невидимое звено:
```
e[R] = a[R-4] + T1[R-1]    mod 2³²
a[R-4] ∉ {H[0]..H[7]}      (выпал за край shift register)
```
→ κ = 0 для SHA-256 (L=4). При L=3: κ = 32. Верифицировано.

Декомпозиция δe:
```
δe[r+1] = δa[r-3] + δT1[r]    mod 2³²    (при δe[r-2..r]=0)
δT1[r] = δh[r] + δΣ₁(e[r]) + δCh[r] + δW[r]
```

## 7. CARRY (побитовая формула)

Сложение a + b mod 2³²:
```
sum[k] = a[k] ⊕ b[k] ⊕ carry[k-1]
carry[k] = MAJ(a[k], b[k], carry[k-1])
carry[-1] = 0
```

GPK-классификация:
```
a[k]=0, b[k]=0 → K (Kill):       carry_out = 0 безусловно
a[k]=0, b[k]=1 → P (Propagate):  carry_out = carry_in
a[k]=1, b[k]=0 → P (Propagate):  carry_out = carry_in
a[k]=1, b[k]=1 → G (Generate):   carry_out = 1 безусловно
```

Carry = однопроходное сканирование GPK-строки (не рекурсия).
P(G) = P(K) = 1/4, P(P) = 1/2 для случайных операндов.

## 8. ДИФФЕРЕНЦИАЛЬНЫЕ ФОРМУЛЫ

XOR-дифференциал Ch при δe=0:
```
δCh[i] = e[i]·δf[i] ⊕ (1⊕e[i])·δg[i]    (линейна по δf, δg)
```

XOR-дифференциал Ch при δe≠0, δf=δg=0:
```
δCh = δe ∧ (f ⊕ g)    (линейна по δe)
```

Sensitivity (Wang):
```
∂(δe[r+1])/∂(δW[r]) = 1   (нечётно → Newton за 1 шаг)
```

Wang chain:
```
δW[r] = -(δd[r] + δh[r] + δΣ₁(e[r]) + δCh[r])   mod 2³²
→ δe[r+1] = 0   гарантированно для r = 1..15
```
