# П-75: CASCADE для 8 регистров — анализ и переориентация (Март 2026)

## Постановка

Попытка расширить CASCADE_6ZEROS (е-регистр) на все 8 регистров одновременно.
Цель: найти DW с Da_r = De_r = 0 для r ∈ {3,6,9,12,15,17}.

---

## Результат 1: нет независимых параметров

В SHA-256 любое W[j] влияет на Da_r И De_r одновременно через T1:

```
T1 = h + Σ1(e) + Ch(e,f,g) + K + W[j]
e_new = d + T1    (через T1)
a_new = T1 + T2   (тоже через T1)
```

Проверено численно: для r=3,6,9 все доступные W[j] дают Da≠0 И De≠0
одновременно. Нет ни одного W[j] влияющего только на a или только на e.

---

## Результат 2: 2D башня барьерна при k≥2

Для нетривиального (d0,d1) с Da_3=De_3=0 mod 2:

```
k=1: 56% сидов имеют нетривиальное решение
k=2: 26% (делится примерно пополам на каждом уровне)
k=8: 0.5%
```

Это T_MOD4_BARRIER (П-47) в 2D — абсолютный барьер подтверждён.
2D система не является p-адически гладкой даже в 1D проекциях.

---

## Результат 3: XOR-пространство работает иначе

Матрица J_{2×16} для {Da_r XOR, De_r XOR}:

```
r=3:  rank≈1.78 → 2^{14}-2^{15} нетривиальных XOR-решений
r=12: rank=2.00 → 2^{14} нетривиальных XOR-решений
r=17: rank=2.00 → 2^{14} нетривиальных XOR-решений
```

GF2 решения для 2 регистров одновременно всегда существуют.
**Но подъём до точности снова ломается через T_FREESTART_NONSMOOTH.**

---

## Главный вывод

```
ТЕОРЕМА T_8REG_CASCADE_IMPOSSIBLE (П-75):

  Аддитивный CASCADE для (Da_r, De_r)=0 одновременно
  не может быть построен каскадным методом потому что:

  1. Нет независимых параметров: W[j] всегда влияет на оба регистра.
  2. 2D p-адический подъём барьерен при k≥2 (T_MOD4_BARRIER).
  3. Единственный путь — joint search 2^64 или альтернативное пространство.

  В XOR-пространстве GF2 решение существует (rank=2, 2^14 решений),
  но подъём до точности ломается так же как в 1D free-start (П-68A).
```

---

## Переориентация: П-76

Аддитивный каскад на 8 регистров — тупик. Правильный следующий шаг:

**Перейти в XOR/дифференциальное пространство**, как это делают учёные.
Взять опубликованный дифференциальный путь Mendel et al. для 28 шагов
и реализовать на нашей кодовой базе с использованием:
- T_DEP как constraint (50% сокращение MILP переменных)
- T_GF2_BIJECTION для SFS начального решения
- T_NONLINEAR_DIFFUSION_RATE для предсказания где путь устойчив
