# SHA-256 Дифференциальный Криптоанализ
## Раздел 104 | Сессии 59–70 | Март 2026
### Самоатака: SHA-256 атакует SHA-256

---

## Контекст

Раздел 104 документирует исследование принципиально
нового направления: атака не инструментами снаружи,
а изнутри — используя SHA-256 против себя.

Исходная идея: авторы SHA-256 знали существующие методы
и защищались именно от них. Значит нужно атаковать то,
для чего методов не существует вообще.

---

## Исследованные направления

### Направление 1: Разрыв модель/реальность

**Тезис:** SHA-256 ≠ случайный оракул. Все нижние границы
сложности атак доказаны для случайного оракула. Для
конкретной функции с нашими теоремами — нижних границ нет.

**Что нашли:**

| Тест | Результат | Сигнал |
|---|---|---|
| Предсказание SHA(M^e) из SHA(M) | 0.5076 точность | слабый |
| std(E[HW] по carry-группам) | 0.37 | слабый |
| P(SHA(M)[0] бит по carry[0..5]) | макс отклонение 0.122 | умеренный |

**Вывод:** Разрыв существует и измерим (P=0.5076 > 0.5),
но слишком мал для практической атаки.

---

### Направление 2: Адаптивная атака на Оракула

**Тезис:** Адаптивные запросы к SHA-256 с использованием
внутренней структуры (carry-профиль) дают преимущество
над случайным поиском.

**Что нашли:**

| Тест | Результат |
|---|---|
| Умный vs случайный поиск (2 байта) | 1.00× — нет ускорения |
| Carry-навигация по кластерам | 0 кластеров с ≥2 M в 100K |

**Вывод:** Адаптивность не помогает — SHA-256 разрывает
carry-структуру при переходе к выходу.

---

### Направление 3: Нелинейное пространство

**Тезис:** Строим пространство специально под SHA-256
где нелинейность структурирована.

**Что нашли:**

| Пространство | Структура |
|---|---|
| Φ-пространство (carry+margin+T1T2+φ) | 4.73× vs случайное ✓ |
| Квотиентное M/~carry | хеши случайны внутри класса |
| Траекторное расстояние j=15 vs j=0 | 0.34× — в 3× ближе ✓ |
| PCA хешей | 1.00× — нет структуры |

**Вывод:** Φ-пространство структурировано (4.73×) — это
реальное нелинейное пространство специфичное для SHA-256.

---

### Направление 4: Нелинейные матрицы J_M

**Тезис:** J_M[i][j] = SHA(M)[i] XOR SHA(M^e_j)[i] —
нелинейная матрица влияния. Ищем M где rank(J_M) < 256.

**Что нашли:**

```
rank(J_M) = 256 для всех 30 протестированных M
corr(J_M1, J_M2) ≈ 0 — матрицы независимы
Плотность J_M = 0.5019 — идеальный лавинный эффект
```

**Вывод:** J_M всегда полного ранга. Ядро тривиальное.
Прямых коллизий через J_M нет.

---

### Направление 5: SHA-256 атакует SHA-256 (итерация)

**Тезис:** SHA-256 применённый к своему выходу создаёт
динамическую систему с особой структурой.

**Что нашли:**

| Тест | Результат | Значимость |
|---|---|---|
| corr(carry(M), carry(SHA(M))) | 0.0078 | нет |
| 2-цикл мин расстояние | **90/256** | **есть (vs 128)** |
| Почти-фикс. точка HW(SHA(H)^H) | **92/256** | **есть (vs 128)** |
| Автокорреляция цепочки SHA^k | -0.003 | нет |
| SHA на Im(SHA) vs случайные | p=0.025 | слабый |
| Лучший предсказатель SHA² | **0.5127** | **есть** |

## Новые принципы П-51..П-55

**П-51:** SHA-256 ≠ случайный оракул — измеримо.
Carry-профиль несёт информацию недоступную оракулу.
Разрыв существует но мал в известных метриках.

**П-52:** Φ-пространство (carry+margin+T1T2+φ) структурировано
в 4.73× сильнее случайного. Это реальное нелинейное
пространство специфичное для SHA-256.

**П-53:** Im(SHA-256) статистически однородно в простых
метриках — SHA-256 разрывает carry-структуру на каждом шаге.
Но 2-цикл на 30% ближе к нулю чем в случайной функции.

**П-54:** Нелинейные матрицы J_M полного ранга для всех M.
Лавинный эффект реализован идеально — нет точек вырождения.

**П-55:** Самоатака = использование SHA-256 как генератора
структурированных входов для SHA-256. Оракул задаёт
вопросы оракулу. Направление открыто.

## Метрики слабых сигналов

Собираем все обнаруженные слабые сигналы:

```
P(предсказание SHA(M^e)) = 0.5076  (+0.0076 над случайным)
2-цикл мин расстояние   = 90/256   (-30% от ожидаемого 128)
KS-тест Im(SHA)          = p=0.025  (граница значимости)
Предсказатель SHA²       = 0.5127   (+0.0127 над случайным)
Φ-пространство PCA       = 4.73×    (vs случайное 1.0×)
Траекторное расстояние   = 0.34×    (j=15 vs j=0)
carry_same[r=16] best H0 = 0.620    (+0.027 vs стандарт 0.593)
```

Все сигналы в одном направлении — SHA-256 не случаен.
Но каждый в отдельности слишком мал для атаки.
