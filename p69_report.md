## Направление 2: Механизм нелинейности

### T_NONLINEAR_DIFFUSION_RATE (П-69B)

Матрица нарушений строго нижнетреугольная — добавление `2·e_j` к δ нарушает бит 0
раунда `i` только при `i ≥ j+2`. Три зоны диффузии:

```
Расстояние i-j | P(нарушение бита 0) | Интерпретация
     1         |      ~0.02          | 1 раунд задержки, почти нет
     2         |      ~0.35          | нелинейность включилась
    ≥3         |      ~0.50          | полное перемешивание
```

**Аналитический механизм:** `Sigma1(x) mod 2 = x[6] XOR x[11] XOR x[25]`.
Добавление `+2` меняет бит 1 входа. Бит 6 выхода Sigma1 меняется ⟺ биты 1..5 все = 1,
то есть с `P = 1/32 ≈ 3.1%`. Это точно совпадает с наблюдаемым `P ≈ 0.029`.
Затем после каждого раунда нарушение amplifies до P → 0.5 за 3 раунда.

```
ТЕОРЕМА T_NONLINEAR_DIFFUSION_RATE (П-69B):

  SHA-256 достигает полного битового перемешивания за 3 раунда
  в контексте возмущения +2 на входном слове.

  Первичный механизм: Sigma1 — rotation XOR — переносит бит 1 входа
  в бит 0 выхода с P = 1/32 (carry через 5 бит).

  Следствие: Hensel lifting обрывается на k=2 потому что коррекция
  бита 1 (добавление 2·e_j) немедленно нарушает бит 0 через Sigma1
  в раундах i ≥ j+2.

  Это даёт аналитическое объяснение T_GLOBAL_BARRIER:
  барьер возникает не из «сложности» SHA-256 в целом,
  а из конкретного carry-механизма в Sigma1 на уровне k=2.
```

## Направление 3: Wang SC

**Аналитическая формула:**

При `De16=0` (что обеспечивается 1D башнёй по целевым раундам):
```
De17 = Dh16 + DW15_schedule  (mod 2^32)
     = De13 + DW15_schedule   (3-сдвиговый регистр: Dh16 = De13)
```

**SC_17:** `De13 + DW15 ≡ 0 (mod 2^32)` — достаточное условие для `De17 = 0`.

**Проблема:** нахождение `W` такого что одновременно:
- `De3 = De6 = De9 = De12 = De15 = 0` (нетривиально, цена ≥ 2^32)
- `De13 + DW15 = 0` (дополнительное условие на расписание)

требует полноценной Wang-атаки с backward-совместимыми модификационными переменными.
Это самостоятельная задача, выходящая за рамки текущей серии.
