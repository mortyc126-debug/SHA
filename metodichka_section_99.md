# SHA-256 Дифференциальный Криптоанализ
## Раздел 99 | Сессии 31–36 | Март 2026
### Специализированные метрики и вырожденные траектории

---

## Контекст

Стандартные инструменты криптоанализа — HW, корреляция Пирсона,
chi-square, энтропия Шеннона — разработаны для случайных процессов.
SHA-256 не случайный процесс. Он детерминированный, арифметический,
структурированный через конкретные константы H0 и K.

В этом разделе впервые применены метрики специфичные для
внутренней алгебры SHA-256.

## T_CARRY_THRESHOLD — пороговая функция

### Формулировка

```
carry[0](W[0]) монотонно неубывает как функция от W[0].

Порог: W[0] > 0x04189370 → carry[0] = 1
       W[0] ≤ 0x04189370 → carry[0] = 0

P(carry[0]=1) = (2^32 - 0x04189370) / 2^32 ≈ 0.0155
```

### Доказательство

```
T1_base[0] = H0[7] + Σ1(H0[4]) + Ch(H0[4],H0[5],H0[6]) + K[0]
           = 0x5be0cd19 + 0x3587272b + 0x1f85c98c + 0x428a2f98
           = 0xf377ed68

T2[0] = Σ0(H0[0]) + Maj(H0[0],H0[1],H0[2])
       = 0x08909ae5

carry[0] = 1 ⟺ T1_base + W[0] + T2 > 2^32
         ⟺ W[0] > 2^32 - T1_base - T2
         ⟺ W[0] > 0x04189370  ✓
```

Нарушений монотонности: 0/10000. Это строгий порог.

## T_T1_DOMINANCE — структурная аномалия раунда 0

```
E[T1/(T1+T2)] в раунде 0:   0.885  (отклонение +0.385)
E[T1/(T1+T2)] в раундах 1+: 0.500  (равновесие)
```

T1 занимает 88.5% суммы в раунде 0 — это следствие T_K0_SPECIAL.
H0 выбраны через √простых чисел так что T1_base ≈ 0.98×2^32.

После раунда 0 система быстро приходит к равновесию T1≈T2.

## T_MARGIN_UNIFORM — конструктивная устойчивость

### Определение

```
margin[r] = |T1[r] + T2[r] - 2^32| / 2^32

= расстояние до ближайшей границы переполнения (нормированное)
```

### Результат

```
E[margin[r]] ≈ 0.333 для всех r = 1..63
std[margin[r]] ≈ 0.235

Раунд 0: E[margin[0]] = 0.472 (особенный — дальше от границы)
```

Это **не случайность**. Равномерное среднее margin = 0.25.
SHA-256 держит margin = 0.333 — на 33% выше среднего.
Это конструктивное свойство обеспечивающее устойчивость.

## T_CARRY_LONG_CORR — межраундовые корреляции carry

### Полный скан 2016 пар раундов (N=100K):

```
Значимых пар (|corr| > 3σ = 0.0095):  4

carry[10] × carry[20]: corr = +0.012  Δr=10
carry[49] × carry[56]: corr = -0.011  Δr=7
carry[0]  × carry[40]: corr = -0.010  Δr=40
carry[6]  × carry[61]: corr = +0.010  Δr=55
```

Корреляции слабые (0.010-0.012) но статистически значимые.
Все корреляции > 3σ при N=100K. Это первые измеренные
дальнодействующие carry-корреляции в SHA-256.

### Значение

Эти корреляции слишком малы для прямой атаки.
Но их существование означает что carry-профиль
не полностью независим по раундам.

## T_DEGEN_TRAJECTORY — вырожденные траектории

### Определение

```
M называется ε-вырожденным если:
sum(carry[r]=0 for r in 17..40) ≥ (1-ε) × 24
```

### Результат

```
Случайное M:      E[нулей в r=17..40] = 12/24
Hill-climbing:    найдено M с 22/24 нулями
Максимум:         20 нулей подряд начиная с r=17
```

### T_DEGEN_CONNECTED

```
Пусть M1, M2 — два ε-вырождённых M.
Тогда существует путь M1 → M2 через последовательные
изменения слов W[i] такой что все промежуточные M
имеют score ≥ 17/24.

Средний score на пути: 24.9/24
Минимум на пути: 17/24
```

Класс вырождённых M — связное многообразие, не изолированные точки.

## T_PROFILE_SPACE — сжатие carry-профилей

### Результат

```
Теоретическое пространство carry-профилей: 2^64
Реально занятых профилей (N=200K):         ~2^18
Степень сжатия:                             2^46
```

46 из 64 carry-битов детерминированы предыдущими.
Только ~18 carry-битов независимы.

### Интерпретация

Carry-профиль несёт ~18 бит информации о M, а не 64.
Остальные 512-18=494 бита M свободны при фиксированном профиле.

## T_LYAPUNOV_NEUTRAL — нейтральный режим

### Показатель Ляпунова

```
λ(r) = E[log2(HW(δstate[r+1])/HW(δstate[r]))]
```

```
r=2:     λ = +2.70  (начальное расхождение)
r=3..6:  λ убывает от +1.15 до +0.13
r=7..63: λ ≈ 0      (нейтральный режим)
```

SHA-256 не экспоненциально расходится после r=7.
Это **не** хаотическая система в обычном смысле.
Псевдослучайность достигается не через хаос, а через
быстрый выход на нейтральный режим.

## Специализированные метрики SHA-256

Впервые предложены и измерены следующие метрики:

| Метрика | Формула | Что измеряет |
|---|---|---|
| Margin[r] | \|T1+T2-2^32\|/2^32 | Расстояние до нелинейности |
| T1-balance[r] | T1/(T1+T2) | Баланс слагаемых раунда |
| Carry-entropy[r] | P(bit\|carry=1) - P(bit\|carry=0) | Энтропия взвешенная по carry |

| Profile-dim | log2(\|{carry-profiles}\|) | Информационная размерность |
| Degen-score | \|{r: carry[r]=0, r=17..40}\| | Степень вырождённости траектории |

## Новые принципы

**П-30:** Стандартные метрики (HW, корреляция, chi-square) слепы
к структуре SHA-256 — они разработаны для случайных процессов.

**П-31:** Margin[r] — специализированная метрика для SHA-256.
E[margin]=0.333 — конструктивное свойство обеспечивающее безопасность.

**П-32:** SHA-256 работает в нейтральном режиме (λ≈0) начиная с r=7.
Это не хаос — это быстрый выход на псевдослучайный режим.

**П-33:** Класс вырождённых траекторий связен и нетривиален.
Вырожденные M не изолированы — они образуют многообразие.

**П-34:** carry-профиль несёт ~18 бит, а не 64.
Пространство профилей сжато в 2^46 раз относительно теоретического.
