## П-107A: T_DA_V2_PROPAGATION — АНАЛИТИКА И ПЕРЕСМОТР

### Раунд 1: точная формула

```
δW[0] = 0x8000 = 2^15
Раунд 1: оба состояния стартуют из одного IV
  δT1[1] = δW[0] = 0x8000,  v2=15
  δT2[1] = 0  (δS0=δMaj=0 — одинаковые входы)
  δa[1]  = 0x8000,  v2=15
  δe[1]  = 0x8000,  v2=15  ✓ (верифицировано)
```

### v2-профиль по раундам r=1..16

```
r= 1:  v2_mode=15  P(da odd)=0.0%    ← δa нечётно? НЕТ (v2=15)
r= 2:  v2_mode= 2  P(da odd)=100.0%  ← Wang компенсирует, нечётность
r= 3:  v2_mode= 1  P(da odd)=32.7%
r= 4:  v2_mode= 0  P(da odd)=67.3%   ← стабилизация ~66%
r= 5..16: P(da odd) ≈ 66%  (устойчивый инвариант)
```

### Пересмотр T_DA_DETERMINISTIC_LOW

```
П-106 показал: mod4(δa[17])=3 в 100% случаев (50 000 цепочек)
П-107 показал: mod4(δa[17]) равномерно 25% каждое (5 000 цепочек)

Разрешение: результат П-106 — артефакт конкретного прогона.
Реальный инвариант: P(δa[r] нечётно) ≈ 66% для r≥4

Новый закон T_DA_ODD_BIAS:
  При δW[0]=0x8000: P(v2(δa[r])=0) ≈ 2/3 для r=4..16
  Это устойчивый ненулевой bias — не полная детерминация,
  но значимое отклонение от равновероятного 50%.

  Причина: mod-арифметика T1+T2 при нечётном δW[0]
  создаёт устойчивое предпочтение нечётных δa.
```

## П-107B: G5 НА ARX-ХЕШАХ

### Blake2s (10 раундов, 512-бит state)

```
DEP по раундам:
  r=1:  79.69%
  r=2: 100.00%  ← оракул

r*(Blake2s) = 2 из 10
r*/Rounds   = 0.200
G5 рейтинг  = 5.0
τ(Blake2s)  ≈ 2.3 раунда
```

### ChaCha20 (20 раундов)

```
DEP = 0% на всех 20 раундах

Причина: ChaCha20 — потоковый шифр, не хеш.
XOR-дифференциал δA=1 проходит насквозь без сжатия.
DEP=0 означает полную детерминацию выходного дифференциала —
это не слабость, а архитектурное свойство ARX без компрессии.

G5 для ChaCha20 = 1.0 (вне шкалы — не хеш)
```

### Обновлённый рейтинг G5

```
Хеш           r*   Rounds  r*/Rounds  G5 рейтинг
SHA-256         3     64     0.047      21.3  ← лидер
Keccak-f[200]   2     18     0.111       9.0
Blake2s         2     10     0.200       5.0
SHA-1          21     80     0.263       3.8
MD5            21     64     0.328       3.0  ← слабейший
ChaCha20       —      20     —           —   (не хеш)
```

**Вывод:** G5 применим только к хеш-функциям с компрессией.
Потоковые шифры (ChaCha20) выпадают из шкалы по архитектурным причинам.

## П-107C: KECCAK θ-ЛИНЕАРИЗАЦИЯ

### Линейность θ: доказано экспериментально

```
P(θ линейна) = 10 000 / 10 000 = 1.0000  ✓

Формула: δA'[x][y] = δA[x][y] XOR δD[x]
где δD[x] = δC[x-1] XOR ROT(δC[x+1], 1)
    δC[x] = XOR_y δA[x][y]

Аналитический расчёт для δA[0][0]=1:
  δC = [1,0,0,0,0]
  δD = [0,1,0,0,2]
  Задеты 11 бит из 200 после θρπ
```

### θ→χ: граница детерминизма

```
После θρπ (до χ): E[HW дифференциала] = 11.00 / 200  (детерминировано)
После полного раунда (θρπχι): E[HW] = 21.99 / 200  (случайно)

χ — нелинейный барьер внутри каждого раунда Keccak.
Аналог барьера r=17 в SHA-256, но происходит
внутри каждого раунда, а не между раундами.
```

### Инструмент 2 для Keccak

```
Детерминированная зона (θρπ):
  δA'[x][y] = δA[x][y] XOR δD[x]  — точная формула
  Аналог T_DE_r_LINEAR для SHA-256

Случайная зона (χ):
  δ(χ) зависит от значений B — случаен
  Аналог барьера δe[17]=0 в SHA-256

Ключевое отличие от SHA-256:
  SHA-256: детерминированная зона = 16 РАУНДОВ (Wang)
  Keccak:  детерминированная зона = часть РАУНДА (θρπ)
  → Wang-цепочка для Keccak невозможна (G6 подтверждена ✓)
```
