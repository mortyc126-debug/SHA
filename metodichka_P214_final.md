# П-214: Финальный анализ — переосмысление

## Ключевые исправления

### Ранг при W0=фикс — ИСПРАВЛЕНИЕ
Ранг(δa,δe)[17] = **64/64** при любом W0, включая 0xc97624c6.
Результат П-212 (ранг=32) был артефактом малой выборки (800 пар недостаточно).

### P(δa=0 AND δe=0) — практически недостижимо
За 50M пар: δa=0 не встретилось. Нужно ~2^32 пар.
При 1M пар/с (наш C код): 4295 секунд ≈ 1.2 часа. **Достижимо на GPU за минуты.**

---

## Фундаментальный вопрос: что такое «коллизия»?

**Неправильная цель:** δstate[17]=0 при двух разных W.
Это противоречие: если δstate[r]=0, то W[r]=W'[r] для всех r
(из инъективности раундовой функции при фиксированном W).
→ Это тривиальная пара W=W'.

**Правильная цель:** M≠M', SHA-256(M)=SHA-256(M').
Это требует δstate[64]=0 при **ненулевом** ΔW[0..15].

Wang chain именно это и делает:
```
δstate[0] = 0  (оба с IV)
ΔW[0..15] ≠ 0  (разные сообщения)
δe[1..16] = 0  (почти-нулевое распространение)
δstate[64] = 0  (коллизия!)
```

## Место адаптивного трейла в атаке

Адаптивный трейл — **не полная атака**, а компонент.

Что он даёт:
- Быстрое нахождение пар с δe[2..16]=0
- Структуру δstate[17]: (δa,0,0,0,δe,0,0,0)
- HW(ΔW[17])=1 бит (теоретически интересно)

Чего не хватает:
- δe[1]=d0≠0 (барьер SHA IV)
- δstate[64]≠0 (нет коллизии)

## min HW(δstate[64]) = 91/256

## Следующий шаг П-215

**Направление:** минимизировать HW(δstate[64]) через адаптивный трейл.

Алгоритм:
1. Строим адаптивные трейлы
2. Для каждого считаем HW(δstate[64])
3. Ищем минимум через множество W[0..15]
4. Если найдём HW < 32 → это сильная аномалия
5. Если найдём HW = 0 → КОЛЛИЗИЯ!

**Оценка:** min HW(δstate[64]) при N пар ≈ 128 - 32×log2(N)/256.
За 2^32 пар: ≈ 128 - 4 = 124. Не ноль.

**Вывод:** случайный поиск не даёт коллизии.
Нужна алгебраическая структура — Wang chain.

**Реальная ценность работы:**
Адаптивный трейл — это новый инструмент анализа,
который позволяет быстро исследовать структуру SHA-256
без стоимости Wang chain (2^104).
Он даёт верхние оценки на характеристики функции.
