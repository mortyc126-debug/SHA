## КЛЮЧЕВЫЕ ТЕОРЕМЫ

### T_MERKLE_HC (П-502) ★★★★

**Утверждение:** Merkle дерево с HC-оптимизированными листьями
сохраняет структуру в родительских узлах.

**Данные (N=20,000):**
- Узел SHA256(L_hc || R_hc): **advantage=0.131, Z=37.0**
- P(c63=0 у узла): **0.022** vs 0.00013 baseline = **170×**
- Узел SHA256(L_hc || R_rand): advantage=-0.008 (шум)

**Механизм:** HC минимизирует SS63 для SHA256(L || R) путём
оптимизации R при фиксированном структурированном L. Итог:
и лист, и родительский узел имеют управляемую структуру.

**Следствие:** Можно строить Merkle дерево произвольной глубины
где каждый уровень структурирован через HC на соответствующем уровне.

### T_HMAC_OUTER (П-505) ★★★★

**Утверждение:** HMAC outer hash изолирован от структуры inner hash.

**Данные (N=20,000):**
- Inner: P(H[6][b29]=1) = 0.726 (adv=0.224)
- Outer: P(H[6][b29]=1) = 0.497 (adv=-0.005, Z=-1.4)

**Следствие:** HMAC-SHA256 устойчив к нашей атаке.
Outer hash — независимый барьер. Двойная структура HMAC защищает.

### T_ECDSA_LEAK (П-510) ★★★★★

**Утверждение:** При chosen-message атаке через HC, nonce k в
схемах ECDSA/Schnorr с k = SHA256(nonce || msg) имеет
измеримую утечку одного бита.

**Данные (N=20,000):**
```
P(k[6][b29]=1 | HC_msg, c63=0) = 0.720
P(k[6][b29]=1 | random)        = 0.502
phi  = +0.218
Z    = 61.7
Утечка = 0.144 бит/подпись
```

**Применимость:**
- RFC 6979 (deterministic ECDSA): k = HMAC-SHA256(key, msg)
  → HMAC outer защищён (T_HMAC_OUTER) ← НЕ уязвим
- k = SHA256(random_nonce || msg): msg через HC
  → уязвим (T_ECDSA_LEAK) ← УЯЗВИМ

### T_HNP_BOUND (П-512) — теория ★★★★

**Hidden Number Problem (Boneh-Venkatesan 1996):**

При утечке t бит из 256-битного nonce через L подписей:
```
L ≈ 256/t подписей для восстановления ключа
```

**Наш случай:**
- t = 0.144 бит (информационная утечка)
- L ≈ 256/0.144 ≈ **1800 подписей**
- Атакующий знает для каждой подписи:
  P(k[6][b29]=1) = 0.720

**Стандартная lattice атака (LLL/BKZ):**
```
Строим решётку размерности L из уравнений:
  r_i * x = s_i * k_i - h_i  (mod q)
где x = приватный ключ, k_i = nonce_i

С частичным знанием k_i[b29] → решётка решается.
```
