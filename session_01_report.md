## Статус ядра

Воспроизведён каскад из методички (T_CASCADE). Ключевое исправление:

- De[k] управляется **аддитивно** через dW[k-1]: `De[k] = De[k]_nat + dW[k-1]`
- Sens = 1 точно (подтверждено)
- Каскад даёт **15 нулей** De[2..16]=0 детерминированно за O(1)
- Барьер: De[17] ≠ 0 с P = 2^{-32}

---

## Направление A: Геометрия Sol_Wang

### A1 — Нейтральные биты в пространстве dW
- Нейтральных бит в dW[0..15]: **0**
- Каждый бит dW влияет на число нулей

### A2 — Линейность окрестности
- Окрестность Sol: **нелинейна**
- XOR двух нейтральных возмущений не даёт нейтральное

### A3 — Связность: расстояние между решениями
- 50 из 50 случайных W_normal дали ≥14 нулей (P=1)
- HW-расстояние между dW: min=144, avg=221, max=290
- Случайное ожидание = 256
- **Вывод:** решения равномерно распределены в пространстве — нет кластеризации

## Направление B: Алгебра переносов ← ГЛАВНЫЕ НАХОДКИ

### B1 — Профиль carry-веса в T1
- Средний carry-вес: **59.55 бит/раунд** (из макс. 4×32=128)
- Самые "тихие" раунды (мало carries): 59, 47, 58, 26, 19
- Самые "шумные": 62, 27, 60, 3, 55
- Разброс: 53.8 — 66.6 бит

### B2 — Реальный SHA vs XOR-только модель ⚠️ АНОМАЛИЯ

| Раунд | Real E[HW(De)] | XOR E[HW(De)] | Разница |
|-------|----------------|---------------|---------|
| 1     | 1.000          | 2.000         | -1.000  |
| 2     | 15.074         | 7.401         | **+7.673** |
| 3     | 15.562         | 8.421         | **+7.141** |
| 4     | 16.044         | 13.227        | **+2.817** |
| 5     | 15.918         | 10.940        | **+4.978** |
| 6+    | ≈16.0          | ≈16.0         | ~0      |

**Интерпретация:** Carry создаёт БОЛЕЕ БЫСТРУЮ диффузию в раундах 2-5.
XOR-модель требует 6 раундов для насыщения, реальный SHA — 2.
Это **защитная роль carry**: он ускоряет лавинный эффект.

### B3 — Мёртвые carry-позиции
- 64 бита из 2048 никогда не генерируют перенос = **3.1%**
- По одному биту на каждый раунд (структурно связано с K[r])

### B4 — Carry вдоль каскадных пар (НОВЫЙ ЭКСПЕРИМЕНТ)
- Carry-вес в De=0 зоне (раунды 2-16): **60.0 ± 0.7** бит
- Carry-вес на барьере (раунд 17): **59.64** бит
- Carry-вес после барьера: **59.77** бит
- **Вывод:** Carry равномерен. Барьер r=17 — НЕ carry-феномен.

## Направление C: Расписание как объект

### C1 — Базовая статистика
- E[HW(ΔW[16..63])] = 16.000 (XOR) и 16.000 (ADD)
- Расписание не создаёт структуры в случайном ΔW

### C2 — Каскадное расписание
- dW[0]=1, dW[4]=0xfffffffe всегда — **HW=31 инвариант**
- dW[2], dW[3] — два паттерна: {0, 0xfffffffe, 0xffffffff}
- E[HW(ΔW[16..63])] для каскадных пар = **16.026** (почти случайное)
- Расписание каскадных пар выглядит случайным в раундах 16+

### C3 — Нулевые слова
- P(ΔW[r]=0) = 0 для r=16..39
- Нет нулевых слов в расписании каскадных пар

## ⭐ НОВАЯ ТЕОРЕМА: T_DE5_DETERMINISTIC

**Формулировка:**
```
При d0=1 и условии De[2]=De[3]=De[4]=0:
  De[5]_nat = 2  (константа, не зависит от W_normal)
```

**Верификация:** 10000/10000 = P=1.000000

**Следствие:**
```
dW[4] = -2 mod 2^32 = 0xfffffffe  (абсолютный инвариант)
HW(dW[4]) = 31  ВСЕГДА при d0=1
```

**Зависимость от d0:**
```
d0 = 1           → dW[4] = 0xfffffffe  (HW=31)
d0 = 2           → dW[4] = 0xfffffffc  (HW=30)
d0 = 4           → dW[4] = 0xfffffff8  (HW=29)
d0 = 0x10000     → dW[4] = 0xfffe0000  (HW=15)
d0 = 0x80000000  → dW[4] = 0x00000000  (HW=0)
```

**Формула:**
```
dW[4] = -2*d0  mod 2^32
```

**Интерпретация:** раунд 5 видит одинаковую "сигнатуру" независимо от W,
потому что к раунду 5 состояние полностью определяется d0 и IV через
детерминированный carry-путь. W[3] (=dW[3]) влияет только начиная с De[4].

## Синтез: что значат эти результаты

### 1. Carry — защитник, не враг
Carry ускоряет диффузию в 2 раза быстрее XOR-модели (раунды 2-5).
Атака на "устранение carry" сделала бы SHA слабее, не сильнее.

### 2. Инвариант dW[4] = -2*d0
Это новая аналитическая связь. При известном d0 мы знаем dW[4] точно.
Открытый вопрос: есть ли аналогичные детерминированные dW[k] для других k?

### 3. Расписание непробиваемо для случайных dW
Каскадный dW выглядит случайным в раундах 16+.
Барьер расписания (разрыв 30 vs 86) подтверждён экспериментально.

### 4. Sol_Wang — равномерно распределено
Решения не кластеризованы. Нет особых "тихих" регионов.

1. **T_DE5_GENERALIZATION:** Существуют ли другие k где De[k]_nat детерминировано?
   Проверить: De[8]_nat, De[11]_nat (Period-3 из методички)

2. **Carry vs диффузия:** Можно ли найти d0 такое что carry-диффузия в раундах 2-5
   минимальна? Это дало бы "более слабый" дифференциал.

3. **dW структура:** dW[2], dW[3] имеют 2 паттерна (0 или ~FFFFFFFF).
   Что определяет выбор? Это функция W[0], W[1]?

4. **Multi-block:** Применить каскад к 2-блочной конструкции.
   T_MULTIBLOCK из методички: min HW=36 за 4 блока — воспроизвести.
