## Корректность: почему каскад работает

По **T_DEP** (П-9): `De_r` зависит от `W[j]` тогда и только тогда, когда `j ≤ r-2`.

Следствие **T_DEP_ZERO**: `De_{3k}` не зависит от `W[3k-1]`.

Поэтому:
- `W[0]` влияет на `De3` ✓, не влияет на `De6` при фиксированных `W[1..5]` ✗
- `W[4]` влияет на `De6` ✓, не влияет на `De3` (которое зависит от `W[0..1]`) ✓
- `W[7]` влияет на `De9` ✓, не влияет на `De3, De6` ✓
- и т.д.

Каждый новый параметр независим от предыдущих нулей — это и есть каскад.

## SC_17: почему не сработало

`SC_17: De13 + DW15_schedule = 0`

В П-72: `DW15_schedule = 0` (мы не меняли `W[15]`), но `De13 = 302296C5 ≠ 0`.

Чтобы выполнить SC_17, нужен дополнительный **шаг 6**:
```
Шаг 6: При DW[0..13] фиксированы, варьируем W[15] такой что:
        De15 по-прежнему = 0  И  De13 + DW15_schedule = 0
        Это два условия одновременно → P ≈ 1/2^64 → нереалистично без структуры.
```

Это новый барьер. SC_17 совместимо с 5 нулями только если можно
подобрать W[15] независимо — но De15 зависит от W[13] и W[14],
а De13 от W[0..11]. SC_17 требует совместного решения, не каскадного.
