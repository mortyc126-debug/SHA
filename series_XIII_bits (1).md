## 1. Методология: что значит "препарировать бит"

Каждый бит W[w][b] — это агент с характером. У него есть траектория через 64 раунда, несколько несущих его нелинейных узла, carry-история, и уникальный выходной отпечаток. Мы изучаем бит по девяти измерениям: schedule DNA, round signature, nonlinear profile, carry birth, carry chain, output bias, anti-bits, noise topology, leverage score.

Главный инструмент наблюдения — дифференциальный прогон: два входа W и W⊕(1<<b), полная запись всех 8 регистров × 64 раунда с побитным разбором carry-событий.

---

## 2. Schedule DNA — след бита в расписании

Расписание SHA-256 линейно над GF(2) в части XOR-ротаций. Бит W[w][b] через функции s0 и s1 оставляет аналитически вычислимый след в словах W[16..63].

Формально: pres[i] = XOR-маска бит в W[i], унаследованных от W[w][b]. Вычисляется рекурсивно через ротации s1 (17,19,>>10) и s0 (7,18,>>3).

Для W[0][b31]: бит присутствует в 44 из 48 расписанных слов. Первый наследник — W[16][b31] (прямо). Максимальный взрыв в W[40]: HW=24 из 32 бит. Это значит, что через XOR-ротации один входной бит к раунду 40 рассеян почти по всему слову расписания.

Для W[1][b27]: присутствует во всех 48 словах W[16..63]. Максимум в W[48]: HW=22.

Важно: это только XOR-часть. Сложение mod 2³² в расписании добавляет carry-нелинейность, которая не учитывается в schedule DNA. Schedule DNA — нижняя граница диффузии.

## 3. Round Signature — биография бита по 64 раундам

Для каждого раунда r фиксируем HW(δreg[r]) для всех восьми регистров a,b,c,d,e,f,g,h.

Закономерность универсальна: первый ненулевой раунд = w (номер слова бита). Лавина достигает HW≥16 на раунде w+1. Полное насыщение — на w+3.

Регистр d особый: δd[r] = δe[r-1] всегда (тождество через сдвиговый конвейер). Поэтому визуал δd — это просто δe со сдвигом на один раунд.

Carry birth — раунд первого изменения переноса в T1+T2 — всегда равен номеру слова w. Исключение: для W[0] carry рождается на раунде 0 мгновенно.

---

## 4. Carry-датчик S2 — три типа carry

Carry_scope разделяет события на три типа:

Carry T1 — перенос из суммы h+S1(e)+Ch(e,f,g)+K[r]+W[r]. Это основной канал.

Carry T2 — перенос из суммы S0(a)+Maj(a,b,c). Для однобитных флипов: P(T2_carry=1 | T1_carry=1) = 0.000 абсолютно для всех 64 раундов. Это алгебраический инвариант, не статистика.

Carry MERGE — carry_sum=1 при carry_T1=0 И carry_T2=0. Это происходит когда T1_low + T2_low переполняется совместно, хотя каждое слагаемое по отдельности не вызывает переноса. На раунде 0 для бита 27: P(merge)=1.00. На раунде 1: 0.76. Затем постепенно снижается к 0.33-0.35.

phi_T1_T2 = −1.000 для всех 64 раундов. T1 и T2 carry никогда не происходят одновременно при однобитном флипе. Это означает: весь carry_sum порождается исключительно через T1.

---

## 5. Carry-антицепочка — главная структурная аномалия

Сканирование всех 512 бит выявило класс с phi(carry[r+1] | carry[r]) < −0.5:

Биты b27 всех слов W[0..15]: phi от −0.88 до −0.94.
Биты b26 всех слов: phi от −0.70 до −0.94.
Бит W[9][b26] — рекордсмен: phi = −0.9375.

Интерпретация: если carry изменился на раунде r из-за бита 27, то с вероятностью 0.88-0.96 на раунде r+1 carry НЕ изменится. Это антицепочка — детерминированное гашение carry.

Физика: bit b27 через S0(a)=Rot(a,2)^Rot(a,13)^Rot(a,22) переходит в позицию b29 (через Rot2). Это старший бит T2 следующего раунда. Высокое значение T2 подавляет переполнение в T1+T2 следующего раунда.

Формально: когда carry[r]=1 из-за b27, значение a_new принимает либо очень большое значение (топ-50%), либо очень малое (нижние 50%) — без промежутков. Через S0.Rot2 эти граничные значения дают high T2 в раунде r+1, что подавляет overflow.

100% бит класифицированы как CARRY-MEMORY — ни один не random. Но внутри класса градация: b27 и b26 на порядок аномальнее остальных.

## 6. Sensitivity Map — сенсорные биты регистра a

На раунде 1 (первом после рождения W[0][b27]) только пять бит регистра a имеют ненулевую корреляцию с carry-событием:

b26: P(a[1][b26]=1 | carry=1) = 0.672 против базовых 0.491. Delta +0.182. Усиливает carry.

b29: delta +0.122. Усиливает.

b30, b31: delta +0.080 и +0.054. Слабые усилители.

b28: delta −0.114. Подавляет carry.

Это и есть "молекулярный уровень": когда бит 27 проходит через нелинейность на раунде 1, он встречает конкретное значение регистра a, и биты b26-b29 этого регистра определяют результат. b26 усиливает, b28 подавляет — два соседних бита действуют противоположно. Это деструктивная интерференция, порождающая антицепочку.

---

## 7. Weight Transfer Matrix — куда идёт каждый бит

S0 и S1 линейны над GF(2). Каждый бит j расщепляется на три выходные позиции:

S0(a): бит j → (j+2)%32, (j+13)%32, (j+22)%32
S1(e): бит j → (j+6)%32, (j+11)%32, (j+25)%32

Ch(e,f,g) и Maj(a,b,c) имеют диагональную матрицу влияния: бит j влияет только на бит j, P≈0.50. Нулевое кросс-смешивание между позициями. Всё позиционное перемешивание — через S0 и S1.

---

## 8. Phase Portrait — осцилляция дифференциала

Матрица переходов HW[r] → HW[r+1] показывает структурные потоки:

Из зоны HW=1-8: переход в HW=9-15 с P=0.695 (доминирующий). Это взрыв лавины — из "малой" зоны почти детерминированно в "среднюю".

Из зоны HW=9-15: в HW=16-22 с P=0.560.

Из зоны HW=16-22: в HW=9-15 с P=0.427. Обратный переход.

Осциллятор качается между зонами 9-15 и 16-22 с вероятностями 0.43/0.56. Это не случайное блуждание — направленный поток с несимметричными переходами.

## 9. Предикторная модель — b26 главный регулятор

Experiment C показал: delta = P(высокий δHW | бит=1) − P(высокий δHW | бит=0).

b26: delta = +0.202. Когда a[1][b26]=1, δHW(a[2]) высокий с P=0.568 против 0.366. Это 20.2 процентных пункта — сильнейший одиночный предиктор перемешивания.

b28: delta = −0.062. Слабый подавитель.

Комбинированный предиктор (a[1][b28] AND a[1][b29] → HIGH): точность 51.50% против базовых 50%. Улучшение всего +1.5% — b26 важнее пары.

## 10. Тёмный вклад — carry_merge как третий член

Анатомия: δ(T1+T2) = XOR(δT1, δT2) + dark_contrib.

Тёмный вклад = то что carry добавляет сверх XOR-модели. На раунде 0: +0.151 — MERGE-событие. На раундах 1+: ±0.05 (статистический шум). Раунд 32: −0.94 (единственный раунд с отрицательным тёмным вкладом — деструктивная интерференция schedule-взаимодействия).

На раунде 0 тёмный вклад значим потому что T2 фиксирован (определяется только IV), а T1_low+T2_low создаёт merge с P≈1/32.

## 11. Контрольная поверхность — первый захват управления

Experiment B показал: группировка W[0] по a[1][b31..b29] даёт диапазон E[δHW(a[2])] от 12.94 до 14.28. Группа "111" аномальна: P(carry[1])=0.469 (высокий carry) при E[δHW]=12.94 (низкий дифференциал). Это управляемый carry — carry происходит, но его эффект подавлен сенсорной зоной b28.

Физически: группа W[0] с a[1][b31..b29]=111 всегда приводит a[1] в зону где b28=1 (подавитель carry). Carry происходит из T1, но T2 следующего раунда блокирует его распространение. Это первый продемонстрированный механизм частичного контроля лавины.

## 12. Leverage Score — ранжирование битов

Рейтинг по совокупному рычагу (0-100):

W[0..11][bX]: leverage=100 (максимум). Все биты первых 12 слов имеют полный рычаг — ранний вход, широкое расписание, максимальная лавина.

W[12..13][b27]: leverage≈80.

W[15][b27]: leverage≈71.

b27 и b26 любого слова — единственные позиции с carry-chain |phi|>0.7. Средний |phi| для b27 по всем 16 словам: 0.81. Для b26: 0.70. Для b28: 0.40-0.45. Для b0..b25,b29..b31: 0.34-0.38.

## 13. Puppet Engine — ниточки управления

Два типа ниточек установлены:

Carry-антицепочка: flip W[w][27] → P(carry[w]=1)=1/32 → с P=0.88 carry[w+1]=0. Двухраундовое окно предсказуемости создаётся одним битом.

Context filter: ищем W[0] с a[1][b28]=0 → P≈1/8 → снижение среднего δHW на 0.5-0.8 бита. Слабый, но воспроизводимый эффект.

Комбо-флип k bit27 одновременно: phi_anti усиливается до k=3, затем нейтрализуется (k=6). Оптимальное комбо: 1-3 бита 27.

Важно: оба эффекта работают на раундах w..w+3. За пределами фазы взрыва (τ=1.8 раунда) информация стирается лавиной.

## 14. Noise Probe — природа шума

Шум определён как: δ(реальный HW) − δ(линейная XOR-модель HW).

На раунде 0: E[noise|carry=1]=4.0, E[noise|carry=0]=0.83. Разница 3.2 — carry РАЗДЕЛЯЕТ шум. Это структурный шум, не случайный.

На раундах 1+: разница шума по carry-условию < 0.2 бита. Шум становится случайным.

Шумовые аттракторы: среди 1000 случайных W[0] диапазон суммарного шума (раунды 0-4) от 2.3 до 8.7 — соотношение 3.8×. Конкретные значения W[0] минимизируют шум первых раундов. Это навигационные точки: двигаясь к аттрактору шума, снижаем случайность carry в первых раундах.

Carry-переходы независимы при lag≥2 (нет carry-памяти через два раунда), НО на lag=1 обнаружена структурная антицепочка для b27/b28.

## 15. Avalanche DNA — уникальность отпечатка лавины

L2-расстояния между профилями δa[1..10]:

W[0][b27] и W[0][b0]: L2=1.14 — похожи (одно слово).
W[0][b31] и W[0][b27]: L2=3.29 — различаются (разные позиции в одном слове).
W[0][bX] и W[7][bY]: L2≈40 — кардинально разные (разные слова).

ДНК лавины определяется номером слова, а не позицией бита. Это следствие закона задержки: каждое слово W[k] начинает свою лавину ровно на раунде k.

## 16. Теоремы по битам

**T_CARRY_NULL:** Корреляции carry между конкретными раундами (phi>0.05) существуют только в условной популяции Sol_Wang. Для случайных W все phi<0.003.

**T_DE2_CLASS_UNIFORM:** De2-класс (8 значений по hw=4) не предсказывает carry_sync или hw63. Разница незначима.

**T_BIT27_ANTICHAIN:** Любой W[w][b27] и W[w][b26] создаёт антицепочку P(carry[w+1]=0 | carry[w]=1)≈0.88-0.96. Физика: b27 через S0.Rot2 → b29 (старший бит T2) → подавление overflow.

**T_T2_CARRY_ZERO:** phi_T1_T2 = −1.000 для всех 64 раундов при однобитном флипе. T1 и T2 carry физически не могут произойти одновременно. Весь carry_sum порождается через T1.

**T_MERGE_PERSISTENT:** MERGE-carry (carry_sum без T1 и T2) сохраняется во всех 64 раундах с P=0.33-1.00. Третий тип carry существует и не исчезает.

**T_B26_PREDICTOR:** a[1][b26] — главный регулятор перемешивания раунда 2. Delta=+0.202. Через S0.Rot2 бит 24 входа попадает в b26 выхода — именно эта позиция наиболее чувствительна к carry в следующем раунде.

**T_DUAL_LOCK_WANG:** Условие двойного нелинейного замка (δa=0 AND δe=0): δ(S0(a[r])+Maj(a[r],b[r],c[r])) ≡ −δa[r−3] (mod 2³²). Wang SC задаёт δa[r]≈0 для r=0..17, что автоматически удовлетворяет двойному замку на r+3.

## 17. Прогресс по битовым инструментам

Все инструменты принимают параметры [word] [pos] [N] и компилируются:
```bash
gcc -O3 -march=native -lm -o TOOL TOOL.c
```

round_cam: режимы 1 (детальный трек), 2 (статистика N сэмплов), 3 (глобальный скан). Выдаёт таблицу всех 9 датчиков по 64 раундам.

carry_scope: осциллограф carry-событий. Режимы: одиночный бит или сравнение всех 32 позиций. Находит MERGE-события, phi_chain, phi_T1_T2.

noise_probe: зонд шума. Вычисляет E[noise|carry] vs E[noise|~carry] для каждого раунда. Находит шумовые аттракторы.

puppet_ctrl: система управления. Режимы: предикция δe через carry-антицепочку, комбо-флип k-бит27, навигация к целевому HW(δe[r]).

weight_mixer: молекулярный уровень. Sensitivity map, interference spectrum, weight transfer matrix, phase portrait, resonance detector.

bit_control: захват управления. Context classifier, control surface, predictive model, mixing anatomy, control loop.

bit_surgeon: полный скан 512 бит. Выдаёт карту рычагов, топ carry-chain аномалий, puppet strings, битовые классы по noise_class.
