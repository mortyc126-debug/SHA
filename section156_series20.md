### 156.1 ФРОНТ 1 — GPU Wang Implementation

#### 156.1.1 Измеренная скорость (CPU baseline)

```
Single-thread (C, -O3, -march=native):  ~3.1M SHA-pairs/sec (bug in reporting)
                реальная:               ~1.6M SHA-pairs/sec
8-thread CPU:                           ~13M pairs/sec
2^32 estimate (8T CPU):                 ~328 sec
```

#### 156.1.2 T_GPU_WANG_PROJECTION ★★★★

```
RTX 4090 (SHA-256 throughput ~10 GH/s):
  Wang pairs: 5000M pairs/sec
  2^32 birthday: 0.86 sec
  2^30.4 (с оптимизациями): 0.28 sec
  
GPU speedup vs CPU: ~3000×
```

**Архитектура GPU-ready кода:**
- Ядро: `sha_diff(W, W2) → hw` — портируется на CUDA trivially
- Birthday loop: stride по W[2], каждый поток независим
- Нет shared memory, нет синхронизации — идеально для GPU

**Нужно для реализации:**
1. CUDA kernel sha256_pair_diff (64 SHA rounds × 2, 80 регистров)  
2. Host code: Wang cascade + launch kernel
3. CUDA SHA-256 реализован публично → взять готовый

#### 156.1.3 Практический итог Фронта 1

На реальном железе (RTX 4090 + правильный код):
```
C_practical ≈ 2^30.4 SHA ≈ 0.28 секунды
```

### 156.2 ФРОНТ 2 — T_MULTILEVEL_BIRTHDAY

#### 156.2.1 T_MLB_EXACT ★★★★★ — НОВАЯ ТЕОРЕМА

```
E[hw(SHA(W1)⊕SHA(W2)) | SHA(W1)[0..k-1] = SHA(W2)[0..k-1]]
= (256 - 32k) / 2
= 128 - 16k
```

**Proof:** первые k слов совпадают → оставшиеся 256-32k бит независимы равномерны. □

**Экспериментальная верификация:**
```
k=1 (32 бит): теория = 112.0, измерено = 111.92 ✓
```

#### 156.2.2 T_MLB_OPTIMAL_K ★★★★

```
Оптимальный k для минимизации C_total = max(C₁, C₂):

C₁(k) = 2^{16k}        (birthday для H[k]-collision)
C₂(k) = 2^{(128-16k)/2} (birthday после H[k]-collision)

Оптимум: 16k = (128-16k)/2 → k = 8/3 ≈ 2.67 → k=3.

k=3: C_total = 2^48
```

| k | C₁ | E[hw] | C₂ | C_total |
|---|----|----|----|----|
| 1 | 2^16 | 112 | 2^56 | 2^56 |
| 2 | 2^32 | 96 | 2^48 | 2^48 |
| 3 | 2^48 | 80 | 2^40 | **2^48** |
| 4 | 2^64 | 64 | 2^32 | 2^64 |

**Главный результат:**
```
C_MLB(standalone) = 2^48  (улучшение baseline 2^128 на 80 бит)
C_Wang = 2^32              (лучше MLB)
C_Wang + MLB = 2^32        (MLB не улучшает Wang)
```

**MLB полезен для:** full SHA атак БЕЗ Wang SC → снижает 2^128 до 2^48.

### 156.3 ФРОНТ 3 — Альтернативные SC

#### 156.3.1 T_FIRST_STATE_FORCED ★★★★★

```
При DW = (ΔW₀, 0, ..., 0), δstate[1] полностью детерминирован:
  δa[1] = ΔW₀  (точно)
  δe[1] = ΔW₀  (точно, = T_IV_BIT0)
```

**Экспериментальная верификация:**
```
Уникальных значений δa[1] за 200 случайных W = 1 → ФИКСИРОВАН ✓
Уникальных значений δe[1] за 200 случайных W = 1 → ФИКСИРОВАН ✓
δa[1] = δe[1] = 0x80000000 (при DW[0]=0x80000000)
```

#### 156.3.2 T_ALT_SC_INFEASIBLE ★★★★

Альтернативные SC (δa=0, δh=0) не поддаются детерминистическому cascade:
- δa[1] = 0x80000000 фиксировано → нельзя обнулить без другого DW
- δh[r] = δe[r-2] → δh=0 эквивалентно δe=0 с задержкой (уже в Wang!)

**Следствие:** Wang SC (δe=0) — единственный эффективный детерминистический cascade для SHA-256 с DW[0]-style дифференциалом.

#### 156.3.3 T_MIDSTATE_ENTROPY ★★★

```
E[hw(δstate[32]) | Wang DW, случайный W] = 128.03 ± 8.02
```

δstate[32] полностью случаен → нет shortcut через MITM на середине.
