## EXP B: Аналитика — что именно защищает S1

```
DP(MD5-F,  δc=1, XOR)     = 0.500328  теор=0.500
DP(MD5-F,  δb=1, XOR)     = 0.500020  теор=0.500
DP(Ch alone, δe=1, XOR)   = 0.499351  теор=0.500  ← тоже 0.5!
DP(Ch+S1,  δe=1, XOR)     = 0.00000000  ← S1 защищает δe
DP(Ch+S1,  δe=1, ADD)     = 0.00000000

DP(ChS1, δf=1, XOR)       = 0.500007  ← f не защищена!
DP(ChS1, δg=1, XOR)       = 0.499524  ← g не защищена!
```

**Ключевое открытие:**
- Ch и MD5-F одинаково уязвимы по δe (DP=0.5) без S1
- S1 полностью защищает только первый аргумент e (DP→0)
- f и g остаются уязвимы (DP=0.5)

## EXP C: S1 — количество вращений

```
Предмиксер перед Ch (XOR δe=1):

Нет (Ch alone):         DP = 0.5000  = 2^-1
ROTR(6) only:           DP = 0.5002  = 2^-1  ← одно не помогает!
ROTR(6)^ROTR(11):       DP = 0.2504  = 2^-2
S1=ROTR(6,11,25) SHA-256: DP = 0.1245  = 2^-3  ← SHA-256 уровень
S0=ROTR(2,13,22):       DP = 0.1246  = 2^-3
```

**Важно:** S1 снижает DP с 0.5 до 0.124 ≈ 2^-3, не до 2^-32.
Но для XOR δe=1 (первый аргумент): DP=0 точно.

## EXP B2: Propagation — реальная защита в цепочке

```
В SHA-256: f_new = e_old, g_new = f_old
После раунда 1: δe[1] случаен (через S1)
→ δf[2] = δe[1] — тоже случаен
→ DP(Ch+S1, δf[prop]) = DP(ChS1, δf=S1(e^1)-S1(e)) = 0.125

После раунда 2: δe[2] случаен → δf[3] случаен
→ DP уменьшается экспоненциально

В MD5: функция обновления без S1-аналога
→ δb[r] сохраняет структуру дольше
→ Wang строит sufficient conditions используя это
```

## G8 — уточнённая формулировка

### G8v3 (финальная):

```
Уязвимость к Wang определяется DP_max_eff —
эффективным DP с учётом цепочки регистров.

DP_max_eff(SHA-256) = DP через propagation ≈ 2^-3 → 2^-6 → 0
  (убывает экспоненциально за счёт S1 в цепочке)

DP_max_eff(MD5) = 0.5 стабильно
  (нет S1-аналога → структура δ сохраняется)

G8v3: DP_max_eff > 2^-(w/2) через K раундов → уязвим
      K = число раундов для насыщения через S1-цепочку
```

### Почему H(XOR) в MD5 не помогает:

```
H = b^c^d линейна → DP(H, δb) = 0 (δH = δb ≠ 0 всегда)
Но MD5 использует H только в раундах 33-48.
Раунды 1-16 (F) и 17-32 (G) имеют DP=0.5.
Wang атакует именно F-раунды (1-16) → H не защищает.
```

## ГЛАВНЫЙ ВЫВОД

```
SHA-256 устойчив к Wang не потому что Ch "лучше" MD5-F.
Обе функции имеют DP=0.5 в изоляции.

Разница в ЦЕПОЧКЕ:
  SHA-256: e → S1(e) → Ch(S1(e), f, g) → new_e
           Цепочка с S1 разрушает структуру δ экспоненциально
  MD5:     b → F(b, c, d) → new_b
           Нет S1-аналога → δ сохраняется → Wang строит условия

S1 = ROTR(6)^ROTR(11)^ROTR(25) — необходимый элемент.
Одно вращение: недостаточно (DP=0.5).
Два вращения: DP=2^-2.
Три вращения (SHA-256): DP=2^-3, с propagation → 0.
```
