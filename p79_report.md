# П-79: SPARSE-CARRY — синтез и честный результат (Март 2026)

## Что тестировали

Попытка соединить три открытия П-77/П-78 в один алгоритм:
- T_BIT_LINEAR_R1 (раунд 1 линеен)
- T_SCHEDULE_SPARSE (37% активных бит в расписании)
- T_CARRY_INDEPENDENCE (carry-free независимы)

---

## Главный результат: разрыв бит/слово

```
ТЕОРЕМА T_WORD_SATURATION (П-79):

T_SCHEDULE_SPARSE верна на уровне БИТА:
  каждый входной бит W[j][b] активирует ~37% выходных бит
  расширения W_expanded[16..63][b'].

Но на уровне СЛОВА (32-битного):
  P(слово W_expanded[k] активно) ≈ 100%.
  При 37% активных бит, распределённых по 48 словам:
  каждое слово содержит ~12 активных бит → почти всегда ≠ 0.

Измерено: при DW[0]=любой, 93.5-94% слов расширения активны.
GF2 предсказание для бит: 37%.
Наблюдение для слов: ~94%.
```

**Следствие:** T_SCHEDULE_SPARSE не даёт практического преимущества
при работе с дифференциалами на уровне слов (Wang-style, CASCADE).

## Что остаётся реальным

### T_BIT_LINEAR_R1 (подтверждена строго)
```
e_1[b] = W[0][b] для всех b=0..31 с P=1.0.
```
Это бит-уровневое свойство. Реально и точно.

### Медленные пути (P ≈ 0.0002)
```
W[r-2][k] → e_r[k-2]: P ≈ 0.0002 (в 2500× медленнее P=0.5)
```
Паттерн повторяется на каждом раунде. Бит-уровневое свойство. Реально.

### T_CARRY_INDEPENDENCE
```
Carry-free состояния между раундами НЕЗАВИСИМЫ.
Carry Skeleton требует конструктивного подхода.
```

## Честный вывод о трёх направлениях

```
Направление 2 (Carry Skeleton):
  Carry-free DW не даёт больше нулей De_r чем случайный DW.
  Статистически: avg нулей = 0.000 при N=10000.
  Причина: T_CARRY_INDEPENDENCE — carry-free e_1 не помогает e_2.
  
  ✗ Алгоритм SPARSE-CARRY не работает как задумано.

Направление 3 (Schedule sparse):
  63% нулей — реально, но только на бит-уровне.
  На слово-уровне: 94% активных слов при любом DW.
  
  ✗ Практического преимущества для Wang-style атак нет.

Направление 5 (Bit matrix):
  T_BIT_LINEAR_R1: e_1[b] = W[0][b] — строго верно.
  Медленные пути (P=0.0002): структурный паттерн.
  
  ✓ Реальные бит-уровневые свойства.
```

## Где реальная новизна

**Разрыв бит/слово** — это и есть главный барьер SHA-256.

XOR-дифференциалы (Wang) работают на уровне бит в GF(2):
расписание разреженное, 37% активных бит.

Аддитивные дифференциалы (наш CASCADE) работают на уровне слов:
расписание полное, 94% активных слов.

**Смешанный дифференциал** (Пробел 3 из П-74) — это именно:
работать с отдельными битами (используя 37% разреженность),
но в аддитивном Z/2^32 пространстве (используя CASCADE структуру).

Конкретно: можно ли задать DW такое что DE_r[конкретный_бит] = 0
для многих r одновременно, используя медленные пути матрицы D?

Это П-80.

## Правило 11 для §0 методологии

```
Правило 11: Различать бит-уровневые и слово-уровневые свойства.
Свойство верное на уровне битов может давать P≈1 на уровне слов
из-за вероятностного насыщения (T_WORD_SATURATION).
Перед практическим применением — проверить на уровне слов.
```
