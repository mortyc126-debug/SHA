## АЛГОРИТМ DISTINGUISHER v4.1

```
Вход:  оракул O(·) — SHA-256 или случайная функция
Выход: "SHA-256" или "random"

1. W[0] ← Ensemble_HC_K64(200 шагов, снайперская функция)
   (64 частицы, выбираем ту что ближе к 2^32 снизу)

2. H ← O(W[0] || 0x00...00)
   (один SHA-256 блок, W[1..15]=0)

3. Вычислить score:
   
   lin = +0.222×(2×H[6][b29]-1)  ← b29=1 смещён
       + 0.212×(1-2×H[6][b31])   ← b31=0 смещён
       + 0.140×(2×H[7][b30]-1)
       + 0.137×(2×H[7][b29]-1)
       + 0.119×(1-2×H[6][b30])
       + 0.118×(1-2×H[6][b28])
       + 0.051×(1-2×H[6][b27])
       + 0.048×(1-2×H[7][b28])
       + 0.038×(1-2×H[6][b26])
       + 0.034×(1-2×H[7][b31])

   pair = +0.496 если H[6][b30]=0 И H[6][b29]=1  (иначе -0.496)
        + 0.460 если H[7][b31]=0 И H[7][b30]=1  (иначе -0.460)
        + 0.367 если H[7][b30]=1 И H[7][b29]=1  (иначе -0.367)
        + 0.284 если H[6][b31]=0 И H[6][b29]=1  (иначе -0.284)

   score = lin + pair

4. if score > -0.246: return "SHA-256"
   else:              return "random"
```

## СТРУКТУРА CARRY-СИГНАЛА В SHA-256

```
После 64 раундов с carry[63]=0:

Раунды от carry:
  H[7] = h[63] + IV[7]      ← ПРЯМО зависит от carry[63]
  H[6] = g[63] + IV[6]      ← через 1 раунд
  H[5] = f[63] + IV[5]      ← через 2 раунда (сигнал исчезает)

Паттерны (биты b31..b28):
  H[7]: (−,+,+,−) = (−0.03, +0.14, +0.14, −0.05)
  H[6]: (−,−,+,−) = (−0.21, −0.12, +0.22, −0.12)
  H[5]: (0, 0, 0, 0) — нет сигнала

Нелинейные корреляции:
  H[6][b30]×H[6][b29]: +0.496 (совместное смещение)
  H[7][b31]×H[7][b30]: +0.460
  H[7][b30]×H[7][b29]: +0.367
  H[6][b31]×H[6][b29]: +0.284

Физика: carry[63]=0 → h[63] мало
         → g[62] = h[63] мало
         → H[7] = g[62]+IV[7] в нижней половине диапазона
         → H[7][b31]=0 (−), H[7][b30]=1 (+) — ожидаемо
         → Через f[62]→H[6]: другой паттерн (−,−,+,−)
         → Биты внутри слова коррелированы через переносы
```
