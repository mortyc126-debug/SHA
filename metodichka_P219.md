# П-219: Многоблочный адаптивный трейл — новый класс

## Главный результат

**4-блочный адаптивный трейл (64 раунда) даёт:**

| Метод | min HW(δstate[64]) | N пар |
|-------|-------------------|-------|
| 1-блочный (r=0..15 адапт.) | 93 | 500K |
| 4-блочный (r=0..63 адапт.) | **36** | 5M |

**Снижение в 2.6× по сравнению с 1-блочным!**
vs ожидаемое random 128.

---

## Структура многоблочного трейла

```
Блок k (раунды 16(k-1)..16k-1):
  δe[2..16]=0  при k=1
  δe[16k+2..16k+16]=0  при k>1  (автоматически!)
  
δstate[16k] = (δa, δb, δc, δd, 0, 0, 0, 0) для ВСЕХ k
  → δe=δf=δg=δh=0 на каждой границе блока
  → следующий блок адаптируется автоматически!
```

### Число нулевых δe
```
1 блок (16 раундов):  14/16 нулей (87%)
4 блока (64 раунда):  60/64 нулей (94%)
N блоков:            (15N-1)/16N нулей → 93.75%
```

### min HW(ΔW[16k]) для k=1..8 (100K трейлов)
```
ΔW[ 16]: min HW = 4
ΔW[ 32]: min HW = 5
ΔW[ 48]: min HW = 4
ΔW[ 64]: min HW = 3  ← лучший
ΔW[ 80]: min HW = 4
...все одинаковы: 3..5
```
Нет деградации по числу блоков — паттерн стабилен!

## T_MULTIBLOCK: новая теорема

**Теорема (T_MULTIBLOCK):** Для любого числа блоков N,
адаптивный трейл на N×16 раундах автоматически строится
с δe=0 в (15N−1) из (16N) раундов.

**Доказательство:** На каждой границе блока 16k:
δstate[16k]=(δa,δb,δc,δd,0,0,0,0), т.е. δe=δf=δg=δh=0.
Это те же условия что и на старте (SHA IV, δe=0).
→ адаптация следующего блока работает идентично блоку 1. □

## Скорость и масштабируемость

```
5M трейлов за 6 секунд = 833K трейлов/с  (64 раунда каждый)
min HW(δstate[64]) = 36 за 5M трейлов

Для HW=0 (коллизия):
  Если min убывает линейно с log(N):
  36 при 5M → 0 при 5M × 2^36 ≈ 2^58 трейлов
  При 833K/с: 2^58 / 833K ≈ 2^37 секунд → нереально

  НО: 4-блочный трейл — бесконечно масштабируем!
  N-блочный при N→∞: все δe=0, трейл полностью нулевой?
  Нет: граница каждого блока имеет ненулевое δstate=(δa,δb,δc,δd,0,0,0,0)
```

## Следующий шаг П-220

**Вопрос:** есть ли взаимодействие между блоками?
При N→∞: δstate[16N] = функция от δa цепочки.
Может ли цепочка δa[r] стать периодической или нулевой?

**Проверка:** измерить E[HW(δstate[16N])] и min HW для N=1..20.
Если min убывает быстрее √(log N) → есть структура!

**Также:** birthday на (δstate[32]=δstate[32]') даёт
два трейла с одинаковым началом с раунда 32 → совпадающий δstate[64].
Это O(2^64) — но при фиксированном W0 может быть меньше.
