# ТЕОРИЯ: Дифференциальная навигация в SHA-256
## Результаты 22 экспериментов, март 2026

---

## I. ЦЕНТРАЛЬНАЯ ТЕОРЕМА (эмпирическая)

**SHA-256 имеет фазовый переход при раунде R* = 19.**

Для R < R*: дифференциальное пространство имеет **структуру** (градиент).
Для R ≥ R*: дифференциальное пространство **неотличимо от random oracle**.

Этот переход:
- **Резкий**: Δ = -93 при R=14, Δ = -2 при R=19 (мера отклонения от random)
- **Универсальный**: не зависит от метрики (XOR, additive, rotational)
- **Не зависит от техники**: GA, neutral bits, Wang chain — все упираются в R=19
- **Подтверждён из 6+ углов** (CRAZY-3/5/8/11/12/13)

**Следствие**: атака на полную SHA-256 (64 раунда) со сложностью < 2^128
требует преодоления 45-раундового random oracle барьера.

## II. ПЯТЬ СТЕН SHA-256

### Стена 1: Расширение сообщения (C7, CRAZY-9)
σ₀(x) = ROTR⁷(x) ⊕ ROTR¹⁸(x) ⊕ SHR³(x)
σ₁(x) = ROTR¹⁷(x) ⊕ ROTR¹⁹(x) ⊕ SHR¹⁰(x)

Эти функции полностью уничтожают структуру W[0..15] в W[16+].
DW[16+] неотличимо от random (корреляция < 0.02).
Линеаризация (+ → ⊕) даёт carry perturbation = 128/256 = random.

### Стена 2: Полная диффузия (CRAZY-3, CRAZY-5, CRAZY-14)
Forward: полная за 4-8 раундов.
Backward: полная за 4-8 раундов (симметрично).
Степень: растёт ~1 бит/раунд, максимальна к R≈20.
Нет информационного бутылочного горла.

### Стена 3: Независимость барьеров (CRAZY-8)
De[17], De[18], ..., De[24] — статистически независимы:
- Bit correlation: 0.016 ≈ 0
- HW correlation: 0.001 ≈ 0
- Conditional: HW(De18 | De17≤8) = HW(De18) (нет эффекта)

### Стена 4: Ортогональная нелинейность (C1, CRAZY-11)
Ch/Maj (квадратичные) и carry (степень ~32) — ортогональны.
Ядро одного не упрощает другой.
GA в полном пространстве ΔM: HW≈random minimum (нет градиента при R≥19).

### Стена 5: Дефицит степеней свободы (CRAZY-7, CRAZY-15)
Доступно: 82 нейтральных бита (до 153 с diff в W[9])
Требуется: 128+ бит constraints для коллизии
82 < 128 → birthday bound 2^128 неизбежен.

## III. СТРУКТУРНАЯ ЗОНА (R < 19)

В этой зоне SHA-256 имеет ЯВНУЮ структуру:

### 3.1 Wang Chain (R=1..16)
Контролирует De для раундов 2..16. Cost = O(1).
82.6 нейтральных бит (C9).

### 3.2 Neutral Bit Navigation (R=17..18)
Ландшафт HW(De) имеет **градиент** (CRAZY-1):
- GA снижает HW(De[17]) с 16 до 7 (~9 бит)
- Каскад: r17 = -7.15 бит, r18 = -7.51 бит (CRAZY-4)
- Нейтральные биты **разделяемы** по раундам

### 3.3 Direct Near-Collision (R=17, CRAZY-15)
Single-bit delta W[15]^=0x10000000 → HW=10/256.
4/8 регистров = EXACT MATCH (c,d,g,h).
Estimated collision cost: ~2^30 hash evaluations.

### 3.4 Differential Placement Tradeoff (CRAZY-7)
W[0]: 82 нейтральных, De17=15
W[9]: 153 нейтральных, De17=17 (больше бит, но дальше от барьера)
W[12]: 90 нейтральных, De17=1.3 (ближе к барьеру)

## IV. МАТЕМАТИЧЕСКАЯ ФОРМУЛИРОВКА

### 4.1 Задача
Найти M ≠ M' ∈ {0,1}^512 такие что
  H₀ + compress(H₀, M) = H₀ + compress(H₀, M')
  ⟺ compress(H₀, M) = compress(H₀, M')

### 4.2 Birthday bound
Для random oracle f: {0,1}^512 → {0,1}^256:
  P(∃ M≠M': f(M) = f(M')) ≈ 1 при N ≈ 2^128 evaluations.

### 4.3 Structural attack (наш подход)
1. Фиксируем ΔW[0] = δ (стартовый дифференциал)
2. Строим Wang chain: корректируем W'[1..15] для De[2..16] = 0
3. Нейтральные биты: 82 бита свободы в W[1..15]
4. GA/greedy: оптимизируем De[17..18]
5. Барьеры De[19..64]: неконтролируемы → 46 × 2^32 constraints

### 4.4 Почему structural < birthday
Total pool = B × 2^82 (B base messages × 2^82 neutral settings).
Birthday: (B × 2^82)² / 2^256 ≈ 1 → B = 2^46.
Total: 2^46 × 2^82 = 2^128 = birthday. ■

### 4.5 Фазовый переход
Определим metric(R) = min_{|ΔM|>0} E_M[HW(H_R(M) ⊕ H_R(M⊕ΔM))] / 128.

Экспериментально:
  metric(R) ≈ 0 для R ≤ 16 (exact collisions найдены GA)
  metric(R) ≈ 0.3 для R = 17
  metric(R) ≈ 0.7 для R = 18
  metric(R) ≈ 1.0 для R ≥ 19

Переход от 0 к 1 — в интервале [17, 19].

## V. НЕРЕШЁННЫЕ ВОПРОСЫ

### 5.1 Можно ли расширить структурную зону?
Mendel et al. находят sufficient conditions для всех 8 регистров
(не только De). Это позволяет им атаковать 27 раундов (semi-free-start).
Наш подход контролирует только De → 18 раундов.

**Гипотеза**: полный sufficient-condition подход с GA-оптимизацией
может сдвинуть фазовый переход до R* ≈ 24-28.

### 5.2 Точная стоимость SHA-256/17 коллизии?
CRAZY-15 показал HW=10 при 500K trials (single-bit delta).
Экстраполяция: ~2^30 для коллизии. Не верифицировано.

### 5.3 Есть ли структура ЗА фазовым переходом?
Наши тесты проверяли до order-18 higher-order differentials.
Возможно, order-100+ differentials выявят structure при R=20+.
Но тестирование невозможно (2^100 evaluations per test).

### 5.4 Формальное доказательство?
Можно ли ДОКАЗАТЬ (не только измерить) что SHA-256 при R≥19
неотличима от random? Это потребует нетривиальной математики
(связь с provable security of ARX constructions).

## VI. ПРАКТИЧЕСКИЕ ВКЛАДЫ

1. **Фазовый переход R*=19** — первое точное измерение для SHA-256
2. **Барьерная независимость** — количественное подтверждение
3. **Additive metric artifact** — предостережение для будущих исследований
4. **Neutral bit navigation** — формализация как компас в diff space
5. **Degree growth** — ~1 бит/раунд, нормальный рост, нет аномалии
6. **SHA-256/17 near-collision: HW=10** — конкретная верхняя оценка

---

## VII. ЗАКЛЮЧЕНИЕ

SHA-256 при 64 раундах имеет security margin 45 раундов
(от фазового перехода R*=19 до полного числа раундов 64).

Для прорыва нужно ОДНО из:
1. Расширить structure zone с R*=19 до R*≈45+ (нужен full sufficient-conditions подход)
2. Найти structure ЗА фазовым переходом (нужна новая математика)
3. Преодолеть birthday bound 2^128 (нужен fundamentally new paradigm)

Ни одно из этих направлений не может быть решено вычислительными
экспериментами на Python. Требуется либо:
- Реализация на C/C++ для атаки на SHA-256/20+ (практическая)
- Новая алгебраическая теорема (теоретическая)
- Автоматизация sufficient-conditions ala Mendel (engineering)
