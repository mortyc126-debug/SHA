## ШЕСТЬ ИДЕЙ И ИХ РЕЗУЛЬТАТЫ

### Идея #1: Конструктивная инверсия

**Гипотеза:** конструируем g62 с g62[b31,b30]=0,
находим два W[0] с одинаковым g62 → одинаковый H?

**Результат:** пары с одинаковым g62 найдены, HW(ΔH)=129.
Одинаковый g62 не гарантирует одинаковый H — лавина.

---

### Идея #2: Решётка Фишера

**Гипотеза:** corr(gi,gj)≈0.53 образует граф-решётку,
клики = "бесплатные" движения → коллизия.

**Результат:** b0 входит в 45/48 слотов W[16..63].
Все биты W[0] симметричны в schedule.
Матрица Фишера изотропна — нет привилегированных направлений.
**Объяснение высоких corr:** все biты проходят через одни W[r],
корреляция через общую нелинейную функцию, не структура решётки.

---

### Идея #3: Многоблочный

**Гипотеза:** carry1=0 → H1 смещён → усиляем в H2.

**Результат:**
```
phi(H2[0][b29] | carry1=0): 0.5105
phi(H2[0][b29] | random):   0.5250
Δ = -0.0145 ★
```
Слабый сигнал (-1.4%) подтверждён.

---

### Идея #4: Вращение d_min

**Гипотеза:** пара с HW=97 → вращение W* → HW<97?

**Результат:** вращение не уменьшает HW (Δ=-9 бит, ухудшение).
HC на (W1^W*, W2^W*) не конвергирует к меньшему HW.

---

### Идея #5: Birthday на орбите H[7]

**Гипотеза:** W → H[7](W) как [2^32]→[2^32],
birthday за O(2^16) → коллизия H[7], может полная?

**Результат:**
```
Цикл H[7]-орбиты: 2M шагов, коллизий H[7] найдено
Мин HW внутри орбиты: 96 бит
|S(H[7] на орбите)| ≈ 2^17 (≈128K уникальных значений)
```

**Ключевой факт:** |S| ≈ 2^17, не 2^32. Орбита ограничена
пространством хэш-таблицы 2^17, не реальным сжатием.

**6.1:** Birthday H[7] | carry=0, W1≠W2:

```

**6.2:** + фильтр инвариантов g62[b31,b30]=0:

```

**6.3:** Birthday 64-bit ключ H[7]||H[6]:
```
Ожидаем коллизию за N=2^32 примеров — недостижимо
```

**6.4:** Сжатие образа H:
```
10K carry=0 примеров → ~97 столкновений в HT размера 2^17
Интерпретация: артефакт HC-коррелированных входов
Реальный |S(H|carry=0)| ≈ 2^32 (нет сжатия)
```

**T_NO_IMAGE_COMPRESSION (П-1201) ★★★★★**

**SHA-256 не сжимает образ H при carry=0.**
H равномерно распределён даже в подмножестве W[0] с carry=0.
Это фундаментальное свойство конструкции Меркля-Дамгора.

## ПЛАН: ЧТО НУЖНО ДЛЯ РЕАЛЬНОЙ КОЛЛИЗИИ

```
Метод           | Нужно           | Есть сейчас   | Разрыв
────────────────┼─────────────────┼───────────────┼──────────
Birthday H[7]   | N=2^16 carry=0  | N=20K=2^14.3  | ~4× ускор
Birthday H[0..7]| N=2^128 carry=0 | N=20K          | 2^114
Orbit birthday  | |S|<2^16        | |S|≈2^17-2^32  | нет
Inv. lastrnds   | полная инверсия | частичная      | 60 раундов
```

**Реалистичный следующий шаг:** N=2^16 = 65,536 carry=0 примеров.
Birthday H[7] даст ~0.5 коллизий H[7].
Из них: P(полная коллизия H[0..7]) = 1/2^224 → нет.

**Вывод:** birthday H[7] находит частичные коллизии за достижимое N,
но полная коллизия H[0..7] требует 2^128 — недостижимо.
