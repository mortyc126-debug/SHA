## НОВЫЕ ПРИНЦИПЫ

**Правило 41 (П-530):** carry[r]=0 смещает выходные слова только при r≥62.
Выходной барьер симметричен входному — SHA-256 защищён с обоих концов.

**Правило 42 (П-532):** Частичная коллизия H[7] имеет ускорение 5.26×
через birthday при N=70K HC входов. Это реальный измеримый эффект.

вперёд через дифференциалы (Wang-style) + назад через carry (наш мостик).

### T_OUTPUT_BARRIER (П-531) ★★★★★

**Утверждение:** SHA-256 имеет симметричную двойную барьерную структуру.

```
Входной барьер (r=0→2):
  MI(W[0]; state[r]) падает в 10427× за 1 раунд
  После r=2: информация о входе исчезает

Выходной барьер (r=61→62):
  carry[r] влияет на выход только при r≥62
  При r≤61: carry полностью поглощается перемешиванием

Управляемая зона: только раунды 62-63 (2 из 64)
```

**Это фундаментальное свойство архитектуры SHA-256.**

Причина: финальное сложение `H[i] = reg_i + IV[i]` — единственное место
без перемешивания. Влияние carry на регистры убывает экспоненциально
при движении к началу.

---

### T_PARTIAL_COLLISION_BOUND (П-532) ★★★★

**Утверждение:** через carry-атаку достижимы частичные коллизии
максимум в H[6] и H[7] — 64 бита из 256.

**Данные (N=70,000 HC входов):**
- Коллизий H[7]: **3** (ожидание случайное: 0.6)
- Фактор ускорения: **5.26×**
- SS→H[7] детерминировано: **99.9%** (7338/7345)
- Найденная пара:
  ```
  W_a = 0xc34e161d → H[7] = 0x60677e11
  W_b = 0x682be40a → H[7] = 0x60677e11  ★ СОВПАДАЕТ
  ```

**Это первая экспериментально найденная частичная коллизия SHA-256
через carry-структуру.**

## РАЗДЕЛ 118: ПУТЬ ВПЕРЁД (П-536..П-545)

### Теорема T_TWO_FRONT (П-536) — гипотеза

**Идея:** соединить два подхода которые работают независимо.

```
ВПЕРЁД (Wang-style дифференциалы):
  W[0] → дифференциальный путь → STATE[r=32]
  Цель: контролировать STATE при r=32

НАЗАД (наш мостик):
  carry[62,63]=0 → H[6], H[7] предсказуемы
  Цель: STATE[r=32] → выход с нужными свойствами

ВСТРЕЧА:
  STATE[r=32] где оба условия выполнены
  → частичная коллизия в H[6,7]
  → расширение на другие слова?
```

---

### Φ-МНОГООБРАЗИЕ КАК МОСТ

**Свободные раунды Φ:** {1, 4, 9, 10, 19, 21}

Это раунды где carry не детерминирован IV и K. Именно здесь
дифференциальные атаки имеют наибольшую свободу.

**Гипотеза T_PHI_DIFFERENTIAL (П-537):**
Φ-свободные раунды создают "окно" для дифференциального пути
который совместим с нашим carry-мостиком.

Если дифференциальный путь проходит через r=4,9,10 (три свободных раунда)
и достигает r=62 с нужным STATE_SUM — это полный путь
от входа до управляемого выхода.

---

### ПЛАН АТАКИ ДВУХ ФРОНТОВ

```
Шаг 1: КАРТОГРАФИРОВАНИЕ
  Измерить corr(STATE[r=32], SS[r=62])
  Если корреляция есть → MITM через STATE[32] возможен
  (выходной барьер не обязательно симметричен входному)

Шаг 2: ДИФФЕРЕНЦИАЛЬНЫЙ ПУТЬ
  Найти W[0]_a, W[0]_b с ΔW = W_a XOR W_b
  такой что STATE[r=32]_a = STATE[r=32]_b
  (нейтральный бит или дифференциальный характеристик)

Шаг 3: CARRY ФИЛЬТР
  Из совпадающих STATE[32] выбрать те
  где carry[62]=0 и carry[63]=0
  → H[6] и H[7] совпадут для обоих входов

Шаг 4: РАСШИРЕНИЕ
  Найти дополнительные условия для H[0]-H[5]
```
