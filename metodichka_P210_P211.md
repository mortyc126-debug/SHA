## П-211: Ранг δstate — ключевой результат

### Ранг по раундам

| Раунд r | Ранг δstate[r] | always-zero | МИТМ O(2^?) |
|---------|---------------|-------------|-------------|
| 17 | 160 / 256 | **96** | 2^80 |
| 18 | 192 / 256 | 64 | 2^96 |
| 19 | 224 / 256 | 32 | 2^112 |
| 20 | 256 / 256 | 0 | 2^128 |

Паттерн: ранг растёт на 32 бит/раунд (один регистр в раунде).

### Структурный факт (новый!)

**96 always-zero бит = δf[17]=δg[17]=δh[17]=0 тождественно**

Это прямое следствие адаптации δe[2..16]=0:
- f[r+1] = e[r] → δf[r+1] = δe[r]
- g[r+1] = f[r] → δg[r+1] = δf[r]  
- h[r+1] = g[r] → δh[r+1] = δg[r]

При δe[14]=δe[15]=δe[16]=0:
```
δf[17] = δe[16] = 0  ✓
δg[17] = δf[16] = δe[15] = 0  ✓
δh[17] = δg[16] = δf[15] = δe[14] = 0  ✓
```

**Это T_STATE17_FGH_ZERO из П-197 — тот же SC, найден независимо!**

После r=17: δg[18]=δh[18]=0 (64 нуля — сдвиг регистров продолжается).

### Ранг δe[r] (только один регистр)
δe[17], δe[18], δe[19], δe[20] — все ранг 32/32.
Birthday по δe: O(2^16) пар с каждой стороны.

## Вывод: МИТМ стоимость

Чистый МИТМ по δstate[17] стоит **O(2^80)**.
Это хуже Wang chain (O(2^104)) — нет смысла.

Но есть новое направление:

**δf[17]=δg[17]=δh[17]=0 всегда** — это фиксирует 96 бит.
Значит matching нужен только по оставшимся 160 битам.
O(2^80) — всё равно дорого.

## Открытие сессии П-207..П-211

Адаптивный трейл — это новый класс дифференциального трейла SHA-256:

```
Свойства:
  δe[2..16] = 0        (15 раундов, адаптация)
  δf[17]=δg[17]=δh[17] = 0  (SC из П-197, бесплатно!)
  ΔW[17] = 1 бит       (П-207, HW=1)
  
Структура расписания:
  ΔW[0] = d0 (1 бит)
  ΔW[2] = d0 (копия, ~50%)
  ΔW[17] = min 1 бит
```

Это аналог Wang chain, но в другой алгебре (не требует SC на биты W[0..15]).
