## ИСПРАВЛЕНИЕ МАТРИЦЫ НУЛЕЙ

Исходная матрица нулей из П-120 содержала off-by-one ошибку.

### Правильная матрица нулей (верифицирована P-123b, 100к Wang-пар)

```
Состояние после раунда r (без условия de[17]=0):

r    da    db    dc    dd    de    df    dg    dh
17   ~16   ~16   ~16   ~16   ~16   ~16   0.00  0.00
18   ~16   ~16   ~16   ~16   ~16   ~16   ~16   0.00
19+  ~16   ~16   ~16   ~16   ~16   ~16   ~16   ~16

Нули заканчиваются на r=18.
```

### Аналитическое объяснение (окончательное)

```
SHA-256 сдвиг: новый h = старый g, новый g = старый f, новый f = старый e

Wang даёт de[k]=0 для k=2..16.
-> df[k]=0 для k=3..17  (df[r] = de[r-1])
-> dg[k]=0 для k=4..18  (dg[r] = df[r-1] = de[r-2])
-> dh[k]=0 для k=5..19  (dh[r] = dg[r-1] = de[r-2]... wait)

Точно: dh[r+1] = dg[r]
de[k]=0 для k=2..16 → dg[r]=0 для r=4..18
→ dh[r+1]=0 для r=4..18 → dh[r]=0 для r=5..19

НО: прямое измерение показывает нули только до r=18 (dh):
  dg[17]=0, dg[18]=0 (из de[15,16]=0)
  dg[19]: df[18]=de[17] — это уже может быть nonzero без de[17]=0

Итоговая правильная матрица:
  r=17: dg=0, dh=0  (из de[15]=de[16]=0)
  r=18: dh=0        (из dg[17]=0)
  r=19+: все ненулевые
```

## КЛЮЧЕВЫЕ РЕЗУЛЬТАТЫ П-116..П-123

```
Доказано экспериментально:
  T_ZERO_H_CHAIN: dh[17]=dh[18]=0 (не до r=20 как думали)
  T_DE18_FORMULA: de[18] = dd[17] + dW64[18]  [точно при de[17]=0]
  P(de[18]=0 | de[17]=0) = 2^-32  [confirmed]

Не подтверждено:
  T_SOL_CASCADE стоит O(2^96) — хуже чем O(2^32) birthday
  Нейтральные биты не дают ускорения для de[18]=0
```

## ФИНАЛЬНЫЙ СТРАТЕГИЧЕСКИЙ ИТОГ

```
БАРЬЕР НЕПРОБИВАЕМ нашими методами:
  Для каждого раунда r>=18: P(de[r]=0) = 2^-32 независимо.
  Нет структуры которая увеличивает эту вероятность.

ЕДИНСТВЕННЫЙ ПУТЬ К ПОЛНОЙ КОЛЛИЗИИ:
  Дифференциальный трейл с P(trail) >> 2^-(32*(R-17)).
  Это требует корреляций между de[r] на РАЗНЫХ раундах.

НАШ ВКЛАД ДЛЯ MILP/SAT TRAIL SEARCH:
  1. T_DEAD_ROUNDS: rounds 1..17 free (initial conditions)
  2. T_ZERO_H_CHAIN: dh[17]=dh[18]=0 (constraints for r=18)
  3. T_DE18_FORMULA: de[18]=dd[17]+dW64[18] (exact equation)
  4. T_STATE17_FGH_ZERO: df[17]=dg[17]=dh[17]=0

  Эти ограничения сужают пространство поиска трейла
  для раундов 17..31, потенциально расширяя Li et al. 2022
  с 31 до 35+ раундов.
```
