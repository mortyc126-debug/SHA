# Collision Flow Algebra (CFA)
## Новая математическая структура для SHA-256
### Апрель 2026

---

## АКСИОМЫ

### A1: Объект — не значение, а ПОТОК коллизий

Стандартный подход: фиксировать M, вычислить SHA(M), искать M'.
CFA: работать с МНОЖЕСТВОМ {(M, M') : SHA(M) = SHA(M')} как единым объектом.

Множество коллизий = подмногообразие в (Z/2^32)^{32} (два сообщения × 16 слов).
Размерность: 512 - 256 = 256 (512 входных бит - 256 уравнений).

### A2: Раунд = оператор на ПОТОКЕ, не на значении

Стандартно: state[r+1] = F_r(state[r]).
CFA: поток коллизий Ω[r] трансформируется: Ω[r+1] = F_r#(Ω[r]).

F_r# — pushforward оператор. Он действует на МНОЖЕСТВО, не на точку.

### A3: Коллизионный поток имеет ТЕМПЕРАТУРУ

Температура T[r] = мера "разброса" потока на раунде r.
- T[0] = 0: все коллизии начинают из одного IV (δstate = 0)
- T[r] растёт: поток расширяется через раунды
- T[R] = T_final: температура на выходе

Если T_final < T_random → поток ХОЛОДНЕЕ random → коллизии кластеризованы.
Если T_final = T_random → поток неотличим от random → birthday optimal.

### A4: Три фазы потока

**Фаза I (r=0..16): Управляемый поток.** Wang chain. δe=0. Поток одномерный (δW[0]).
**Фаза II (r=17..59): Свободный поток.** "Шум". 42 раунда хаоса. T растёт.
**Фаза III (r=60..63): Кристаллизация.** Последние раунды. Судьба хеша определяется.

N1 (каскад рождений) = описание Фазы III.
N3 (детерминированный шум) = описание Фазы II.

### A5: Температура — НЕ метрика

Температура = свойство МНОЖЕСТВА, не расстояние между точками.
Не нарушает запрет на метрики (термостат убивает метрики на парах).
Температура — свойство ПОТОКА ЦЕЛИКОМ.

## ОПРЕДЕЛЕНИЯ

### D1: Коллизионный поток

Ω(M₀) = {δ ∈ (Z/2^32)^16 : SHA(M₀ ⊕ δ) = SHA(M₀)}

|Ω| ≈ 2^256 для случайного M₀.

### D2: Промежуточный поток

Ω_r(M₀) = {(δ, state_r(M₀⊕δ)) : δ ∈ Ω(M₀)}

Проекция потока на состояние в раунде r.

### D3: Температура потока

T[r] = H(state_r | δ ∈ Ω) / 256

H = энтропия Шеннона проекции потока на state в раунде r.
T ∈ [0, 1]. T=0 = все коллизии проходят через одно состояние. T=1 = random.

### D4: Плотность потока

ρ[r](s) = |{δ ∈ Ω : state_r(M₀⊕δ) = s}| / |Ω|

Вероятность что случайная коллизия проходит через состояние s в раунде r.

Для random: ρ = 2^{-256} (равномерно).
Если ρ неравномерно → поток структурирован → можно эксплуатировать.

### D5: Коллизионная корреляционная функция

C[r₁, r₂] = E_{δ∈Ω}[state_{r₁}(δ) · state_{r₂}(δ)] - E[state_{r₁}] · E[state_{r₂}]

Корреляция состояний ВНУТРИ потока между раундами r₁ и r₂.

Для random потока: C = 0 для всех r₁ ≠ r₂.
Если C ≠ 0 → поток имеет "память" → структура.

## ТЕОРЕМЫ (для проверки)

### T_CFA_1: Температура Фазы I

T[r] = 0 для r = 0..4 (δstate = 0, все коллизии тривиально совпадают).
T[r] ≈ r/64 для r = 5..16 (линейный рост, Wang chain расходится на a-ветви).

### T_CFA_2: Температура Фазы III

T[63] < 1? Или T[63] = 1?

Если T[63] < 1: поток ХОЛОДНЕЕ random → коллизии кластеризованы на выходе.
Если T[63] = 1: birthday optimal.

ЭТО КЛЮЧЕВОЙ ТЕСТ CFA. Никто его не проводил.

### T_CFA_3: Плотность потока в зоне кристаллизации

ρ[60](s) — равномерна или нет?

Раздел 167: "судьба определяется последними 3 раундами."
Это значит: ρ[60] МОЖЕТ быть неравномерна — некоторые state[60] 
ведут к коллизиям ЧАЩЕ.

### T_CFA_4: Корреляция Phase I → Phase III

C[16, 60] = ? 

Если C > 0: информация из Phase I (Wang chain) ДОХОДИТ до Phase III.
Если C = 0: фазы полностью декоррелированы.

Раздел 173: state coupling lift 2.0-2.5× для cross-block.
Это НАМЁК на C > 0.

## ПЛАН ВЕРИФИКАЦИИ

1. Для мини-SHA (n=4, R=8): вычислить Ω полностью (уже есть).
2. Измерить T[r] для r = 0..8.
3. Измерить ρ[r] для ключевых раундов.
4. Измерить C[r₁, r₂] для всех пар.
5. Сравнить с random потоком (случайное подмножество того же размера).

Если T[R] < T_random или C ≠ 0 → CFA нашла структуру.
Если T[R] = T_random и C = 0 → CFA не даёт преимущества.

## СВЯЗЬ С N1-N5

- N1 (каскад рождений): CFA отслеживает поток через рождения
- N2 (безметрическая): температура = свойство множества, не метрика
- N3 (дет. шум): фаза II = шумовая фаза, T[r] измеряет её эффект
- N5 (schedule): schedule определяет W[16+], влияя на фазу II

CFA — единая рамка для N1-N3.
