# П-218: Три направления — итоги

## A: SHA-256/k Birthday коллизия

### Результат
За 200K адаптивных трейлов:
- min HW(δa[64]) = **4**
- **6 birthday коллизий** по δa[64] найдено (ожидалось ~5)

```
BIRTHDAY! δa[64]=0x1eac59c7: W0₁=0x2b30322c W0₂=0xbfc42090
BIRTHDAY! δa[64]=0x60f6e09b: W0₁=0x31ef2875 W0₂=0x38f8f7ae
BIRTHDAY! δa[64]=0xb3ab087b: W0₁=0xa47caf4f W0₂=0xb08d0dd4
```

### Что это означает
δa[64]₁ = δa[64]₂ означает: у двух разных сообщений первые 32 бита
**дифференциала хэша** совпадают. Это **НЕ** SHA-256/32 коллизия —
для этого нужен δa[64]=0 у одного трейла.

Для полной SHA-256/32 коллизии нужно ~2³² пар (HW=0 за ~2³² трейлов).
Это ~4300 секунд на нашем C коде. **На GPU — минуты.**

## B: Двухблочный адаптивный трейл

### Результат
Двухблочный трейл r=0..31 (блок 1: r=0..15, блок 2: r=16..31):
- b2_success = 100% (адаптация блока 2 работает автоматически!)
- min sum HW(ΔW[17..31]) = **190** за 500K пар (среднее 190/15=**12.7**/раунд vs 16 random)
- min HW(ΔW[32]) = **4** за 200K пар

### Ключевой факт
δstate[16]=(δa,δb,δc,δd,0,0,0,0) → δe[16]=δf=δg=δh=0 ✓
Это автоматически создаёт условия для адаптации блока 2!

**Двухблочный трейл:** δe[2..16]=0, δe[18..32]=0 — 30 нулевых δe из 32!

## C: Ранг δstate — Wang W0 vs random

### Структура δstate[16] (все одинаковые!)
```
Конфиг              | ранг_full | ранг_δa | нулевые регистры
--------------------|-----------|---------|------------------
random W0, d0=8     |    128    |   32    | δe δf δg δh
Wang W0, d0=1       |    128    |   32    | δe δf δg δh
random W0, d0=1     |    128    |   32    | δe δf δg δh
```

Все три конфига дают **одинаковую структуру**! Wang W0 не особенный.

### Ранг по раундам (Wang vs random)
```
r= 5: random=123, Wang=97  → Wang на 26 меньше (при r=5!)
r=10: 128 = 128
r=16: 128 = 128
```
При r=5 Wang W0 даёт ранг=97 vs 123 random — **значимое отличие**.
При r≥10 разница исчезает.

## Главный результат П-218: Двухблочный трейл

**T_2BLOCK_ADAPTIVE**: Адаптивный трейл автоматически продлевается
через границу блоков. При δe[16]=0 (гарантировано блоком 1),
блок 2 строится с δe[18..32]=0 без дополнительных условий.

```
Структура двухблочного трейла:
  r=0..15:  δe[2..16]=0  (15 нулей)
  r=16..31: δe[18..32]=0 (15 нулей)
  ΔW[32] = 4 бита (min, аналог ΔW[17]=1 бит)
  δstate[32] = (δa,δb,δc,δd,0,0,0,0)
```

Это **30 из 32 раундов** с δe=0! Прямой аналог Wang chain
но в адаптивной (не birthday) модели.

## Следующий шаг П-219

**Продолжить двухблочный трейл до 48 раундов (3 блока):**
- Блок 1: r=0..15, δe[2..16]=0
- Блок 2: r=16..31, δe[18..32]=0
- Блок 3: r=32..47, δe[34..48]=0

Ожидаем: δe=0 в **45 из 48 раундов**.
ΔW[48] = ? (аналог ΔW[17] и ΔW[32]).

Если ΔW[48] мало → **48-раундовый адаптивный трейл**!
Это превысит рекорд Li et al. (31 раунд) в адаптивной модели.

**Но:** это free-start атака (блоки независимы), не Wang chain.
