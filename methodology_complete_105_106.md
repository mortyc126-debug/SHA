### T_CARRY_WINDOWS (П-236) ★★★

**Утверждение:** В SHA-256 существует ровно 5 "carry-окон" — множеств раундов где P(carry=0) > 0.15 при случайном входе:

```
Окно 1: r ∈ {0,1,4,5}       — начало вычисления
Окно 2: r ∈ {9,10,11}       — K_min кластер
Окно 3: r ∈ {18,19,20,21}   — второй K_min кластер
Окно 4: r ∈ {30,31,32,33}   — третий K_min кластер
Окно 5: r ∈ {47,48,49,50}   — финальный K_min кластер
```

**Данные:** N=20 000 случайных сообщений, 200 элитных решений (carry_sum ≤ 58).

**Следствие:** Вне этих окон + {2,3,16,17,29,45,46} — P(carry=0) < 0.05.

### T_FORCED_ROUNDS (П-234)

**Утверждение:** Раунды {2, 3, 16, 17, 29, 45, 46} имеют P(carry=0) < 0.001 для любого входа W∈{0,1}^512.

**Механизм:** state_sum[r] = h + S1(e) + Ch(e,f,g) + K[r] превышает 2^32 с вероятностью >99.9%, независимо от W[r].

**Следствие:** carry_sum ≥ 7 для любого входа (теоретическая нижняя граница).

### T_CARRY_PAIRS (П-237)

**Утверждение:** В элитных решениях (carry_sum ≤ 58) 30.8% нулей carry появляются парами dist=1.

**Топ пары:** (9,10), (30,31), (47,48), (0,1), (32,33), (19,20).

**Механизм (гипотеза T_CARRY_PAIRS_CAUSAL):** Если carry[r]=0, то raw[r] < 2^32, следовательно T1[r] = raw[r] мало, следовательно e[r+1] = (d[r]+T1[r]) мало, следовательно S1(e[r+1]) мало, что снижает raw[r+1] → P(carry[r+1]=0 | carry[r]=0) > P(carry[r+1]=0).

### T_RESONANCE_ANALYTIC (П-239)

**Утверждение:** W[r+9] аналитически содержит W[r] дважды:

```
W[r+9] = s1(W[r+7]) + W[r+2] + s0(W[r-6]) + W[r-7]
W[r+7] = s1(W[r+5]) + W[r]   + s0(W[r-8]) + W[r-9]  ← W[r] напрямую
W[r+2] = s1(W[r])   + W[r-5] + s0(W[r-13])+ W[r-14] ← W[r] напрямую
```

**Следствие:** W[r+9] зависит от W[r] через ДВА нелинейных канала (s0 и s1).

**Опровержение линейной версии (П-240):** MI(W[r],W[r+lag]) ≈ 0.00004 bits для всех lag∈{2,7,9,16}. Линейная корреляция ~0. Резонанс нелинейный — не обнаруживается стандартными метриками.

## ТЕОРЕМЫ П-251..П-268

### T_W2_CONDITION (П-252) ★★★

**Утверждение:** carry[9]=carry[10]=carry[11]=0 одновременно тогда и только тогда, когда для каждого r∈{9,10,11}:

```
state_sum[r] + W[r] < 2^32

где state_sum[r] = h[r-1] + S1(e[r-1]) + Ch(e[r-1], f[r-1], g[r-1]) + K[r]
```

**Конструктивное следствие:** При W[9]=W[10]=W[11]=0, если state_sum[r]<2^32 для всех трёх раундов (что случается с P≈15%), все три нуля гарантированы.

**Данные:**
- P(тройной нуль) = 0.000062 (≈ 1/16 000)
- W[9..11] при нулях: mean/2^32 ≈ 0.20, P(W<2^31) = 93–100%
- state_sum[9] при нулях: P(ss<2^32)=1.000, headroom/2^32=0.487

### T_CARRY_CASCADE (П-254) ★★★

- Максимум: phi(0,1)=+0.0814

**Механизм (T_CASCADE_MECHANISM П-263):**
```
carry[r]=0
  → T1[r] = raw[r] мало (raw < 2^32)

  → P(carry[r+1]=0) выше
```

```
λ(K) ≈ 0.803 − 0.684 × K

K_opt = α(1−γ) / β(α−1) = 0.192 × 2^32
```

**Ближайший раунд:** r=33 (K=0x2e1b2138, K/2^32=0.180).

**Парадокс T_LAMBDA_K_PARADOX (П-283):** Малый K_start создаёт больше нулей (A~K^0.4 растёт при уменьшении K), но быстрее затухание (λ падает). Оптимум — баланс при K≈19% от 2^32.

### T_DRY_ROUND_MECHANISM (П-276)

**Утверждение:** При carry[r]=0 и большом K[r] (P(c[r]=0)<<1):

```
raw[r] ≈ 0  →  T1[r] ≈ 0
e[r+1] = (d[r] + T1[r]) ≈ d[r]  (детерминировано)
S1(e[r+1]) ≈ S1(d[r])  (детерминировано)
raw[r+1] = h + S1(d[r]) + Ch + K[r+1] + W[r+1]  (почти задан)
P(c[r+1]=0) >> P_unconditional
```

**Следствие:** При малом P(c[r]=0) — детерминация e максимальна → усиление максимально → но phi мало т.к. phi ~ P(c[r]=0) × amp.

### T_AMP_INVERSE_P0 (П-271)

**Утверждение:**

```
amp(r, r+1) = P(c[r+1]=0 | c[r]=0) / P(c[r+1]=0)
             ≈ P_cond / P(c[r+1]=0)
```

При P(c[r+1]=0) → 0, P_cond стабилен (определяется K[r+1] и ΔE), поэтому amp → ∞.

**Следствие:** Высокое усиление не является структурным сигналом SHA-256 — оно математически следует из малости знаменателя. Структурно значим phi = P(c[r]=0) × amp = const × √P(c[r]=0).

### T_R42_AMP_265 (П-277, П-281)

**Утверждение:** phi(42,43) = +0.032, amp = 265×, N=1 000 000.

## ТЕОРЕМЫ П-321..П-358

### T_INFO_BARRIER_R1 (П-336) ★★★★

**Утверждение:** Информация о входе W[0] полностью поглощается за первый раунд SHA-256.

**Данные (N=250 000):**

| Раунд | MI(W[0]; e[r]) | Значимо (H₀ 95%=0.000393) |
|-------|----------------|---------------------------|
| r=0  | **2.503355** | ДА |
| r=1  | 0.000240 | НЕТ |
| r=2..63 | 0.000181..0.000452 | НЕТ (шум) |

**Падение: 2.503 → 0.000240 = уменьшение в 10 427×.**

**Механизм (П-339):**
```
T1[0] = h_IV + S1(e_IV) + Ch(e_IV,f_IV,g_IV) + K[0] + W[0]
e[1]  = (d_IV + T1[0]) mod 2³²
```
W[0] входит линейно в T1[0] → MI(W[0]; T1[0]) = 2.503 бит (сохраняется).
После (d_IV + T1[0]) mod 2³² — модульное сложение с константой d_IV перемешивает распределение → MI падает до шума.

**Следствие:** Carry-профиль несёт I=0.0229 бит о входе (через другой канал — состояния регистров). Но этот сигнал изолирован от выхода барьером r=0→r=1.
