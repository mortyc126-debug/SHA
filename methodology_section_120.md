## КЛЮЧЕВЫЕ ТЕОРЕМЫ

### T_BACKWARD_EXACT (П-571) ★★★★★

**Утверждение:** Обратный раунд SHA-256 вычислим точно.

**Формулы (обратный раунд r):**
```
Вход: STATE_after[r], W[r], K[r]

T2 = S0(b_after) + Maj(b_after, c_after, d_after)
T1 = a_after - T2
d_before = e_after - T1
h_before = T1 - S1(f_after) - Ch(f_after, g_after, h_after) - K[r] - W[r]
[a,b,c,e,f,g]_before = [b,c,d,f,g,h]_after
```

**Верификация:** 64 обратных раунда от STATE[64] → STATE[0] = IV точно.

**Следствие:** Зная H[0..7] и W[0..63], можно точно вычислить STATE[r] для любого r — от конца к началу.

---

### T_H7_DIRECT (П-572) ★★★★

**Утверждение:** Последние 3 слова выхода определяются STATE[62] напрямую.

```
H[7] = g[62] + IV[7]    (g-регистр после 62 раундов)
H[6] = f[62] + IV[6]
H[5] = e[62] + IV[5]
```

**Следствие:** Коллизия H[7] ⟺ коллизия g[62]. Атака на g[62] — атака на H[7].

---

### T_CARRY_BIAS_CHAIN (П-573) ★★★

**Утверждение:** carry[63]=0 создаёт сжатие g[62], но не структуру:

- E[g[62] | carry=0] = 187M vs E[g[62] | random] = 2147M (11.5×)
- Это следствие T_ANTICORR_GEOMETRY (усечение по SS+W63 < THRESH)
- |S(g[62])| > 2^18 — пространство не сжато структурно

### Путь 1: Свободный блок (ВЫСОКИЙ ПРИОРИТЕТ)

Текущие атаки используют только W[0] свободным (W[1..15]=0).
Если разрешить все 16 слов свободными:

```
Степеней свободы: 512 бит (не 32)
→ можно задавать STATE[R] точно для R ≤ 16 раундов
→ MITM через STATE[16]: 2^8 с каждой стороны
→ Коллизия STATE[16] → одинаковый выход!
```

Это полная атака — но она требует реализации на полном блоке.
