## П-94: НЕЙТРАЛЬНЫЕ БИТЫ + НЕЗАВИСИМОСТЬ Sol_Wang vs Sol_32

### B: Нейтральные биты Wn[12..15]

```
Wn[12]: 32/32 нейтральных битов  avg|Δδe17| = 0  ← ПОЛНАЯ свобода
Wn[13]: 32/32 нейтральных битов  avg|Δδe17| = 0  ← ПОЛНАЯ свобода
Wn[15]: 32/32 нейтральных битов  avg|Δδe17| = 0  ← ПОЛНАЯ свобода
Wn[14]:  2/32 нейтральных        avg|Δδe17| = 85M (малые изменения)
Wn[11]:  1/32 нейтральных

Итого: 99/480 нейтральных позиций в Wang-пространстве.
```

**Структурная причина (Задача 3, П-95):**
```
W64[16] = s1(W[14]) + W[9] + s0(W[1]) + W[0]
Зависит ТОЛЬКО от W[0], W[1], W[9], W[14]
НЕ зависит от W[2..8], W[10..13], W[15]

Wn[12,13,15] не входят в W64[16] ни прямо, ни через расписание.
→ Изменение Wn[12,13,15] компенсируется адаптивными DW[r],
  δe[2..16]=0 сохраняется, δe[17] не меняется.
```

Верифицировано прямым экспериментом:
```
Flip Wn[12]^=0xFFFFFFFF: δe[17] unchanged ✓
Flip Wn[13]^=0xFFFFFFFF: δe[17] unchanged ✓
Flip Wn[15]^=0xFFFFFFFF: δe[17] unchanged ✓
Flip Wn[11]^=0xFFFFFFFF: δe[17] CHANGED ✗
Flip Wn[14]^=0xFFFFFFFF: δe[17] CHANGED ✗  (входит в W64[16] через s1)
```

### C: Sol_Wang ⊥ Sol_32

```
Корреляция Пирсона δe17_Wang vs δe17_Add: r = 0.0019 ≈ 0
→ Статистически независимы!

P(Оба = 0):       0/100000
P(Только Wang):   0/100000
P(Только Add):    0/100000
→ Оба P ≈ 2^-32 по отдельности, независимы.
```

### T_DA14_CONTROLLABLE (П-94..П-95)

```
dδa[14]/dWn[13] = ce9f6526 (v2=1)
→ v2=1 → нечётная чувствительность → Ньютон применим:
   δa[14] = target при изменении Wn[13] одним шагом

Wn[13..15] нейтральны для δe[17], но контролируют δa[14..16].
96 свободных битов для управления a-регистром.
```

## П-95: MITM + СТРУКТУРА РАСПИСАНИЯ

### Зависимость W64[r] от W[0..15]

```
W64[r] для r=0..15: W64[r] = W[r] (прямое)
W64[16] = s1(W[14]) + W[9] + s0(W[1]) + W[0]

Для раунда 17: участвует только W64[16] из новых слов.
W[2..8], W[10..13], W[15] → W64[r] только для r≥17 (вне окна).
```

### MITM результаты

```
P(δe17[0:k]=0) = 2^-k точно (ratio ≈ 1.0 для k=1..8)
→ Нет структуры для ускорения birthday.

Коллизии δe17(W0)+δe17(W0')=0: 3 за 2^17 ops (0.03s)
→ Два разных блока, не одна пара.

Частичный барьер δe17[0:16]=0: 5 за 231k iter (1.8s)
→ P = 2^-16 точно.
```

## П-96: NEWTON — ОТРИЦАТЕЛЬНЫЙ РЕЗУЛЬТАТ

### T_NEWTON_INAPPLICABLE_WANG

```
Newton success: 0/10000

Причина: DW[r] адаптируются каскадом при изменении Wn[r].
Изменение Wn[r] вызывает пересчёт DW[r+1..15] → δW[16] меняется →
δe17(Wn[r]) — сложная нелинейная функция, не линейна.

P(sens нечётна) ≈ 0.5 для r=1..11 — условие Newton выполняется,
НО после Newton шага каскад пересчитывает DW[r+1..15],
что разрушает предполагаемую линейность.
```

### Ключевое наблюдение

```
δe17(Wn[1]+d) для d=1..8:
  4a90fb6b, eba6bd5f, 868a1fb6, cff1a54f...  (хаотичные скачки ~2^30)

δe17(Wn[11]+d) для d=0..15:
  4a90fb6b, 4a80fb6a, 4a910b6b...  (малые скачки ~2^20, почти периодичны)

Wn[11] "почти нейтрален" — маленькие Δ из-за позднего входа в расписание.
```

## ГЛАВНЫЙ ВЫВОД СЕРИИ П-81..П-97

```
1. CASCADE_17 (аддитивный): 14/15 нулей за O(1), барьер De17=0 за O(2^32)
2. WANG_CHAIN: 15/16 нулей за O(1), барьер δe17=0 за O(2^32)
3. Wang 12.8× быстрее аддитивного (скорость вычисления)
4. Оба барьера: P=2^-32, Hensel неприменим, нейтральные биты → 2^104 решений

ФУНДАМЕНТАЛЬНЫЙ БАРЬЕР SHA-256:
  P(De17=0) = P(δe17=0) = 2^-32 в ЛЮБОМ пространстве.
  Это информационный барьер, не алгоритмический.
  Нелинейность Ch/Maj/carry обеспечивает полное перемешивание за ≤4 раундов.
  Никакой аналитический метод (Newton, Hensel, MITM) не преодолевает барьер.
```
