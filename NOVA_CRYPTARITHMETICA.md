# Nova Cryptarithmetica SHA-256 (НК) — Рабочая карта v1.0

*Формальный каркас новой математики для SHA-256.*
*Каждое утверждение трассируется к эксперименту из methodology_v20.md.*

---

## 0. Назначение

НК — не «ещё одна алгебра хеша», а язык для:

- учёта **конкретных битов** и их носителей (слово, индекс, раунд);
- разделения **полного состояния** и **наблюдаемого канала**;
- **стоимости** дифференциальных гипотез в терминах «P-позиций»;
- **жёсткого расписания** как подмногообразия в {0,1}^1024;
- **параметризации** линейной части (длина сдвигового регистра L) и вывода birthday-границы как функции (L, N).

Любая новая идея пропускается через **фильтр §9**.

## 1. Алфавит битов (уровень представления)

**Определение 1.1 (битовая ячейка).** Множество индексов I: тройки (раунд r, регистр, бит b ∈ {0..31}), плюс биты W[r]_b, carry-позиции. Элементарный символ — значение в F_2 с привязкой к I.

Абстракции без индекса (например, «высокая энтропия» без координаты) в НК не являются первичными объектами.

**Мотивация (методичка):** коллизионные и различительные конструкции опираются на побитовые условия (Wang SHA-1; биты H[6][29], H[7][30] у различителя; П-92, П-966, П-72) — §63, §113, §134, §150.

## 2. Два уровня: состояние и наблюдение

**Определение 2.1 (уровень 0).** Полная эволюция SHA-256 на одном блоке:
```
Ψ : {0,1}^512 × IV → S_64
```
S_r — кортеж всех промежуточных слов, переносов и производных величин до раунда r. Уровень 0 **детерминирован**.

**Определение 2.2 (уровень 1).** Наблюдатель задаётся каналом:
```
Π : S_64 → O
```
Типично O = {0,1}^256 (дайджест). Промежуточное состояние S_r для r<64 в Π не входит.

**Определение 2.3 (разрыв).** Безопасность в смысле НК — количественные характеристики того, какие функции на S_r невыразимы как функции от Π(Ψ(·)).

**Мотивация:** §106, §110-113, §121.

## 3. Дифференциальный путь и стоимость

**Определение 3.1 (путь).** Для пары входов: XOR-дифференциалы δa[r], δe[r] на раундах r=0..63, согласованные с арифметикой mod 2^32.

**Определение 3.2 (GPK-строка).** Для каждой операции сложения: строка над {G, P, K}. P-позиции — биты, где carry зависит от контекста (неопределён без знания конкретных значений).

**Определение 3.3 (стоимость пути).**
```
C(path) = 2^Λ(path)
Λ(path) = Σ_{r, оп} |P(оп, r)|
```
Λ — число бит «угадывания». Wang chain: раунды 1-16 дают 0 P-бит, раунд 17 даёт 32 → Λ=32, C=2^32.

**Мотивация:** §19-21 (T_DIFFERENTIAL_WEIGHT), §33 (T_MOD_ADD_DIFF_PROB), §145 (Zone-2).

## 4. Расписание как ограничение

**Определение 4.1.** W[0..15] свободны (512 бит). Расширение:
```
Σ : {0,1}^512 → {0,1}^2048,  (W[0..15]) ↦ W[0..63]
```
Линейно над F_2 (XOR + ротации).

**Утверждение 4.2 (полный ранг).** rank(Σ) = 512. Нет нетривиального δW[0..15], обнуляющего всё расширение. Любой ненулевой δW[0..15] порождает ненулевой δW[16..63].

**Мотивация:** §92 (T_SCHEDULE_DIFFUSION), §102 (T_SCHED_FULL_RANK), §155 (T_SCHEDULE_ISLAND_BOUND).

В НК расписание — жёсткое алгебраическое подмногообразие; дифференциалы обязаны лежать в Im(Σ).

## 5. Параметр сдвига L и индекс κ

**Определение 5.1.** MD-хеш с N регистрами по 32 бита и сдвиговым регистром длины L. SHA-256: N=8, L=4.

**Экспериментальный факт (verify_shift_length.py):**
```
L=3 → κ=32, birthday=2^112
L=4 → κ=0,  birthday=2^128
L=5 → κ=0,  birthday=2^128
```

**Роль в НК:** L и N входят в формулу для κ и birthday bound. Численные коэффициенты верифицируются кодом.

## 7. Предсказания P1-P4

### P1. Shield potential S(r) для r=17..63

**Гипотеза.** S(r) = f(HW(δa[r-3]), Struct_GPK(r), Mask(W[r])) — монотонна по числу незаэкранированных P-позиций.

**Проверка:** на Wang-парах вычислить эмпирическую S_emp(r) и сравнить с формулой.

### P2. Минимальная стоимость через r=17..20 в GPK-терминах

**Гипотеза.** Минимальное число P-позиций при данном δW[16..19] — явная функция от профиля δW[16..19] (через Im(Σ)).

**Проверка:** перебор по δW[0], построение GPK, подсчёт P-позиций.

### P3. Формула κ(L, N)

**Гипотеза-кандидат:**
```
κ(L, N) = max(0, 32 × (N/2 - L))
```
Согласуется с (L=4, N=8) → κ=0 и (L=3, N=8) → κ=32.

**Проверка:** семейство модифицированных хешей с разными L, N.

### P4. Классификация экранирующих узлов

**Типы:**
1. Ch-shield: e константен на бите → Ch линейна
2. Maj-shield: 2 из 3 аргументов совпадают
3. Carry-shield: GPK=K или G → carry детерминирован

**Проверка:** статистика по Wang-парам и случайным парам, доли типов по раундам.

## 8. Зоны применимости

Для полного SHA-256 в зоне раундов 17-59 простые скалярные метрики не несут устойчивой информации (термостат τ≈1.8 раунда). В НК: C(path) и P-позиции валидны для конкретного пути; глобальная метрика на δstate без привязки к пути — не объект атаки.

**Мотивация:** §149, §153, §192.

## 10. Глоссарий

- **НК** — Nova Cryptarithmetica SHA-256
- **Ψ** — полная эволюция (уровень 0)
- **Π** — наблюдательный канал (уровень 1)
- **Σ** — расписание как линейное отображение 512→2048 бит
- **Λ** — число P-позиций на пути; C = 2^Λ
- **κ(L,N)** — индекс зависимости выходных слов
- **Shield** — узел где △ поглощается (Ch/Maj/Carry типы)
