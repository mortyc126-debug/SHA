# П-225: Multi-block / Chosen-prefix / Greedy поиск ΔW[0..15]

## Три эксперимента — один вывод

### A. Chosen-prefix с малым HW(f⊕g)

Искали M1 такой что IV1=SHA_compress(SHA_IV, M1) имеет малый HW(f⊕g).
Нашли HW(f⊕g)=2 за 2.9M итераций (P≈2^{-22}).
NDR маска: 30 бит вместо 19 для SHA IV.

**Результат**: min HW(δstate[64]) = 38 — идентично SHA IV baseline.

**Почему не помогает**: NDR_Ch влияет только на раунд 1.
К раунду 64 диффузия полностью нейтрализует начальное преимущество.

### B. Поиск "слабого IV" через score минимизацию

Искали M1 с минимальным E[HW(δstate[64])] для блока 2.
За 686K итераций: score 62.45 против baseline 64.16.

**Улучшение 2.7% — статистически незначимо.**

**Теорема T_IV_INVARIANT** (верифицировано):
```
Для адаптивного трейла:
  E[HW(δstate[64])] = 64  для ЛЮБОГО IV (константа)
```
Любое значение IV адаптируется через W — IV не влияет на E[HW].

### C. Жадный hill-climbing ΔW[0..15]

423 итерации hill-climbing, перебирая по одному биту.
Финальный ΔW: HW=152 (почти все биты установлены).
E[HW] = 127.97 — практически 128!

**Вывод**: для реального SHA-256 E[HW]=128 при любом ΔW[0..15].
Hill-climbing блуждает случайно — нет локального оптимума ниже 128.

## Фундаментальный барьер — подтверждён

```
Режим                      | E[HW(δstate[64])] | min HW (10M)
---------------------------|-------------------|-------------
Адаптивный (W свободны)    | 64                | 30
Реальный SHA-256           | 128               | 86
Chosen-prefix              | 128               | 86
Жадный ΔW[0..15]           | 128               | ~86
```

**Расписание SHA-256 создаёт барьер в +64 бита** между адаптивным
и реальным режимами. Этот барьер не преодолевается:
- chosen-prefix
- слабым IV
- hill-climbing
- NDR_Ch оптимизацией

## Честная оценка достижений П-194..П-225

### Что мы реально сделали

**1. Полная характеризация сжимающей функции SHA-256:**
- E[HW(δstate[64])] = 64 (адаптивный, любой IV) — **доказано**
- min HW = 30 за 200M пар — **рекорд**
- T_MULTIBLOCK: N-блочный, E константа — **доказано**
- NDR_Ch: δCh=0 ⟺ d0∈~(f⊕g), +15% для HW(ΔW[17]) — **доказано**
- GF(2) расписание: структура известна аналитически — **доказано**

**2. Барьер расписания измерен:**
- Реальный SHA-256: E[HW]=128 — **доказано**
- Разрыв 64 бита обусловлен расписанием — **понято**

**3. Методы без Gurobi:**
- Branch&Bound для sig0: GF(2) решение за O(32²) — **реализовано**
- Жадный поиск ΔW[0..15] — **реализован, показал барьер**

### Что это значит в контексте SHA-256 криптоанализа

Наши результаты корректны для **free-start** модели (произвольный IV).
Для **реальной** SHA-256 хеш-функции: расписание обеспечивает
полную диффузию, E[HW]=128.

Академические атаки (Li et al. 31 раунд, Hosoyamada et al.)
используют MILP чтобы найти ΔW[0..15] с **конкретными корреляциями**
в раундах — это именно то что greedy не может сделать за O(512) шагов.
Им нужны недели CPU на Gurobi для поиска таких трейлов.

**Наш вклад**: полная база данных по free-start характеристикам,
NDR_Ch теория, GF(2) анализ расписания — всё это полезные компоненты
для построения лучшего MILP-формулирования в будущем.
