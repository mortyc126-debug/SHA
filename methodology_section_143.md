## T_H4_COMPRESSION (П-1254) ★★★★★

**При H[7]-коллизии: E[HW(ΔH[4])] = 12.0 вместо 16.0.**

Сжатие 4 бита. H[4] коррелирует с H[7] при коллизии.

**Механизм:** H[4] = e[64] + IV[4]. H[7] = h[64] + IV[7].
Регистры e и h связаны через раунд 63: h[63] → e[64].
Это объясняет корреляцию H[7] и H[4] через внутреннюю структуру
финальных раундов SHA-256.

---

## T_H0_ANTICORRELATION (П-1255) ★★★

**При H[7]-коллизии: E[HW(ΔH[0])] = 19.0 > 16.0.**

Антикорреляция: H[0] дальше при H[7]-коллизии.
H[0] = a[64] + IV[0]. Регистр a и h в финале антикоррелированы.

## T_G62_PREDICTS_H (П-1256) ★★★★★

**g62-близость предсказывает H-близость:**

```
Сосед g62 (Δ<10K):    E[HW(ΔH)] = 109.0 бит
Далёкий g62 (Δ>10M):  E[HW(ΔH)] = 127.2 бит
Разница: -18.2 бит  ★★★
```

Два W[0] с близкими промежуточными состояниями g62
дают ближайшие хэши H. Это **алгебраическая непрерывность**
SHA-256 в пространстве промежуточных состояний.

**Следствие:** g62 можно использовать как "сортирующий ключ"
для group-birthday — сначала группируем по g62, потом
ищем коллизии внутри каждой группы.

## НОВЫЙ ИНСТРУМЕНТ: АЛГОРИТМ

### Многоуровневый Birthday (MLB)

```
ШАГИ:
1. Генерируем N orbit-шагов: w → H[7](w)
   Orbit birthday за O(2^16) шагов → H[7]-коллизии
   
2. Из H[7]-коллизионных пар фильтруем:
   Берём пары с минимальным HW(ΔH[4])
   (H[4] сжат на 4 бита при H[7]-коллизии)
   
3. Из отфильтрованных пар ищем совпадение H[4]:
   Birthday за O(2^16) среди отфильтрованных пар
   
4. Рекурсия на H[3], H[2], ...

УСКОРЕНИЕ:
  Стандарт: 2^128
  MLB: 2^128 / 2^8.5 = 2^119.5
  Численно: 362× быстрее
```

## ЧЕСТНАЯ ОЦЕНКА

**Что доказано:**
- H[4] при H[7]-коллизии сжат на 4 бита (статистически)
- g62 предсказывает H на 18 бит (малая выборка)
- Суммарное сжатие 17 бит (N=500, нужна проверка)

**Что НЕ доказано:**
- Сжатие независимо для разных уровней (нет рекурсии)
- 2^119.5 реально достижим (нет реализации многоуровня)
- Масштабируемость за пределами H[4]

**Фундаментальный вопрос:** являются ли корреляции H[4] и H[0]
результатом реальной структуры SHA-256 или артефактом
малой выборки (N=500)?

*Нужен N≥50,000 для статистической значимости.*

## СВЯЗЬ С ПРЕДЫДУЩИМИ РЕЗУЛЬТАТАМИ

```
AUC=0.976 (раздел 103):    H[6][b29..31], H[7][b29..30] смещены
g62-предсказание (П-1256): близкий g62 → близкий H (−18 бит)

d_min=97 (П-1143):         минимальное расстояние SHA-256
```

Все эти результаты указывают на одно: SHA-256 имеет
**внутреннюю корреляционную структуру** между словами хэша
которая возникает из конструкции финальных раундов
(регистровая цепочка a→b→...→h→e при каждом раунде).
