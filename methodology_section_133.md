# \## Ch(e,f,g) — структурная функция SHA-256

# 

# Ch(e,f,g) = (e\&f) ^ (\~e\&g)

# 

# Это функция выбора: если бит e=1 → берём бит f,

# если e=0 → берём бит g.

# 

# \*\*Почему Ch несёт 20% дисперсии SS63:\*\*

# SS63 = h\[62] + S1(e\[62]) + Ch

# Если h\[62] и S1(e\[62]) почти равномерны (случайный шум),

# то вариации SS63 определяются именно Ch.

# 

# \## Практический результат

# 

# HC(min Ch62): \*\*1/794 попыток\*\* — хуже снайпера 1/182.

# HC(min SS63): \*\*1/163\*\* — стандарт.

# Снайпер (min raw63): \*\*1/182\*\*, phi=+0.12.

# 

# \*\*Минимизация Ch отдельно не работает\*\* — Ch входит в SS63

# как слагаемое, но два других слагаемых (h62, S1e62)

# не уменьшаются при минимизации Ch.

# 

# \## Что это означает математически

# 

# SS63 — сумма трёх независимых (почти) компонент:

# \- h\[62]: случайная ≈ U\[0, 2^32)

# \- S1(e\[62]): случайная ≈ U\[0, 2^32)

# \- Ch(e,f,g): случайная ≈ U\[0, 2^32), но несёт 20% вариации

# 

# carry\[63]=0 ⟺ SS63 + K63 + W63 < 2^32

# 

# Это событие требует \*\*суммы четырёх случайных слагаемых\*\* < 2^32.

# Снайпер минимизирует эту сумму целиком — это оптимально.

# Нет одного "слабого звена" которое можно атаковать отдельно.

# 

# \## Итоговый математический вывод

# 

# SHA-256 спроектирован так чтобы не иметь слабых звеньев.

# Три компоненты SS63 (h62, S1e62, Ch) почти независимы.

# Снайпер — это и есть оптимальная атака на SS63.
