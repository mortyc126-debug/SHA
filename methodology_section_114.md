## КЛЮЧЕВЫЕ ТЕОРЕМЫ

### T_COMMIT_HIDING (П-482) ★★★★★

**Утверждение:** Chosen-prefix adversary нарушает hiding property
commitment scheme Commit(m) = SHA256(r || m) через выбор r.

**Данные (N=5000, c63=0, HC):**

| Бит | P_HC | P_base | phi | Z |
|-----|------|--------|-----|---|
| H[6][b29] | **0.715** | 0.499 | **+0.216** | 30.5 |
| H[6][b31] | 0.284 | 0.502 | −0.218 | −30.8 |
| H[6][b28] | 0.376 | 0.502 | −0.126 | −17.8 |
| H[7][b29] | 0.638 | 0.499 | **+0.139** | 19.6 |
| H[7][b30] | 0.637 | 0.500 | +0.137 | 19.4 |

**Комбинированный score предсказатель:**
- P(score>0 | HC, c63=0) = **0.802**
- P(score>0 | random) = 0.502
- Advantage = **0.300**, Z = **42.5**

**Применимость:** любая схема где:
1. Committer выбирает r (часть входа)
2. Верификатор проверяет конкретный бит выхода хеша

### T_POW_NEGATIVE (П-481) ★★★

**Утверждение:** HC не даёт преимущества в Bitcoin PoW.

**Данные:**
- Avg leading zeros: HC=0.980 vs random=0.998
- Улучшение: **−0.018** (HC хуже случайного)
- Причина: PoW смотрит на начало хеша, bias в конце H[7]

**Следствие:** Смещение строго локализовано в H[6]-H[7][b28-b31].
Ведущие биты хеша не затронуты.

---

### T_KEY_BIAS (П-484, П-486) ★★★

**Данные (N=5000):**
- Предсказуемых битов: **8 из 256**
- Суммарная потеря энтропии: **0.494 бит**
- Эффективная энтропия: 255.5 / 256 бит

**Структура смещённых битов:**
```
H[6][b28]: phi=−0.126  H[6][b29]: phi=+0.216
H[6][b30]: phi=−0.127  H[6][b31]: phi=−0.218
H[6][b27]: phi=−0.055  H[7][b28]: phi=−0.050
H[7][b29]: phi=+0.139  H[7][b30]: phi=+0.137
```

**Следствие:** Снижение < 0.5 бита из 256 — практически незначимо
для криптографических применений.

---

### T_CHAIN_DEAD (П-490) ★★★★

**Утверждение:** Пассивная цепочка линейности не существует.
SHA256(H_biased) = случайный без структуры.

**Данные (N=30000):**

| Уровень | Значимых/256 | max|phi| | Вывод |
|---------|-------------|----------|-------|
| 0 (H_hc) | **11** | **0.214** | Структура |
| 1 (SHA256(H0)) | 0 | 0.024 | Шум |
| 2 | 0 | 0.019 | Шум |
| 3-5 | 0 | 0.018 | Шум |

**Уничтожение за ровно 1 итерацию.**

**Причина:** Информационный барьер r=0→r=1 (T_INFO_BARRIER_R1, П-336)
работает для любого входа. H_biased не является "особым" входом
для следующего SHA-256 — он случаен с точки зрения компрессии.

---

### T_BARRIER_UNIVERSAL (П-492) ★★★★

**Утверждение:** Каждый вызов SHA-256 является независимым
информационным барьером. Структура входа не передаётся через хеш.

**Следствие для атак:**
- SHA256d (Bitcoin): второй хеш независим от структуры первого
- HMAC-SHA256: внутренние итерации независимы
- KDF с итерациями: каждая итерация сбрасывает структуру

**ЕДИНСТВЕННЫЙ способ сохранить структуру** — контролировать
вход на каждой итерации (Q93, управляемая цепочка).

## ОТКРЫТЫЙ ВОПРОС Q93 (ВЫСОКИЙ ПРИОРИТЕТ)

**Управляемая цепочка линейности:**

```
Шаг 1: W0 = HC_optimize() → SHA256(W0) = H0 (c63=0, структурирован)
Шаг 2: W1 = HC_optimize_from(H0) → SHA256(H0 || W1) = H1 (c63=0?)
Шаг 3: W2 = HC_optimize_from(H1) → SHA256(H1 || W2) = H2 (c63=0?)
...
```

**Вопрос:** Можно ли на каждом шаге применить HC чтобы
следующий выход тоже имел c63=0?

**Если да:**
- Структура поддерживается произвольно долго
- Атака применима к итерированным схемам
- HMAC(key, msg) потенциально затронут если msg контролируется

**Сложность:** каждый шаг требует ~55K HC рестартов.
N шагов = N × 55K итераций SHA-256.
