## П-99: FREE-START + δa АДАПТАЦИЯ

### Структура W64[17]

```
W64[17] = s1(W[15]) + W[10] + s0(W[2]) + W[1]
Зависит от: W[1], W[2], W[10], W[15]
НЕ зависит от: W[0], W[3..9], W[11..14]

→ W[13] нейтрален для δe[18] (как и для δe[17])!
```

**T_NEUTRAL_BITS_17_AND_18 (П-99):**
```
43 бита нейтральны одновременно для δe[17] И δe[18]:
  Wn[13]: 32/32 нейтральных для обоих ← главное открытие
  Wn[14]:  7/32 нейтральных для обоих
  Wn[15]:  4/32 нейтральных для обоих

Для δe[18]=0 нужно: δa[14] = -δW64[17] = 9bc84a16
(не ноль, а конкретное значение!)
```

### Парадокс Wn[13] разрешён

```
Wn[13] нейтрален для δe[18] И не влияет на δa[14]:
  Wn[13]+1 → Δ(δa[14])=0, Δ(δW64[17])=0, Δ(δe[18])=0

Значит: Wn[13] просто НЕ ПРОХОДИТ через расчёт δa[14]!
Это структурное свойство расписания SHA-256.
```

## П-100: ЦЕПОЧКА НЕЙТРАЛЬНОСТИ Wn[r]

### Аналитическая карта

```
W[r] входит в W64[s] через рекуррентность:
W64[s] = s1(W64[s-2]) + W64[s-7] + s0(W64[s-15]) + W64[s-16]

Первый W64[s] содержащий W[r] (через зависимости):
  W[0,1,9,14] → W64[16]  → нейтральны для de[17..16]
  W[2,10,15]  → W64[17]  → нейтральны для de[17]
  W[3,11]     → W64[18]  → нейтральны для de[18]
  W[4,12]     → W64[19]  → нейтральны для de[19] аналитически
  W[5,13]     → W64[20]  → нейтральны для de[20] аналитически
```

### Экспериментальная верификация (Wn[13])

```
Flip Wn[13]^=0xFFFFFFFF:
  r=17: de unchanged ✓
  r=18: de unchanged ✓
  r=19: de CHANGED ✗ ← здесь цепочка обрывается!

Аналитическое объяснение:
  W[13] не входит в W64[16..19] напрямую
  НО: δa[14] зависит от W[0..13] через компрессию
  δa[14] меняется → δa[15] меняется → δe[19]=δa[15]+δW64[18] меняется!
```

### Таблица нейтральных слов для данной пары

```
Wn[r]   нейтрален до   nb(de17)  объяснение
Wn[0..11]  r=16           0       входят в W64[16] или раньше
Wn[12]     r=17          32      W[12] в W64[19] не входит → свобода
Wn[13]     r=18          32      W[13] в W64[20] → нейтрален для de[17,18]
Wn[14]     r=16           7      W[14] в W64[16] через s1 → частично
Wn[15]     r=17          32      W[15] в W64[17] → нейтрален для de[17]
```

## П-101: T_DE_r_LINEAR + T_SOL_SIZE

### T_DE_r_LINEAR (верифицирована)

```
δe[r] = δa[r-4] + δW64[r-1]   для r=17, 18 ✓

Верификация для найденной пары W0=c97624c6:
  r=17: δa[13]=39d2b820, δW64[16]=c62d47e0, сумма=00000000 ✓
  r=18: δa[14]=e8e4709e, δW64[17]=6437b5ea, сумма=4d1c2688 ✓
  r=19: ✗ нарушается (δa-цепь нелинейна)
  r≥19: ✗ формула неверна из-за нелинейного распространения δa
```

### T_SOL_SIZE (П-101)

```
|Sol_17| ≥ 2^96  (Wn[12,13,15] нейтральны для de[17]: 3×32=96 бит)
|Sol_18| ≥ 2^64  (Wn[13,15] нейтральны для de[17] И de[18]: 2×32=64 бит)

Смысл: имея одну пару (de[17]=0), мы бесплатно получаем
2^96 других пар с de[17]=0 и 2^64 пар с de[17]=de[18]=0
варьируя нейтральные биты Wn[12,13,15].
```

### Почему нет дальнейшего расширения без birthday

```
de[19]=0: нужно δa[15]=-δW64[18]
δa[15] = f(δa[14], ...) — нелинейная функция δa[14]
δa[14] = f(W[0..13]) — depends on W[13] through compression!
Wn[13] нейтрален для W64[18], но не для δa[15] → нет нейтральных битов!

→ Для каждого нового нуля de[r] нужен новый birthday (P≈2^-32)
→ Стоимость N нулей de[17..16+N] = (2^32)^N
→ Нейтральные биты расширяют |Sol|, но не снижают P нахождения
```

```
5. Барьер P=2^-32 фундаментален для каждого нового нуля de[r]
```

### Фундаментальный барьер SHA-256

```
P(de[17]=0) = P(de[18]=0 | de[17]=0) = 2^-32

Нейтральные биты:
  |Sol_17| ≥ 2^96 — огромное пространство решений для de[17]=0
  |Sol_18| ≥ 2^64 — для de[17]=de[18]=0

Но: нахождение первого решения требует 2^32 birthday независимо.
SHA-256 достигает этого через:
  - Нелинейность Ch/Maj: полная диффузия за ≤4 раундов
  - Расписание W64[r]: каждое новое слово вносит свежую энтропию
  - T_BARRIER17_EXACT: любой дифференциал даёт P=2^-32 для r=17
```
