## EXP A: DP_eff(r) — оптимальный δ

```
r    MD5(best δ)   SHA-1(δW0=1)  SHA-256(δW0=1)
1..15  1.000000       0.000000       0.000000
16     0.000000       0.000000       0.000000
```

**MD5 DP=1.0 на r=1..15 — детерминированный ноль дифференциала!**

---

## EXP B: Объяснение феномена DP=1.0

```
δM[11]=1 для MD5:
  M[11] используется в раундах: 11, 18, 34, 61
  До раунда 11: δM[11] не влияет ни на одно состояние
  → P(a==a' AND b==b' AND c==c' AND d==d') = 1.0 для r=0..11

Верификация (10 000 пар):
r= 1..11: P = 1.000000  ← состояния ИДЕНТИЧНЫ
r=12:     P = 0.000000  ← δ появляется
```

**Это не статистика — это детерминированный факт:**
до первого использования M[k] в раунде, δM[k] не оказывает
никакого влияния на состояние. DP=1.0 точно.

## УНИФИЦИРОВАННАЯ ТЕОРИЯ: T_DEAD_ROUNDS

### Мёртвые раунды для MD5 и SHA-256

```
ОПРЕДЕЛЕНИЕ: "Мёртвые раунды" для δM[k] =
  раунды r < first_use(M[k]) где δM[k] не влияет на состояние
  → DP(state, δM[k]) = 1.0 детерминированно

MD5:     first_use(M[k]) = k (раунд k)
  δM[11]: мёртвые раунды 0..10 (11 раундов)
  δM[0]:  мёртвые раунды — нет (сразу использован)

SHA-256: first_use_in_schedule(W[k]) = k+16 (через расписание W64)
  δW[11]: W[11] входит в W64[27] → мёртвые раунды 0..26 (27 раундов!)
  δW[0]:  мёртвые раунды — нет (W64[0]=W[0] сразу)
```

### Парадокс: у SHA-256 БОЛЬШЕ мёртвых раундов для W[11]!

```
MD5 δM[11]:    11 мёртвых раундов (0..10)
SHA-256 δW[11]: 27 мёртвых раундов (0..26)

Но SHA-256 устойчив, а MD5 взломан. Почему?
```

### Разгадка: барьер r=17 блокирует использование мёртвых раундов

```
SHA-256 барьер r=17:
  До r=17: Wang-цепочка детерминированно обнуляет δe[2..16]
  На r=17: P(δe[17]=0) = 2^-32 — информационный барьер

  δW[11] имеет мёртвые раунды 0..26, НО:
  Wang-атака не может распространить условия ДАЛЬШЕ r=17,
  потому что δe[17] — случайный оракул.
  Мёртвые раунды 17..26 недостижимы через Wang.

MD5 — нет аналогичного барьера:
  Каждое M[k] используется 4 раза в разных раундах.
  Wang строит sufficient conditions для ВСЕХ 64 раундов,
  используя мёртвые периоды каждого M[k] по очереди.
```

---

## НОВЫЙ ВЗГЛЯД НА БАРЬЕР r=17

```
Барьер r=17 = первый раунд где НЕТ ни одного
незадействованного δW с DP=1.0.

После r=16: все W[0..15] уже были использованы
хотя бы один раз в W64[0..16].
Нет ни одного δW с "мёртвыми" раундами дальше r=16.
→ Wang не может строить sufficient conditions для r≥17.

Это объясняет барьер r=17 через расписание, а не через
нелинейность функции раунда. Это более глубокое объяснение.
```

## ГЛАВНЫЙ ВЫВОД П-113

```
Wang-атака = использование "мёртвых раундов" каждого M[k].

MD5: M[k] используется в раундах k, k+16, k+32, k+48.
     До раунда k: δM[k] невидим → 100% свобода условий.
     Wang использует все 16 слов поочерёдно → 64 раунда.

SHA-256: аналогично, но барьер r=17 = конец мёртвых раундов
     для всех W[0..15]. После r=16 нет новых мёртвых раундов.
     Wang не может продолжить цепочку условий.

Барьер r=17 — это не просто "случайный оракул".
Это момент когда ВСЕ слова расписания исчерпали
свои мёртвые раунды одновременно.
```
