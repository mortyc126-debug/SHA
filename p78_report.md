# П-78: Три задачи — итоги (Март 2026)

## Задача A: Матрица бит-битовой диффузии

### T_BIT_LINEAR_R1 подтверждена строго:
```
r=1: 32/32 диагональных элементов P=1.0
     e_1[b] = W[0][b] точно для всех b=0..31.
```

### Медленные пути (P < 0.001) существуют на каждом раунде:
```
r=1: W[0]б2 → e_1б14: P=0.0002  (через carry в sigma0/sigma1)
r=2: W[0]б15 → e_2б3: P=0.0002
r=3: W[1]б13 → e_3б0: P=0.0002
r=4: W[2]б14 → e_4б1: P=0.0002
r=5: W[3]б12 → e_5б0: P=0.0002
```
Паттерн: бит k входного слова → бит ≈k-2 выходного, с вероятностью ~0.0002.
Это в 2500× медленнее чем случайный бит (P≈0.5).

**Применение:** дифференциальный путь по «медленным» битам имеет
экспоненциально меньший вес характеристики.

## Задача B: Кластеры в расписании — ГЛАВНОЕ ОТКРЫТИЕ

### T_SCHEDULE_SPARSE (новая теорема):
```
Каждый входной бит W[j][b] влияет только на ~37% бит
расширенного расписания W_expanded[16..63].
Остальные 63% нулей — структурное свойство кода SHA-256.

(Случайный линейный код: 50% нулей. SHA-256: 63%. Разница: 13%.)
```

Это означает: расписание SHA-256 — **разреженный код** со структурой
которая даёт естественные нули без специального подбора.

### Лучшие кластеры по HW:
```
HW=1: W[0]бит15  → 1070/1536 = 69.7% нулей в W_exp[16..63]
HW=2: W[13]б12 + W[13]б16 → 1044/1536 = 68.0%
HW=3: W[13]б1+W[13]б14+W[0]б16 → 997/1536 = 64.9%
HW=4: W[8]б19+W[5]б18+W[7]б16+W[3]б14 → 966/1536 = 62.9%
```

**Важное наблюдение:** при увеличении HW процент нулей ПАДАЕТ
(от 69.7% до 62.9%). Это означает что нули НЕ суммируются —
разные входные биты активируют разные выходные биты.

**Практический вывод:** W_input с малым HW автоматически даёт
разреженное расписание — большинство W_expanded[k] = 0.
Это прямой вход для построения разреженного дифференциального пути.

## Задача C: Carry Correlation — честный результат

### T_CARRY_INDEPENDENCE (новая теорема):
```
P(e_{r+1} carry-free | e_r carry-free) ≈ P(e_{r+1} carry-free)
Отношение: 0.96..1.05 (не значимо от 1.0).

Carry-free события между раундами СТАТИСТИЧЕСКИ НЕЗАВИСИМЫ.
streak=4 в П-77 был случайным совпадением.
```

**Следствие для Carry Skeleton:** случайный перебор не работает.
Нужен **конструктивный** подход — задать биты 1..5 = 1 аналитически.

**Алгебраическое условие:**
```
e_r биты 1..5 = 1 все  ⟺
(d_{r-1} + T1_{r-1}) & 0x3E = 0x3E
⟺ d_{r-1} + T1_{r-1} ≡ 0x3E (mod 0x40)  (6-битное уравнение)
```
P(случайного) = 1/64. Конструктивное решение: задача в Z/64.
