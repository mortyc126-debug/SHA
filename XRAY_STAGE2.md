# SHA-256 РЕНТГЕН — Этап 2: Количественные параметры

## 1. ТЕРМОСТАТ (главный закон)

```
HW(δstate[r+1]) = β × HW(δstate[r]) + γ

β = 0.625          (коэфф. затухания)
γ = 48.3           (приток)
HW* = γ/(1-β) = 128.7   (аттрактор)
```

Замкнутая форма:
```
HW(δstate[r]) = 128.7 - 124.7 × 0.625^r    (при HW[0]=4)
```

Точность: < 1% ошибки для r ≥ 5. Верифицировано N=3000.
Конвергенция: 90% за 5 раундов.

## 2. ФАЗОВЫЕ ПЕРЕХОДЫ

```
k* = ⌈log₂(w)⌉ = ⌈log₂(32)⌉ = 5

Раунд k*: одновременно происходит:
  - HW(δstate) → 128          (лавина)
  - Алгебраическая степень → 32  (насыщение)
  - Кривизна → 0.50              (пространство решений полностью кривое)
  - Маячок: 248 → 0 дет.бит     (все биты перемешаны)
  - GPK-diff → 16/32             (50% carry различается)
```

## 3. УСИЛЕНИЕ ПО РАУНДАМ

```
r=0: HW=0 → α=∞
r=1: HW=4, α=6.25
r=2: HW=25, α=2.20
r=3: HW=55, α=1.58
r=4: HW=87, α=1.32
r=5: HW=115, α=1.09
r=6: HW=126, α=1.02
r=7: HW=128, α=1.00     ← РАВНОВЕСИЕ
r=8..64: HW≈128, α≈1.00  (стабильно навсегда)
```

α зависит от HW, не от r:
```
HW ∈ [0,8):    α = 6.2
HW ∈ [8,16):   α = 2.8
HW ∈ [48,56):  α = 1.6
HW ∈ [120,128): α = 1.01
HW ∈ [128,136): α = 0.99
HW ∈ [152,160): α = 0.94
```

## 4. ТРИ ИСТОЧНИКА ШУМА (НЕЗАВИСИМЫЕ)

```
┌─────────────┬───────────────────────┬──────────────┬────────────────┐
│ Источник     │ Что стирает            │ Без него     │ Ось            │
├─────────────┼───────────────────────┼──────────────┼────────────────┤
│ Σ₀/Σ₁       │ ПОЗИЦИЮ бит            │ r=25 (12.5×) │ пространственная│
│ Ch/Maj       │ АРИФМЕТИКУ (mod k)     │ r=62 (31×!)  │ числовая       │
│ Carry        │ БИТОВУЮ границу слоёв  │ r=4 (2×)     │ вертикальная   │
└─────────────┴───────────────────────┴──────────────┴────────────────┘

SHA-256 = Σ × (Ch/Maj) × Carry = три ортогональных шума.
Убери один → структура выживает на соответствующей оси.
```

## 5. GPK-ИНВАРИАНТЫ

```
P(P) = 50.0% на каждом раунде (архитектурный инвариант)
P(G) = P(K) = 25.0%
E[P-bits per round] = 112/224 = 50%
E[max P-chain length] = 7 бит
E[число P-chains] = 58 на раунд

Carry match (Wang пары):
  Операции h+Σ₁, +Ch, +K: 100% match (δe=0 → идентичны)
  +W: 67% (Wang коррекция меняет W)
  Σ₀+Maj: 50% (δa≠0)
  d+T1: 65-100%
  T1+T2: 50-68%
  ИТОГО: 60.5% match (10.5% выше random)
```

## 6. SCHEDULE КАК КОД

```
Линейный код [1536, 512] над GF(2)
rank(Σ) = 512 (полный)
dim(ker) = 0 (тривиальное ядро)

Минимальное расстояние: ~466 бит (для single-bit δM)
Min active words: 40 из 48 (для optimized multi-bit δM)
Max zero words: 8 (W[16..23]=0 при определённых δM)

Schedule kernel:
  dim(ker(R=48)) = 1
  dim(ker(R=52)) = 128
  dim(ker(R=56)) = 256
  dim(ker(R=57)) = 288
  dim(ker(R=60)) = 384
  dim(ker(R=63)) = 480
```

## 7. ПОЗИЦИОННАЯ НЕОДНОРОДНОСТЬ

Survival (бит δstate при δM=1, порог P>0.55):
```
Регистры: a(0.6) < b(0.9) < c(1.2) < d(1.6) < e(0.5) < f(0.8) < g(1.0) < h(1.2)
d и h максимум: 6 раундов (= shift L + 2)

Биты:
  bit 10: avg survival = 4.0 раунда  ★ ЛУЧШИЙ
  bit 7:  avg survival = 3.5
  bit 26: avg survival = 3.5
  bit 0:  avg survival = 3.0
  bit 25-31: avg survival = 0.0      ★ ХУДШИЕ

Анизотропия: 4× между лучшим и худшим.
Причина: ротации Σ₁={6,11,25} и Σ₀={2,13,22}.
```

## 8. BACKWARD (обратный ход)

```
Backward divergence (random M2, state[64] fixed):
  r=64: δ=0
  r=63: δ=16
  r=62: δ=32
  r=61: δ=48
  r=60: δ=64
  r=56: δ=128 (полная диффузия)

Обратный раунд (точный):
  T2 = Σ₀(b') + Maj(b', c', d')
  T1 = a' - T2
  d_prev = e' - T1
  h_prev = T1 - Σ₁(f') - Ch(f', g', h') - K[r] - W[r]
  a_prev=b', b_prev=c', c_prev=d', e_prev=f', f_prev=g', g_prev=h'
```

## 9. СТОИМОСТИ (Λ-формула)

```
Λ(path) = Σ P-позиций на пути

Wang cascade (r=1-16):  Λ = 0 (бесплатно)
Один барьерный раунд:   Λ = 16 (avg P-bits)
                        Λ_min = 4-8 (best case)
Зона 2 (r=21-63):      Λ = 43 × 16 = 688
ИТОГО:                  Λ = 688 >> 128 (birthday)

Birthday: 2^{256/2} = 2^128
Thermostat escape: 2^186 (16σ, HARDER than birthday)
```

## 10. ВЫХОД (что видит атакующий)

```
H[0..7] = state_final[0..7] + IV[0..7]   mod 2³²

Слова H: попарно НЕЗАВИСИМЫ (Pearson < 0.008)
Биты H: попарно НЕЗАВИСИМЫ (P(agree) = 0.500 ± 0.01)
κ = 0 (нет алгебраических зависимостей между словами)

При carry[63]=0 (P ≈ 1/10000):
  Ch[62][b30,b31] = 0 строго
  g[62][b30,b31] = 0 строго
  phi(H[7][b29]) = +0.129
  Distinguisher AUC = 0.976 (chosen-prefix)

Без chosen-prefix: SHA-256 ≡ random oracle.
Oracle Distance = 5.3 к раунду 8.
```

## 11. ПАРАМЕТР L (конструктивная безопасность)

```
κ(L, N) = max(0, 32 × (N/2 - L))

SHA-256: L=4, N=8 → κ = 0, birthday = 2^128
Если бы L=3: κ = 32, birthday = 2^112 (на 16 бит слабее!)
Если бы L=5: κ = 0, birthday = 2^128 (без улучшения)

L = 4 = МИНИМАЛЬНОЕ значение для полной безопасности.
```
