# П-216..П-217: Структура δstate и финальный анализ

## П-216: Профиль E[HW(δstate[r])] по раундам

### Главный результат (50K пар, 3 типа трейлов)

| r | случайный | адаптивный | free-start | Δ(адапт−сл) |
|---|-----------|------------|------------|-------------|
| 1 | 2.00 | 2.00 | **0.00** | 0 |
| 2..16 | 128.0 | **64.0** | **0.00** | **−64** |
| 17 | 128.0 | 80.0 | 2.00 | −48 |
| 18 | 128.0 | 96.0 | 31.6 | −32 |
| 19 | 128.0 | 112.0 | 63.2 | −16 |
| 20+ | 128.0 | 128.0 | 128.0 | 0 |

Адаптивный трейл держит E[HW]=**64** (половина) на протяжении
раундов 2..16, затем за 3 раунда возвращается к 128.

### Число нулевых δe не влияет на min HW(δstate[64])
k=0..15 нулевых δe: min HW(δstate[64]) = 92..93 (одинаково).
Разница 3 бита (П-215) — статистический шум.

## П-217: Правильная структура δstate

### Полный разбор (верифицировано на 5000 пар)

```
δstate[r=2..16] = (δa[r], δa[r-1], δa[r-2], δa[r-3], 0, 0, 0, 0)
```

Это следствие сдвигового регистра:
- δe[r]=0 → δf[r+1]=δe[r]=0, δg[r+2]=0, δh[r+3]=0
- δb[r+1]=δa[r], δc[r+2]=δa[r], δd[r+3]=δa[r]

**E[HW]=64 = 4 × E[HW(δa)] = 4 × 16**

### Ранги δstate[r]

| r | ранг_full | ранг_δa | МИТМ по full | МИТМ по δa |
|---|-----------|---------|--------------|------------|
| 2..16 | **128** | 32 | O(2^64) | O(2^16)* |
| 17 | 160 | 32 | O(2^80) | — |
| 20+ | 256 | 32 | O(2^128) | — |

*МИТМ по δa[16] — birthday, но не коллизия (δb..δd разные)

### Почему МИТМ O(2^16) НЕ работает как атака

Birthday по δa[16]: два трейла с δa₁[16]=δa₂[16].
Но δb₁[16]=δa₁[15] ≠ δa₂[15]=δb₂[16] (разные W[15]).
→ δstate[16] у них разные → нет связи между трейлами.

Для полного МИТМ нужно совпадение всего δstate[16] (128 ненулевых бит).
Стоимость: O(2^64) — не лучше стандарта.

## Честный итог сессии П-207..П-217

### Верифицированные новые факты

```
T_DW17_HW1:     HW(ΔW[17])=1 (C, 20M пар, 19с)
T_STATE_STRUCT: δstate[r=2..16]=(δa,δa[r-1],δa[r-2],δa[r-3],0,0,0,0)
T_HALFSTATE:    E[HW(δstate)]=64 для r=2..16 (вдвое меньше random)
T_RANK_128:     ранг(δstate[r=2..16])=128 (не 256) из-за 4 нулевых регистров
T_RANK_DA:      ранг(δa[r])=32 для r≥5 (полный ранг одного регистра)
T_NO_CORR_64:   HW(ΔW[17]) не коррелирует с HW(δstate[64])
```

### Границы подхода

- МИТМ: min O(2^64) (из-за ранга 128 в δstate[16])
- Коллизия SHA-256: не найдена
- Преимущество над случайным поиском: 3 бита в min HW(δstate[64]) — статистика

### Практическая ценность

**Адаптивный трейл** — новый инструмент анализа SHA-256:
- P=100% построения (vs 2^{-104} Wang chain)
- Измеримая структура δstate за 15 раундов
- Новый параметр: ранг δstate[r] = 128 при r=2..16

Это может быть полезно для:
- Оценки безопасности усечённых версий (SHA-256/k)
- Быстрой проверки гипотез о структуре SHA-256
- Нижних оценок на атакующие характеристики
