### §2. Архитектура SHA-256 (справка)

**Параметры:** 64 раунда, 8 регистров (a-h) по 32 бита, блок сообщения 512 бит = 16 слов W[0..15].

**Расписание сообщений (message schedule):**
- W[i] = sigma1(W[i-2]) + W[i-7] + sigma0(W[i-15]) + W[i-16], для i = 16..63

**Линейные операторы (над GF(2)):**
- Σ0(a) = ROTR2(a) ⊕ ROTR13(a) ⊕ ROTR22(a)
- Σ1(e) = ROTR6(e) ⊕ ROTR11(e) ⊕ ROTR25(e)
- σ0(x) = ROTR7(x) ⊕ ROTR18(x) ⊕ SHR3(x)
- σ1(x) = ROTR17(x) ⊕ ROTR19(x) ⊕ SHR10(x)

**Нелинейные функции:**
- Ch(e,f,g) = (e ∧ f) ⊕ (¬e ∧ g)
- Maj(a,b,c) = (a ∧ b) ⊕ (a ∧ c) ⊕ (b ∧ c)

**Раундовая функция:**
- T1 = h + Σ1(e) + Ch(e,f,g) + K[r] + W[r]
- T2 = Σ0(a) + Maj(a,b,c)
- Новые регистры: e_new = d + T1, a_new = T1 + T2

**Feedforward:** H[i] += register[i] после 64 раундов.

**Ch differential (точная формула):**
- Ch(e,f,g) ⊕ Ch(e⊕δ,f,g) = δ ∧ (f ⊕ g)

**Ключевое свойство:** HW(Σ1(2^k)) = 3 для всех k = 0..31.

## Часть II. Базовые теоремы

### §4. Ядро теорем

Каждая теорема приведена один раз в финальной формулировке.

**T_DEP (Декаплинг расписания).** Δe_r зависит только от W[0..r−2].
*Следствие:* Δe_17 зависит от W[0..15], т.е. от всего блока сообщения.

**T_CASCADE_UNIQUENESS.** Da_{pos+1}(v) = CONST + v mod 2^32 — линейна, наклон = 1. Единственное решение Δe_{pos+1} = 0 определяет DW_pos однозначно.

**T_ONE_CONSTRAINT.** Каждое свободное DW даёт ровно 1 степень свободы: либо δe = 0, либо δa = 0, но не оба одновременно.

**T_DE17_DECOMPOSITION.**
```
Δe17 = Da13(W[0]) + DW16
```
- Da13 = Da13(W[0]) — нелинейная биекция (image size = 2^32)
- DW16 = σ1(W[14]) + W[9] + σ0(W[1]) + W[0] — полностью определено Wang chain

**T_BARRIER_16 (Фундаментальный барьер).** Переход от 15 к 16 нулевым Δe стоит O(2^64), не O(2^32).
Доказано 5 независимыми методами:
1. Da13 — биекция → P(Δe17 = 0) = 2^{−32}
2. T_ONE_CONSTRAINT: каждое DW = одно уравнение
3. T_SCHEDULE_FULL_RANK: расписание не вносит зависимостей
4. p-адическая несовместимость: Hensel lifting не работает
5. Информационно-теоретический: 256 бит выхода из 256 бит входа

**T_SCHEDULE_FULL_RANK.** XOR-матрица расписания: rank = 512/512 над GF(2). Расписание — quasi-ideal, структурные зависимости не дают ускорения.

**T_WANG_ADAPTIVE.** Wang chain: δe[1..16] = 0 за O(1) через адаптивную коррекцию:
```
W[r] = (e1_new − d2 − h2 − Σ1(e2) − Ch(e2,f2,g2) − K[r]) mod 2^32
```
Барьер на раунде 17: P(δe[17] = 0) = 2^{−32}.

**T_2D_BIRTHDAY_NEGATIVE.** 2D-birthday (одновременный δe = 0 И δa = 0) не даёт ускорения: dim(пересечения) = 0.

**T_INFINITE_TOWER.** p-адическая башня: Sol_k существует для ВСЕХ k = 1..32. Башня бесконечна, но каждый подъём стоит O(2^k) — Hensel НЕ работает.

**T_FGH17_ZERO.** При Δe[3..17] = 0 и Wang chain: δf = δg = δh = 0 на раунде 17.
Следствие: Δe18 = Da14 + DW17 + δΣ1(e17) + δCh17.

**T_GLOBAL_BARRIER.** Глобальная система уравнений для 16+ нулей Δe имеет ранг, равный числу неизвестных. Нет свободных параметров.

**T_W1_INDEPENDENCE.** W[1] не влияет на Δe17 при фиксированном W[0] (при условии Wang chain). Нейтральность W[1] подтверждена вычислительно.

### §5. Барьерная теория

**Анатомия раунда 17:**
- Δe17 = Da13(W[0]) + DW16 mod 2^32
- Da13(W[0]) — нелинейная биекция, HW выхода uniform ≈ 16 бит
- P(Δe17 = 0) = P(Da13 = −DW16) = 2^{−32}

**Стоимость последовательных барьеров:**

| Нулевых Δe | Стоимость | Метод |
|------------|-----------|-------|
| 15 (r=3..17) | O(2^32) | Birthday search |
| 16 | O(2^64) | Барьер × 2^32 |
| 15+k | O(2^{32k}) | Каждый новый барьер × 2^32 |

**Два барьера (W[0] + W[15]):**
- W[0] контролирует δe[17] через Da13
- W[15] контролирует δe[18] через σ1(W[15]) → W[17]
- W[15] нейтрален для барьера 1 (δe[17] = const при варьировании W[15])
- Стоимость: 2^32 + 2^32 = 2^33 (при независимости)
- Экспериментально: сепарабельность ≈ 100%, корреляция ≈ 0

**Обобщённые барьеры:**
- h = e_{r−3}: пятый регистр создаёт дополнительное ограничение (T_BARRIER_H5)
- Условная независимость: P(Δe_{k+1} = 0 | Δe_k = 0) ≈ 2^{−32} (T_COND_INDEPENDENCE)
- XOR-глубина: каждый раунд добавляет 1 бит глубины в DAG (T_XOR_DEPTH)

**Информационно-теоретическая нижняя граница:**
Для любого алгоритма коллизии SHA-256: C ≥ 2^{128} = 2^{n/2}.
Доказательство: 256 бит выхода, полная диффузия, отсутствие shortcut.

### §6. p-адическая башня

**Определение:** Sol_k = {W[0] : Δe17 ≡ 0 mod 2^k} для k = 1, 2, ..., 32.

**Результаты:**
- Sol_1 существует (P = 1/2)
- Sol_k существует ∀k = 1..32 (проверено вычислительно)
- |Sol_k| ≈ 2^{32−k} (как для random map)
- Подъём Sol_k → Sol_{k+1}: P(lift) ≈ 1/2 (не Hensel!)
- Sol_32 ≠ ∅ — это и есть birthday search

**Почему Hensel не работает:**
1. F(W0) = Da13(W0) + DW16 mod 2^k
2. dF/dW0 включает carry-нелинейности всех битов
3. Якобиан имеет нетривиальное ядро на многих уровнях
4. Sol_k disconnected: решение mod 2^k не продолжается единственным образом

**Формула подъёма:** P(lift | level k) = (1 + (3/4)^k) / 2 → 1/2 при k → ∞.

**q-Pochhammer произведение:**
```
∏_{k=1}^{∞} P(lift | k) = ∏_{k=1}^{∞} (1 + (3/4)^k) / 2 ≈ 12.4829 / 2^∞
```
Гипотеза ∏(1 + (3/4)^k) = e²√3 **ОПРОВЕРГНУТА** вычислительно (разница ≈ 0.1%).

**Высота башни:** height₂(SHA-256) ≥ 32. Нижняя граница: башня бесконечна (Sol_k ≠ ∅ для всех проверенных k).

**РЕТРАГИРОВАНО:** Ранняя оценка height₂ = 6 была артефактом ошибки (| вместо ^). Исправленный код даёт height ≥ 32.

## Часть III. ALG — Арифметико-Логическая Геометрия

### §7. Формализация

**Определение 1 (Bi-Ring).** R_n = (Z/2^n, ⊕, +) — множество с двумя групповыми операциями: XOR (⊕) и сложение mod 2^n (+). Обе — абелевы группы, но дистрибутивность НЕ выполняется. Это не кольцо — это Bi-Ring.

**Определение 2 (Ψ-оператор).** Ψ(a,b) = (a + b) ⊕ (a ⊕ b) = carry(a,b) ≪ 1.
Измеряет несовместимость двух операций. Ψ(a,b) = 0 ⟺ нет переноса.

**Определение 3 (TLC-декомпозиция).** ΔH = L ⊕ Q ⊕ C:
- L = линейный слой (Σ0, Σ1, σ0, σ1, schedule)
- Q = квадратичный слой (Ch, Maj)
- C = carry-слой (mod 2^32 сложение)

**Определение 4 (Барьерная функция).** b(j) = v₂(Da_{13,j}) + 1, где v₂ — 2-адическая оценка.

**Определение 5 (Carry-энтропия).** h_carry = H(carry_i | carry_{i−1}) = −(3/4)log₂(3/4) − (1/4)log₂(1/4) = 0.811 бит.

**Аксиомы ALG:**
- A1: Операции + и ⊕ порождают ВСЕ нелинейности ARX-конструкций
- A2: Ψ(a,b) = 0 ⟺ нет переноса ⟺ совместимость
- A3: Carry — марковская цепь с P(propagation) = 3/4
- A4: Каждый раунд SHA-256 = одна итерация Ψ-машины
- A5: Security = iterations × entropy_rate × width

### §8. Главная теорема ALG

**Теорема.** Для ARX-хэша с параметрами (n бит, R раундов, w слов):

**(A) Детерминистическая фаза:** Существует адаптивная стратегия, дающая δe[1..K] = 0 для K = R_det раундов при стоимости O(1). Для SHA-256: K = 16 (Wang chain).

**(B) Барьер:** На раунде K+1 возникает барьер с P(δe[K+1] = 0) = 2^{−n}. Причина: carry делает Da_K биекцией.

**(C) Birthday cost:** C(collision) = Θ(2^{n/2}). Оптимально: никакая структурная атака не даёт суб-birthday для полного SHA-256.

**(D) Число раундов:**
```
R_opt = ⌈n / (k · h_carry)⌉ × (1 + margin)
SHA-256: ⌈32 / (5 × 0.811)⌉ × 1.39 ≈ 46 × 1.39 = 64
```

### §9. Четыре столпа безопасности ARX

| # | Столп | Механизм | Формализация |
|---|-------|----------|--------------|
| I | Алгебраическая несовместимость | + и ⊕ не дистрибутивны | Ψ(a,b) ≠ 0 для ~75% пар |
| II | Диффузия расписания | 1 бит → 567 бит | rank(Schedule) = 512/512 |
| III | Carry-барьер | Перенос → нелинейность | h_carry = 0.811 бит/бит |
| IV | Информационный предел | 256 бит выхода ← 512 входа | C ≥ 2^128 |

### §10. Карта атак: почему каждый путь закрыт

| # | Семейство | Идея | Причина неудачи |
|---|-----------|------|-----------------|
| 1 | Линейная/XOR | XOR-разности через раунды | Carry разрушает линейность на r=17 |
| 2 | Additive | Mod 2^32 разности | Da13 — биекция, P = 2^{−32} |
| 3 | Бумеранг | Середина = тождество | Нет S-box, переключение невозможно |
| 4 | Rebound | Inbound + outbound | Нет SPN-структуры |
| 5 | MITM | Разделение раундов | Schedule связывает всё: rank = 512 |
| 6 | Algebraic/SAT | Gröbner / SAT | Степень = 32, ~2^32 мономов |
| 7 | Slide | Связь ключей | Нет ключа (хэш, не шифр) |
| 8 | Lattice/LLL | Решётки на W | Rank = 16, слишком мал |
| 9 | Neural/ML | Обучение различителя | Обучается на артефактах, не генерализует |

## Часть IV. Carry Theory — Теория переноса

### §11. Carry как фундаментальный объект

**Carry-бит:** carry[i] = MAJ(a[i], b[i], carry[i−1]), carry[0] = 0.

**Carry-оператор:** C(a,b) = (a + b − a ⊕ b) / 2.

**Ключевые свойства:**
- Carry — 2-коцикл: c(a,b) + c(a+b,d) = c(a,b+d) + c(b,d) mod 2
- Бит 0 свободен от переноса (carry-free LSB)
- НЕлинейность проекции: NL = 99.94%
- Carry_i ∈ TC⁰ (пороговая функция: MAJ = «2 из 3»)
- Ext¹(R/2R, 2R) = F₂ — carry как единственный нетривиальный класс расширения

### §12. Марковская цепь переноса

**Состояние:** carry_i ∈ {0, 1}.

**Переходные матрицы** (зависят от a_i, b_i):
- T₀₀ = [[1, 0], [1/2, 1/2]] — оба бита 0
- T₀₁ = T₁₀ = [[1/2, 1/2], [0, 1]] — биты различны
- T₁₁ = [[1/2, 1/2], [0, 1]] — оба бита 1

**Стационарное распределение:** P(carry = 1) = 1/2.

**Корреляция:** ρ₁ = 1/2, ρ_k → 0 экспоненциально.

**Спектральная плотность:** S(f) = 1/(5/4 − cos 2πf). Carry — красный шум (НЧ доминирует).

**Self-healing:** Два вычисления с разными начальными carry сходятся за ~5 бит (экспоненциальное затухание с rate 1/2).

### §13. Carry-алгебра

**Метрика:** d(a,b) = v₂(a − b) — 2-адическая метрика. Carry создаёт ультраметрическое пространство.

**Кривизна:** κ ~ 10⁸. Малые изменения входа непредсказуемо меняют дальние carry.

**ANF (алгебраическая нормальная форма):**
- carry_i содержит мономы степени от 2 до i+1
- Число мономов: M(i) = 2^{i+1} − 1
- Полная степень carry₃₁ = 32

**Carry-гармонические функции:** |Harm_C| ≈ 2^{n/2}.

**Комбинаторика:** N(c) = 3^{n − transitions(c)} — число пар (a,b), порождающих carry-вектор c. Transitions = число 0↔1 переключений.

**Carry выживаемость:** P(carry propagation) = 3/4. После k бит: P(survive) = (3/4)^k.

**Carry-корреляционная алгебра:** Контекстно-зависимая. Блочная изоляция: 42–111× между блоками W-слов.

### §14. Фазовый переход

**Три фазы:**

| Фаза | Биты | Характеристика |
|------|------|----------------|
| Frozen | 0–4 | Carry предсказуем, степень мала |
| Critical | 5–6 | Фазовый переход |
| Equilibrium | 7–31 | Стационарное распределение |

**Критический бит:** k* = ⌈log₂(w)⌉ = 5 (для w = 32).
Подтверждено тремя независимыми методами:
1. ANF: степень carry_k достигает максимума ≈ k = 5
2. Марковская цепь: t_mix = 5 бит
3. Эксперимент: bias(carry_k) → 0 при k > 5

**Ляпуновская экспонента:** λ ≈ 0.85 (на раунд). Полное перемешивание ≈ 20 раундов.

**Gray zone (серая зона):**
- D(16) = 0.593, D(17) = 0.527
- Длина серой зоны = 2 раунда
- Формула: D(r) ≈ (3/4)^{k_eff(r)}

**Carry-термодинамика:**
1. Энтропия S(r) = min(NC, 2Cr) — линейная, НЕ экспоненциальная
2. Энтропия не убывает при композиции раундов
3. Полная рандомизация при S = NC
4. Прирост: +64 бит/раунд (step function)

**OCR (Oracle Convergence Round) = 5.** Подтверждено пятью независимыми выводами:
1. Carry: t_mix = 5
2. ANF degree: насыщение при k = 5
3. Статистика: bias < 2^{−16} после 5 раундов
4. Спектральный: собственные значения < 0.01 после 5 итераций
5. Информационный: H(output | input) > 31.9 бит после 5 раундов

## Часть V. Математические структуры

### §15. Bi-Ring и когерентность

**Когерентность:** μ(j) = 1/(2^j · sin(π/2^{j+1}))

**Фундаментальная когерентность:** lim_{j→∞} μ(j) → 2/π (произведение Виета).

**Несовместимость-энтропия:** H_⊥ = 0.811n бит — скрытая информация в carry.

**Bi-мультипликативная алгебра:** U = (R, +_R, ×_R, ×_S) — новый алгебраический объект, объединяющий ring и XOR структуры.

### §16. GF(2)-структура операторов

**Характеристические полиномы:**
- Σ0: char poly = (1+x)^32 — единственный блок Жордана, Σ0+I нильпотентна степени 32
- Σ1: char poly разложим, min factor degree > 1
- σ1(1) = 40960 = 5 × 2¹³ (амплификация v₂ = 13)

**Неподвижные векторы (Mv = v):**
- Σ1: {0x76d58cb8, 0x9b7e95c8} (dim = 2)
- Σ0: {0xFFFFFFFF} (dim = 1)
- Общее для всех 4 операторов: dim = 0 (нет общего инвариантного подпространства)

**Произведение Σ0·Σ1:** Фактор x²+x+1 → 2D инвариантное подпространство с векторами 0x2a811e81, 0x80955cba.

**Фильтрация Жордана (Σ0):** V_k = ker((Σ0+I)^k). Цепь: V₁ ⊂ V₂ ⊂ ... ⊂ V₃₂ = GF(2)^32.

**Результат эксплуатации:** При 64 раундах фиксированные/инвариантные векторы НЕ дают преимущества. Mean HW(ΔH) = 128 для всех типов дельт.

### §17. Тропическая алгебра барьера

**Тропическое полукольцо:** (R ∪ {∞}, min, +).

**Барьерная функция:** b(j) = j + 1 (тропически линейна).

**Тропический объём = birthday bound.** Cost(k barriers) = Σ_tropical b(i) = k(k+1)/2.

### §18. TLC — Three-Layer Collision Theory

**Три слоя каждого раунда:**
- L (Linear): Σ, σ, schedule — линейны над GF(2)
- Q (Quadratic): Ch, Maj — степень 2
- C (Carry): mod 2^32 — степень до 32

**Соотношение вкладов:** L:Q:C = 25.8 : 25.3 : 48.9 — carry доминирует.

**T1-доминантность:** T1 определяет 88.5% выходной разности.

**Oracle Distance:** Δ_oracle ≈ 2^{−26}. Для обнаружения отличия от random oracle нужно ≈ 2^52 запросов.

**Метрический тензор:** g = (m/4)(I + J) — универсален для ЛЮБОГО хэша с avalanche property. Проверено на SHA-256, SHA-3, BLAKE2s, MD5.

**Pipe Conservation Law:** (a+e)[r] = (b+f)[r+1] = (c+g)[r+2] = (d+h)[r+3]. Попарные суммы регистров сохраняются (сдвигаются по раундам).

**Универсальное уравнение хэширования:** H(N,C,r) = Random iff r ≥ 3N + 5.
- N = число бит слова, C = число столбцов
- SHA-256: 3×32 + 5 = 101 ≪ 64? Нет: реальная формула учитывает параллелизм
- Проверено: 18/18 предсказаний подтверждены

### §27. RAYON и DFS-анализ

**Отдельное направление:** RAYON = DFS с constant propagation для Circuit-SAT.

- SHA-1 с расписанием: ε = 0 (XOR barrier)
- Carry algebra: {G, K, P, ?} — четыре состояния
- Rayon equation для SHA-256: специализированный DFS solver

**Связь с основным исследованием:** XOR-барьер в RAYON-анализе = тот же carry-барьер в ALG. Фундаментальная несовместимость + и ⊕ проявляется в обоих подходах.
