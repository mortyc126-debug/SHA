# Гипотеза Birthday = Carry-DOF Bound
# Верификация и новые направления

---

## §1. Точная формулировка

**Гипотеза.** Для ARX хеш-функции H: {0,1}^{2n} → {0,1}^n:

  independent_carry_DOF(H) = n/2
  collision_cost(H) = 2^{carry_DOF} = 2^{n/2}

Birthday bound определяется числом независимых carry-степеней свободы,
а число carry-DOF = ровно половина выходных бит.

---

## §2. Верификация: откуда 128 carry-DOF?

### 2.1 Carry-interference graph SHA-256

448 сложений (7 × 64 раунда). Каждое порождает 32-битную carry-цепочку.

Зависимости (рёбра графа):
- Внутри раунда: T1-цепочка ADD₁→₂→₃→₄ (выход → вход)
- ADD₅ = d + T1: зависит от ADD₁-₄
- ADD₇ = T1 + T2: зависит от ADD₁-₄ и ADD₆
- Между раундами: через сдвиговый регистр (e→f→g→h, задержка 3 раунда)
  ADD₁[r+3] получает h = e[r] = выход ADD₅[r]

### 2.2 Подсчёт independent carry-DOF

Определение: carry-DOF = размер максимальной антицепи в carry-interference DAG,
умноженный на число бит carry на вершину.

НО: "128 = n/2" получено из грубой оценки. Проверим точнее.

Каждое сложение: 32 carry-бит. Но carry — Марковская цепочка (бит k зависит от k-1).
Число НЕЗАВИСИМЫХ carry-бит на сложение: 1 (начальное условие c₋₁ = 0 фиксировано,
вся цепочка определяется входами). Carry-бит НЕ являются степенями свободы —
они детерминированные функции входов.

ОШИБКА В ИСХОДНОЙ ОЦЕНКЕ: carry-DOF ≠ 128.

Carry-DOF = 0. Все carry полностью определены входом.
Настоящие DOF = входные биты = 512.
Выходные биты = 256. Birthday = 2^{256/2} = 2^128.

### 2.3 Что на самом деле значит "128"?

Хроматическое число χ(G_I) = 257 означает: максимальный путь в DAG = 256.
Это глубина зависимости, НЕ число степеней свободы.

Максимальная антицепь ≈ 448/257 ≈ 2 сложения. Это число ПАРАЛЛЕЛЬНО
вычислимых carry — мера параллелизма, не свободы.

128 = n/2 = 256/2 — это просто birthday formula, НЕ carry-DOF.

---

## §3. ВЕРДИКТ

**Гипотеза Birthday = Carry-DOF: НЕВЕРНА в первоначальной формулировке.**

Carry-DOF = 0 (carry детерминирован входом).
Birthday bound = 2^{n/2} — следствие теории вероятностей (парадокс дней рождения),
НЕ carry-структуры.

Совпадение 128 = n/2 — арифметическое тождество, не глубокая связь.

### Что верно:
- χ(G_I) = 257 (хроматическое число carry-графа) — ВЕРНО как структурный факт
- Carry-граф имеет глубину 256 (максимальный путь) — ВЕРНО
- Параллелизм carry ≈ 2 на раунд — ВЕРНО

### Что неверно:
- "128 independent carry-DOF" — carry не имеет DOF, он детерминирован
- "Birthday = Carry-DOF" — birthday из комбинаторики, не из carry

## §4. Переформулировка: Carry-ENTROPY

Carry не имеет DOF, но имеет ЭНТРОПИЮ (из perspective наблюдателя, не знающего M).

H(carry | output H(M)) = ?

Наблюдатель видит H(M) (256 бит). Carry = 14,336 бит, детерминированных M.
Но M неизвестно (512 бит). Из 512 бит M, 256 бит определяются из H(M)
(SHA-256 compress обратима → M однозначно восстановимо при известном H и IV...
Нет! compress: (IV, M) → H. При фиксированном IV: compress — функция 512 → 256.
Каждый H имеет ~2^256 прообразов. M НЕ определяется из H(M).)

H(M | H(M)) = 512 - 256 = 256 бит (энтропия входа при известном выходе).
H(carry | H(M)) ≤ H(M | H(M)) = 256 бит.

Carry определяется M (функционально). Значит H(carry | H(M)) ≤ H(M | H(M)) = 256.

Но carry — функция M, не биекция. Разные M могут давать одинаковый carry.
H(carry | H(M)) ≤ 256 бит. Точное значение: зависит от carry-отображения.

Из carry-web: ~2^18 carry-профилей (overflow). Полный carry: ~2^{512} конфигураций
(по числу входов). H(full carry | H(M)) ≈ 256 бит.

Carry-энтропия = 256 = n. Birthday = 2^{n/2} = 2^{128}.

Birthday = 2^{carry_entropy / 2}. Но carry_entropy = n = output_size. Тривиально.

## §5. Новый вопрос: ЭФФЕКТИВНАЯ carry-энтропия

Carry-энтропия = 256 — это МАКСИМАЛЬНАЯ (worst-case).
А ЭФФЕКТИВНАЯ carry-энтропия (с учётом carry-корреляций)?

Carry-биты коррелированы (Марковская цепь, self-healing).
14,336 carry-бит, но корреляции уменьшают эффективное число.

Эффективная энтропия: H_eff(carry) = Σ H(c_i | c_{i-1}, c_{i-2}, ...)

Из Markov: H(c_i | c_{i-1}) = H(MAJ(a,b,c_{i-1}) | c_{i-1})
При случайных a,b: H = H(3/4) = 0.811 бит (при c_{i-1}=0) или H(3/4) (при c_{i-1}=1).
Среднее: 0.811 бит на carry-позицию.

H_eff = 14,336 × 0.811 = 11,627 бит.

Но наблюдатель видит только 256 бит выхода.
H(carry | output) ≤ min(H_eff, H(M|output)) = min(11627, 256) = 256.

Снова 256. Bottleneck — не carry, а OUTPUT SIZE.

## §6. ФУНДАМЕНТАЛЬНЫЙ ИНСАЙТ

Birthday bound = 2^{n/2} определяется ВЫХОДНЫМ РАЗМЕРОМ n, не carry-структурой.

Carry-структура влияет на КОНСТАНТУ (не экспоненту):
- Идеальная функция: birthday = 2^{n/2}
- SHA-256: birthday = 2^{n/2} × (1 ± O(2^{-26}))

Oracle Distance 2^{-26} — единственный измеримый эффект carry-структуры на стоимость.

Экспонента 128 = n/2 = ИНВАРИАНТ, не зависящий от carry.

Это объясняет почему ВСЕ наши атаки возвращают 2^{128}:
экспонента определяется n, а carry влияет только на мультипликативную константу.
