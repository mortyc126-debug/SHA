# П-74: Три идеи — мост к результатам Li et al. (Март 2026)

## Контекст

Лучший публичный результат 2024: SFS-коллизия для 39 шагов SHA-256
(Li et al., EUROCRYPT 2024). Практическая коллизия: 31 шаг (ASIACRYPT 2024).
Их метод: дифференциальный путь через MILP + SAT-солвер.

Задача П-74: найти где наши результаты серии П-1..П-73 пересекаются
с их подходом и могут его усилить.

---

## Идея 1: T_DEP как constraint для MILP

**Результат:** T_DEP даёт точное ограничение на активность переменных.

В классическом MILP для поиска дифференциального пути переменная
`a[r][j]` кодирует активность слова W[j] в раунде r. Без дополнительных
знаний: 272 потенциально активных пары (r,j) для r=1..17.

С T_DEP: `De_r` зависит от `W[j]` только при `j ≤ r-2`. Это **жёсткий
constraint**: `a[r][j] = 0` обязательно при `j ≥ r-1`.

```
Сокращение: 272 → 136 активных переменных (50%) для раундов 1..17.
```

**Практическая польза:** умеренная. 50% — значительное теоретическое
сокращение, но MILP для 28+ раундов имеет тысячи переменных, и
T_DEP применим только к первым 17.

**Концептуальная польза:** объясняет почему все известные пути начинаются
с малого числа активных входных слов — это не случайность, а T_DEP.

## Идея 2: GF2 rank=5 и free-start

**Результат:** ключевое открытие — SFS коллизия = наш free-start GF2.

**Стандартная модель r=28:** rank(J_{28×16}) = 16 < 28. Im(J) мал,
P(путь существует в GF2) = 2^{−12} ≈ 0.02%.

**Free-start r=28:** rank(J_{28×28}) = 28 (верифицировано на 20 сидах,
min=max=28). Im(J) полный → P = 1.0 гарантированно.

```
КЛЮЧЕВОЕ НАБЛЮДЕНИЕ:
SFS-коллизия учёных = free-start GF2 задача + подъём до точности.

Наш T_GF2_BIJECTION (П-61) математически эквивалентен тому что
делает SAT-солвер на первом уровне в SFS атаке.
```

**Где ломается:** p-адический подъём с k=1 до k=32. По T_FREESTART_NONSMOOTH
(П-68A), добавление 2·eⱼ к решению нарушает бит 0 инварианта.
Именно здесь учёным нужны SAT+CAS, а не просто GF2.

## Идея 3: T_NONLINEAR_DIFFUSION_RATE → структура пути

**Результат:** измеренный diffusion rate объясняет форму всех известных путей.

| Раунд | avg HW(De_r) при HW(DW)=1 | % активных бит |
|-------|--------------------------|----------------|
| 1 | 0.10 | 0.3% |
| 3 | 1.90 | 5.9% |
| 5 | 3.56 | 11.1% |
| 10 | 8.77 | 27.4% |
| 17 | 15.84 | 49.5% |
| 19 | 16.04 | **50% = лавина** |

Переход от sparse к полной лавине: **4-19 раундов** (не 3, как казалось —
P-адический барьер на k=2, но HW-лавина медленнее).

**Constraint на форму пути:**
- Раунды 1-3: sparse (≤ 2 бита) — T_DEP запрещает больше
- Раунды 4-18: нарастание — стоимость пути растёт экспоненциально
- Раунды 19+: ~50% активных — дальше "бесплатно" (уже случайные)

Это объясняет почему пути типа Mendel: sparse вход → dense середина →
cancellation на выходе. Cancellation возможна потому что диффузия
симметрична — если войти правильно, можно выйти с малым весом.

## Дорожная карта к 28-шаговой коллизии

```
Шаг 1 (выполнен): CASCADE_6ZEROS для e-регистра, r≤17 (П-72/П-73)
Шаг 2 (следующий): расширить CASCADE на все 8 регистров, r≤17
  Стоимость: 8 × 6 × 2^32 = 2^37.5 — реалистично за ~день
Шаг 3: расширить на r=18..28 (free-start)
  Здесь начинается настоящая криптоаналитическая задача
Шаг 4: найти дифференциальный путь для r=28 с нашими constraints
  Это требует MILP + наши T_DEP constraints
```
