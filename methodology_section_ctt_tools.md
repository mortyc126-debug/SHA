## 0. КОНТЕКСТ СЕРИИ

Серия VI начинается после П-1300 (H[7]-коллизия за O(2^17.5), MLB алгоритм, барьер r=17 подтверждён 5+ методами). Цель — построить **рабочий инструментальный стек** для SSB-Wang и сформулировать **новую математику** (CTT) как основу для будущих атак.

## 1. CARRY TENSOR THEORY (CTT) — П-1301..П-1320

### Центральное уравнение CTT-MAIN

```
ΔF(W, ΔW) = J_W · ΔW  ⊕  N(W, ΔW)

Коллизионное условие:
  J_W · ΔW = N(W, ΔW)    [CTT-MAIN]

256 уравнений над 𝔽₂
1024 переменных (W + ΔW)
768 степеней свободы (глобально)
```

### Теоремы CTT (доказаны экспериментально)

| Теорема | Статус | Содержание |
|---------|--------|------------|
| **T_CTT_MAIN** | ✓ | ΔF = J·ΔW ⊕ N(W,ΔW), rank(J)=32 |
| **T_N1_ZERO** | ✓ | 1-бит ΔW: нелинейность N₂ точно 0 |
| **T_N2_BASELINE** | ✓ | Квадратичная нелинейность N₂≈127 бит (измерено) |
| **T_QUIET_POINTS** | ✓ | min Q=94 через SA, тихие точки существуют |
| **T_Q_PERP_DE** | ✓ | Q(W,ΔW) и De_r статистически независимы (p=0.98) |
| **T_Q_PERP_SAC** | ✓ | Q и лавинный критерий независимы (r=−0.01) |

### Объект Q(W, ΔW) — поверхность нелинейности

```
Q(W, ΔW) = HW(N₂(W, ΔW))

Свойства (измерены):
  E[Q]         = 128 бит (baseline)
  min Q        = 94  (SA, 30 рестартов)
  Q глобальный min = 96 (SA, 30×SA)
  Хессиан      = indefinite (3+/3-/2·0)
  SAC ↔ Q      = независимы (r=−0.012)
  Бассейн min  = 0.1% пространства
  Барьеры      = высокие (+34)
```

**Вывод:** Q — многомодальная невыпуклая поверхность. Каждый ΔW имеет свои W* с минимальным Q. Тихость не нарушает лавину.

## 2. НОВЫЕ УРАВНЕНИЯ ДЛЯ КОЛЛИЗИИ — П-1321..П-1340

### Полная система (568 уравнений, 1024 переменных)

**Блок I — Расписание [S.r], r = 16..63 (48 уравнений):**
```
δW[r] = δs₁[r−2] + δW[r−7] + δs₀[r−15] + δW[r−16]  mod 2³²

Новое разложение [N.S.r]:
  δW[r] = Δlin[r] ⊕ Δcarry[r]
  Δlin   — линейно в GF(2)
  Δcarry — carry-тензор расписания (~9 бит/слово)
```

**Блок II — Раундовые функции [T1/T2.r] (128 уравнений):**
```
Новое [N.T1.r]:
  T₁[r] = L_T1[r] ⊕ 2·Carry_T1[r]
  L_T1  = h ⊕ S₁(e) ⊕ Ch ⊕ K[r] ⊕ W[r]   (линейно в GF(2))
  δT₁[r] = δL_T1[r] ⊕ 2·δCarry_T1[r]
  
  Измерено: HW(δL_T1) ≈ 16 бит, HW(δCarry_T1) ≈ 15 бит
```

**Блок III — Переходы [R.r] (384 уравнения):**
```
  δe[r+1] = δd[r] + δT₁[r]
  δa[r+1] = δT₁[r] + δT₂[r]
  δb,δc,δd,δf,δg,δh — сдвиги
  Граничные: δa[0]=...=δh[0]=0
```

**Блок IV — Условие коллизии [C.i] (8 уравнений):**
```
  δH[i] = 0, i=0..7
  ⟺  δa[57..64] = 0   AND   δe[57..64] = 0
  (через структуру регистровых сдвигов)
```

**Новая форма через CTT:**
```
[CTT-COL]:
  ∑_r [δS₁(e[r]) ⊕ δCh[r] ⊕ δW[r]]  =  ∑_r Carry_r
  Левая часть — линейна в GF(2)
  Правая часть — нелинейна (измеряема)
```

### Уравнение барьера расписания

```
[N.BARRIER]:
  Wang-нули ≥ k  ⟺  r_first(W,ΔW) ≥ k+1

[N.SCHED.16]:
  δs₁(W[14]) + δW[9] + δs₀(W[1]) + δW[0] = 0  mod 2³²
  
  Newton-решение:
    δW[0]_{n+1} = δW[0]_n ⊕ actual_δW[16]
  Сходимость: 100% за ≤50 шагов (измерено)
```

## 3. ДИАГНОЗ WANG — П-1341..П-1360

### Ключевые исправления модели

**T_DE1_CONSTANT (новая):**
```
При аддитивном ΔW[0] = c:
  T₁[0] = IV[7] + S₁(IV[4]) + Ch(IV[4..6]) + K[0] + W[0]
  T₁'[0] = IV[7] + S₁(IV[4]) + Ch(IV[4..6]) + K[0] + W[0] + c
  
  => De_1 = T₁[0] ⊕ T₁'[0] = c  (КОНСТАНТА!)
  
Следствие: искать De_1=0 при аддитивном ΔW ≠ 0 бессмысленно.
Wang ищет SHA(W) = SHA(W+ΔW) — полную коллизию за 64 раунда.
```

**Правильная модель Wang:**
```
1. ΔW — аддитивный дифференциал (не XOR!)
2. De_r ≠ 0 для r=1..16 — это ДИФФЕРЕНЦИАЛЬНЫЙ ПУТЬ
3. Sufficient Conditions (SC) на W гарантируют конкретные De_r
4. Коллизия: накопленные разности сокращаются после 64 раундов
5. Без SC: P(SHA(W)=SHA(W+ΔW)) = 2^{-256} — нереально
```

### Структура DeltaTracer (измерена)

```
r=1: HW(De_r) = 1.0  (линейно, управляемо!)
r=2: HW(De_r) = 6.9
r=3: HW(De_r) = 13.8
r=9: HW(De_r) = 16.0 (полная лавина)

Окно управляемости: r = 1..2
```

## 4. СЕМЬ ИДЕЙ ДЛЯ ИНСТРУМЕНТОВ — П-1361..П-1380

| # | Идея | Оценка | Статус |
|---|------|--------|--------|
| 1 | Carry Split: T₁=L⊕2C → SHA линейна | 1/10 | Частично (1-2 раунда) |
| 2 | Carry-Space: (val, mask) → линейная часть | 5/10 | Требует проверки |
| 3 | MITM по carry → 2^128 | 4/10 | state[32] ранг=256 |
| 4 | Carry Signature Birthday | 6/10 | carry не предсказывает H |
| 5 | Soft Carry / STE | 7/10 | soft_add ошибка 50% |
| 6 | Обратный расчёт ZC⁻¹ | 8/10 | a_prev совпадает ✓ |
| **7** | **Трёхзонная система** | **9/10** | **Перспективна** |

### Трёхзонная декомпозиция

```
Зона A [r=0..15]:  W[r] входят напрямую → квазилинейна
Зона B [r=16..48]: барьерная зона, MITM или birthday
Зона C [r=49..63]: ОБРАТИМА детерминированно!

ZC⁻¹: target=0 → state[49] за O(1)
  a_prev = b_next, b_prev = c_next, ...
  T2_prev = S₀(a_prev) + Maj(a_prev, b_prev, c_prev)
  T1_prev = a_next − T2_prev
  d_prev  = e_next − T1_prev
```

## 5. SSB-WANG АЛГОРИТМ — П-1381..П-1400

### Спецификация

```
SHA-Specific Birthday with Wang Structure (SSB-Wang)

Теоретическая стоимость: O(2^44) vs стандартный O(2^128)

Шаг 1 [WangPhase]:   O(2³³)  — найти W_base: SHA(W)=SHA(W+ΔW)
Шаг 2 [NeutralExpand]: O(1)  — W[12,13,15] дают 2⁹⁶ вариантов
Шаг 3 [MLO Birthday]: O(2^44) — birthday внутри Sol_Wang
```

### Калиброванные данные (измерено в сессии)

| Параметр | Значение | Метод |
|----------|----------|-------|
| |Sol_Wang| | ≥ 2^104 | П-97 |
| Нейтральных бит | 96 (W[12,13,15]) | П-94 |
| Сжатие H[4] при H[7]-коллизии | 8 бит | MLO C-код, 2000 пар |
| MLO реальная стоимость | O(2^112) | 2000 пар, H[0..6]≈16 |
| WangPhase скорость | 3670M/s | 4 потока C |

**Честная оценка:** оценка O(2^44) предполагала сжатие по всем 8 словам H. Реально сжатие только H[7]=16 бит, остальные ≈0. MLO даёт O(2^112), не O(2^44). Для O(2^44) нужна Wang-структура (Sol_Wang).

## 6. ACB — ADAPTIVE CARRY BIRTHDAY — П-1401..П-1415

### Три фазы исследования

**Фаза 1 — Overflow carry:**
```
P(c_T1[r+1] = c_T1[r]) = 0.984 (сильная корреляция)
P(все 64 carry=1) = 0.563 (большинство W!)
carry overflow теряется mod 2³² → структурно бесполезен
```

**Фаза 2 — Внутренний carry:**
```
avg carry_total = 3809, std = 58
Бакеты 3950-3969: HW=122 при N=30 (артефакт малой выборки)
N=10K: HW=128.2 (шум, нет сжатия)
```

**Фаза 3 — Корреляция:**
```
max|r|(carry_total, H) = 0.052 при пороге 0.018 (2.8σ)
carry_total не предсказывает H
```

**Вывод:** SHA-256 эффективно перемешивает carry. ACB как независимый инструмент не работает. Ускорение < 1 бит. ACB полезен только как диагностика (CarryScope).

## 7. ИНСТРУМЕНТАЛЬНЫЙ СТЕК v3 — П-1416..П-1420

### Рабочие C-инструменты

**wang_phase_c.c** — WangPhase v1 (баг)
```
Баг: искал De_1=0 при аддитивном ΔW — невозможно по определению
Скорость: 16809M/s (корректна)

```

**wang_v2.c** — WangPhase v2 (исправленный)
```
Цель: прямой поиск SHA(W) = SHA(W + 0x80000000)
Без SC: P = 2^{-256} → практически нереально
Статус: теоретически правильный, практически требует SC
```

**sa_wang_final.c** — SA-Wang (рабочий)
```
Цель: min HW(SHA(W) XOR SHA(W + 0x80000000))
Метод: SA + population crossover (100 рестартов)
Результат: HW=80 за 2 мин на 4 потоках
           HW=128 → 99 → 86 → 80 (стабильное снижение)
Компиляция: gcc -O3 -march=native -pthread -o sa_wang sa_wang_final.c
```

**tools_final.c** — Toolbox (рабочий)
```
Инструменты:
  mode 1: MLO Birthday   — 2000 пар, H[7] сжатие 16 бит
  mode 2: SA Gradient    — HW=85-86 из 256
Компиляция: gcc -O3 -march=native -o tools_final tools_final.c
```
