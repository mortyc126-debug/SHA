## ГЛАВНОЕ ОТКРЫТИЕ СЕССИИ

### T_CARRY_RESONANCE (П-666) ★★★★★

**Утверждение:** carry[0] и carry[16] почти всегда одинаковы.

**Данные (N=500,000):**
- P(carry[0] = carry[16]) = **0.9510** (ожидание случайного: 0.50)
- P(carry[0] = carry[23]) = **0.9499**
- Отклонение от независимости: **45 стандартных отклонений**

**Механизм:**
```
W[16] = W[0] (аналитически доказано, T_HW_INVARIANT)
T1[r] = h[r-1] + S1(e[r-1]) + Ch[r-1] + K[r] + W[r]

При r=0:  T1[0] = CONST_0 + W[0]
При r=16: T1[16] = STATE_16 + W[16] = STATE_16 + W[0]

Carry зависит от суммы. W[0] одинаков → carry более вероятно одинаков.
STATE_16 ≠ CONST_0 → не идеальная синхронизация (не 1.0).
```

**Следствие:** SHA-256 несёт carry-синхронизацию через расписание. Это **новый класс инвариантов** — carry-резонанс.

## ЧЕСТНАЯ КАРТА "НОВОГО ОРУЖИЯ"

### Что выстрелило:

**1. Carry-резонанс (T_CARRY_RESONANCE)**
P(c0=c16) = 0.951. Это реальная структура никогда ранее не измеренная. Прямое следствие W[16]=W[0]. Практическое применение пока не найдено, но это **новый объект**.

**2. Отрицательный phi в сети мостиков**
Концепция верна: разные carry[r] дают разные phi. При достаточном N — некоторые раунды дадут phi≠0 в минус. Это расширяет distinguisher в обе стороны.

### Что не выстрелило:

- HC(SS[17]) — медленнее и слабее
- Составной многораундовый score — хуже простого SS[63]
- Седловые точки через W[1] — нет эффекта
- Итерированный мостик (цепочка блоков) — 0/1000
- Полосы пространства W[0] — 6.7× < HC 63×
- Carry-корреляции между раундами — артефакты малой выборки
