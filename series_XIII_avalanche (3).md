## 1. Архитектура лавины

### 1.1 Структурная модель

SHA-256 лавина — это **не однородное распространение**. Это два независимых механизма:

```
СДВИГОВЫЙ КОНВЕЙЕР (детерминированный, r=1.000):
  δb[r+1] = δa[r]
  δc[r+1] = δb[r]
  δd[r+1] = δc[r]
  δf[r+1] = δe[r]
  δg[r+1] = δf[r]
  δh[r+1] = δg[r]

ДВОЙНОЙ НЕЛИНЕЙНЫЙ УЗЕЛ (единственная реальная нелинейность):
  δa[r+1] = δ(T1 + T2)  ← через S0, Maj, S1, Ch, W[r]
  δe[r+1] = δ(d + T1)   ← через d и T1
```

**EXP6 (Inter-register Causality):** 6 из 8 причинных связей имеют корреляцию r=1.000 — это алгебраические тождества, не статистика. Только δa и δe — настоящие нелинейные узлы.

### 1.2 Три фазы лавины (EXP5: Velocity)

| Фаза | Раунды | Новых бит | Потерянных бит | Нетто |
|------|--------|-----------|----------------|-------|
| Рождение | r=0 | +1.95 | 0 | +1.95 |
| Взрыв | r=1 | +12.3 | −0.87 | +11.4 |
| Насыщение | r≥3 | ≈8.0 | ≈8.0 | ≈0 |

**Точка насыщения: раунд 3.** После этого лавина осциллирует в равновесии.

### 1.3 Симметрия по регистрам (EXP2: Symmetry)

Регистры насыщаются с задержкой по конвейеру:

```
δa, δe  → насыщение на r=2  (лидеры)
δb, δf  → r=3
δc, δg  → r=4
δd, δh  → r=5  (хвост конвейера)
```

К раунду 6 — полное выравнивание. Spread между регистрами на r=1: **13.4 бита** (a=13.4 vs c=0).

## 7. Двойной нелинейный замок (Q15, Q16)

### 7.1 Концепция

Из EXP6: два нелинейных узла — δa и δe. Если **одновременно** δa[r]=0 и δe[r]=0:
- δb[r+1]=δa[r]=0 (тождество)
- δf[r+1]=δe[r]=0 (тождество)
- лавина "замораживается" в конвейере

### 7.2 Аналитическое условие (Q16)

Уравнения двойного замка на раунде r:
1. `T1_0 + T2_0 ≡ T1_1 + T2_1 (mod 2³²)` → δa[r+1]=0
2. `d0 + T1_0 ≡ d1 + T1_1 (mod 2³²)` → δe[r+1]=0

Из (1): `δT2[r] ≡ −δT1[r]`  
Из (2): `δd[r] ≡ δT1[r]`  
Подставляя: `δT2[r] ≡ −δd[r]`

Через сдвиговый конвейер: `δd[r] = δc[r−1] = δb[r−2] = δa[r−3]`

**Итоговое условие:**

```
╔════════════════════════════════════════════════════╗
║  δ(S0(a[r]) + Maj(a[r],b[r],c[r])) ≡ −δa[r−3]   ║
║                              (mod 2³²)             ║
╚════════════════════════════════════════════════════╝
```

### 7.3 Физическое объяснение Wang SC

**Теорема T_WANG_SC_GEOMETRY:**  
Wang Sufficient Conditions задают `δa[r] ≈ 0` для r=0..17.  
Это автоматически удовлетворяет условию двойного замка на раунде r+3.  
Wang SC = геометрическое условие замораживания лавины через нелинейные узлы.

Это первое **физическое объяснение** Wang SC через структуру SHA-256, а не эмпирический подбор.

## 12. Ключевые теоремы серии XIII

**T_AVALANCHE_STRUCTURE:** Лавина SHA-256 = сдвиговый конвейер (6 тождеств) + двойной нелинейный узел (δa и δe). Только 2 из 8 причинных связей нелинейны.

**T_DECAY_LAW:** Автокорреляция лавины: C(k) = 1.37·exp(−k/1.80). Время памяти τ=1.8 раунда. Обнуление при k=4.

**T_SATURATION:** Точка насыщения лавины = раунд 3. После этого new_bits=lost_bits≈8 (осциллятор).

**T_OPTIMAL_DELTA:** Wang ΔW[0]=0x80000000 занимает 30-е место из 32 по E[hw59]. Оптимальный однобитный: b6 (−0.54 бита). Оптимальный многобитный: находится SA (hw59=79).

**T_THEORETICAL_BOUND:** std(HW[59])=8.0 бит. За M итераций E[min]≈128−8·√(2·ln(M)). Рекорд hw59=79 = теоретический предел для M=10⁷.

**T_WANG_SC_GEOMETRY:** Wang SC ↔ условие двойного нелинейного замка: δ(S0(a[r])+Maj)≡−δa[r−3] (mod 2³²). Wang SC замораживает лавину через нелинейные узлы.

**T_DNA_WORD:** ДНК лавины определяется номером слова W[k], а не позицией бита. L2(W[0][bX], W[0][bY])≈1-3, L2(W[0][bX], W[7][bY])≈40.

Бинарное: 0100_0100_1010_0000_0011_0010_0100_0011

### 13.4 Семейство оптимальных ΔW (Q21)

Wang ΔW (HW=1) — в нижней трети по всем метрикам.

**Вывод:** Wang выбрал ΔW[0] по соображениям дифференциального пути (Zone-1 SC), не оптимизируя hw59. Для задачи минимизации hw59 без Wang SC — любой ΔW с HW≥6 лучше Wang.

### 13.5 Теорема T_OVERBURN

```
T_OVERBURN:
  Дифференциал ΔW создающий HW(δa[1]) > 10
  даёт E[hw59] < 128 (статистически лучше Wang).
  
  Оптимум при HW(δa[1]) ≈ 21.
  
  Физика: быстрое насыщение → больше раундов в равновесии
  → SA находит глубокие минимумы хвоста распределения.
  
  Следствие: режим "свободного ΔW" в SA принципиально
  отличается от Wang-режима и не ограничен Wang SC.
```

### 13.6 Почему SA hw59=79 = теорграница

std(hw59)=8.0 одинакова по всем HW(δa1).  
SA с free-ΔW за ~5.5M итераций нашёл 5.6σ событие.  
По теоретической формуле: за M=10⁷ E[min]≈82.5, но SA эффективнее случайного поиска — находит структуру в ландшафте.

**Q32:** При hw59<90 — E[HW(δreg[r])] = 0 для всех регистров и всех раундов r=58..64. Это не статистический эффект: когда hw59 мал, лавина буквально схлопывается к нулю в зоне r=55..59.

**Q35 — ключевое открытие:** Обратная волна. При hw59<90:
- r=59: hw_total=89 (определение — мало)
- r=60: hw_total=106 (+17, лавина начинает расти)
- r=61: 127 (+21)
- r=63: 138 (+11, выше нормы 128)
- r=64: 138 (финальное состояние ВЫШЕ нормы)

При hw59>110:
- r=59..64: hw_total ≈ 128 стабильно

**Теорема T_REBOUND:** Когда лавина схлопывается к малому hw59 в раунде 59, Zone-3 (r=60..63) создаёт обратный отскок — hw_total вырастает до 138, что делает inv=0 неизбежным. Нет способа иметь малый hw59 и малый hw63 одновременно через Zone-3. Это фундаментальное геометрическое ограничение, не статистическое.

**Причинная цепочка:**  
малый hw59 → лавина схлопнулась к r=59 → в Zone-3 нет ничего кроме конвейера + нелинейных узлов → конвейер разгоняет дифференциал обратно → hw63 большой → inv=0.

**Следствие для атаки:** Классический путь (Wang SC → Zone-2 → Zone-3 ZC⁻¹) работает потому что Wang SC ЗАМОРОЗИЛ лавину в Zone-1, и Zone-2 никогда не входит в насыщение. С free-ΔW без Wang SC — лавина насыщается к r=10 и Zone-3 всегда видит насыщенную лавину.

**Новый путь (Q36, следующий):** Нужен дифференциал который создаёт малый hw59 И при этом входит в Zone-3 с малым hw_total — т.е. лавина должна схлопываться постепенно через Zone-2, а не резко в r=59. Это требует не free-ΔW, а структурного дифференциального пути через Zone-2.
