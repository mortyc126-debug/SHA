## T_GRAY_ZONE — главный результат

### Формулировка

```
При δW[j]=1 (однобитовый) и барьере barrier=j:

P(δcarry[j]=0)   = 1.000  (до барьера включительно)
P(δcarry[j+1]=0) = 0.593  (серая зона, раунд 1)
P(δcarry[j+2]=0) = 0.527  (серая зона, раунд 2)
P(δcarry[j+3]=0) = 0.500  (случайность восстановлена)
```

Значения 0.593 и 0.527 — универсальные для SHA-256,
не зависят от j.

### Значение для атаки

Раунды серой зоны стоят дешевле в вероятности характеристики:

```
Обычный раунд:        log2(1/0.5)  = 1.000 бит
Серая зона раунд 1:   log2(1/0.593) = 0.754 бита  (-0.25)
Серая зона раунд 2:   log2(1/0.527) = 0.924 бита  (-0.08)
```

Суммарный выигрыш на 2 раундах серой зоны: 0.33 бита.

## T_SOFTDET_HW16 — устойчивость детерминизма

### Результат

```
P(δcarry=0) при δstate=0:
  HW(δW)=1:   0.987
  HW(δW)=4:   0.963
  HW(δW)=8:   0.943
  HW(δW)=16:  0.935
```

Детерминизм устойчив. Даже при полностью случайном δW
(HW=16) вероятность остаётся 0.935.

**Аналитически:** P(δcarry=0) = 1 - P(T1+T2 = граница ± δ/2).
При больших δ зона граница расширяется линейно с HW(δW).

## Профиль детерминизма SHA-256

```
Раунды:   1    2    ...  j   j+1   j+2   j+3  j+4...
Профиль:  1.0  1.0  ...  1.0  0.59  0.53  0.50  0.50
          ←────── барьер ─────→ ←серая→ ←────── случайно
```

Это универсальный профиль для любого однобитового δW[j].

**П-27:** SHA-256 имеет мягкий переход от детерминизма к случайности
через серую зону длиной ровно 2 раунда.

# Новые гипотезы H-22..H-26

---

## H-22: СТЕКИРОВАНИЕ СЕРЫХ ЗОН

**Основа:** T_GRAY_ZONE — серая зона длиной 2 после барьера.
T_DOUBLE_GRAY — δW[14]+δW[15] даёт серые зоны в r=15,16.

**Гипотеза:** δW[13]+δW[14]+δW[15] (тройка):
- barrier = 13 (закон минимума)
- серая зона: r=14,15
- НО: вход дифференциала в r=14 создаёт свою серую зону в r=15,16
- Итого: r=14,15,16 — три серых раунда подряд?

**Если да:** характеристика через три серых раунда стоит
3×0.85 бита вместо 3×1.0. Суммарно -0.45 бита.

---

## H-23: СЕРАЯ ЗОНА ДЛЯ МНОГОБИТОВОГО δ

**Основа:** T_SOFTDET_HW16 — даже HW=16 даёт P=0.935.

**Гипотеза:** при δstate[barrier+1] с HW=2 (серая зона)
дифференциал не "взрывается" сразу — он остаётся малым ещё раунд.

Конкретно: P(HW(δstate[barrier+2]) ≤ 4) > P_random?

Это бы означало что серая зона имеет "малый размер δstate"
как дополнительное свойство — не только малую P(carry).

---

## H-24: ОПТИМАЛЬНЫЙ ДИФФЕРЕНЦИАЛ ДЛЯ РЕКОРДА

**Прямая атака на рекорд.**

Мы знаем:
- barrier(δW[j]) = j
- Серая зона: 2 раунда после барьера
- T_DW_ODD_ZERO: нули в W[17,19,21] при δW[0]

**Гипотеза:** оптимальный дифференциал для максимального числа
раундов — это δW[15] с message modification в раундах барьера,
который использует серую зону (r=16,17) и нули δW (r=17,19,21).

Конкретно: построить характеристику:
```
r=1..15:  δstate=0  (барьер, бесплатно)
r=16:     δstate мало, P=0.59
r=17:     δW[17]=0 (T_DW_ODD_ZERO) + серая зона P=0.53
r=18..20: обычная характеристика
```

Это потенциально лучшая характеристика на 20 раундов из доступных.

---

## H-25: НИЖНЯЯ ГРАНИЦА СЕРОЙ ЗОНЫ

**Теоретическая.**

P(δcarry[barrier+1]=0) = 0.593.
Это больше 0.5 на 0.093.

**Гипотеза:** 0.593 = 1/2 + 1/2 × P(HW(δstate[barrier+1]) ≤ k)
для некоторого k.

Если верно — это даёт аналитическую формулу для серой зоны
через распределение HW(δstate).

---

## H-26: ЦЕПОЧКА СЕРЫХ ЗОН ДЛЯ δW[8]

**Основа:** δW[8]=1 даёт barrier=8 и 7 нулей в δW[16..22].

**Гипотеза:** при δW[8]=1:
- Серая зона в r=9,10 (сразу после барьера)
- Нули δW[16..22] создают ВТОРУЮ серую зону в r=16..18
- Итого: две серые зоны разделённые активными раундами

Это "двойная серая зона" — новый паттерн.
Характеристика через неё стоит дешевле в двух местах.
