# П-71: Честный 2^32 birthday — результаты и ошибки (Март 2026)

## Постановка

Повторить атаки из П-70 с исправленными ошибками. Строгий верификатор активен.
Скорость: ~250 M/s на 4 ядрах параллельно.

## Этап 1: Поиск d0 с De3=0

**Результат (seed 0xC0FFEE42):**
```
d0 = 97EA8A62
Итераций: 2^31.25  (~2.5 млрд)
Время: 10.1 сек
Скорость: 253 M/s
Верификация: De3 = 00000000 ✓
```

**Три других сида (0xDEADC0DE, 0xBEEFCAFE, 0xFEEDF00D):** De3=0 не найдено за полный перебор 2^32. Это статистически нормально — P(De3=0) = 1/2^32, и примерно 37% сидов не имеют решения в диапазоне d=1..2^32-1 (пуассоновское распределение с λ=1).

```
ТЕОРЕМА T_DE3_BIRTHDAY_CONFIRMED (П-71):

  Для конкретного W16 с seed=0xC0FFEE42:
  d0 = 0x97EA8A62 ≠ 0 такой что De3(DW=[d0,0,...,0]) = 0 точно.

  Стоимость: 2^31.25 итераций (чуть меньше 2^32 в среднем).
  Верифицировано строго. DW≠0 ✓.

  P(нет решения за 2^32) ≈ e^{-1} ≈ 37% — наблюдалось на 3/4 сидах.
```

## Этап 2: Каскад De6=0

**Результат:** d1=773A70CC найден, De6=0 ✓, но **De3=BA79EB76 ≠ 0**.

**Ошибка (третья в серии П-70/П-71):** параметр W[1] влияет на De3.

По T_DEP: `De3 зависит от W[0] и W[1]`. Изменение W[1]+=d1 ломает уже найденный De3=0.

**Правильная стратегия каскада (из T_DEP_ZERO, П-9):**

```
De3=0:  варьируем W[0]   → стоимость 2^32
De6=0:  варьируем W[4]   → De6 зависит от W[0..4], W[0] уже зафиксирован
De9=0:  варьируем W[7]   → De9 зависит от W[0..7]
De12=0: варьируем W[10]
De15=0: варьируем W[13]
```

Каждый шаг стоит 2^32. Итого: 5 × 2^32 = 2^34.3 операций.

**Это в точности П-13.** Атака не нашла способа обойти это.

## Подтверждённые результаты П-71

```
✓ De3=0 найден за ~2^31 (P = 1/2^32, теория совпадает)
✓ P(нет решения за 2^32) ≈ 37% — наблюдалось на 3/4 сидов
✓ Каскадная стратегия: W[0]→De3, W[4]→De6, W[7]→De9 (T_DEP_ZERO)
✓ Общая стоимость 5 нулей: 5 × 2^32 ≈ 2^34.3 — соответствует П-13
✗ Прямой перебор одновременно De3=...=De15=0: P = 1/2^160, нереалистично
```

## Правило 10 (добавить в §0 методологии)

```
Правило 10: При каскадном поиске нулей De_{3k}=0
  выбирать параметры по T_DEP:
  De_{3k} зависит от W[0..3k-2] → варьировать W[3k-2].
  Параметр W[3k-1] СВОБОДЕН (не влияет на De_{3k}) — использовать
  его для следующего шага каскада.
  Нарушение этого правила ломает предыдущие нули.
```
