## 2. ОБОЗНАЧЕНИЯ И ФУНКЦИИ SHA-256

### Символы

| Символ | Определение |
|--------|-------------|
| `De_r` | `(e_r(M') − e_r(M)) mod 2^32` — ADD-разность e-регистра на раунде r |
| `Da_r` | `(a_r(M') − a_r(M)) mod 2^32` — ADD-разность a-регистра |
| `δX`   | `X' XOR X` — XOR-разность |
| `ΔW_i` | `(W_i' − W_i) mod 2^32` — ADD-разность i-го слова |
| `W[0..15]` | 16 входных 32-битных слов (512 бит блока) |
| `W[16..63]` | Расширенное расписание |
| `MASK` | `0xFFFFFFFF` |

### Функции SHA-256

```python
MASK = 0xFFFFFFFF

def rotr(x, n):   return ((x >> n) | (x << (32-n))) & MASK
def sig0(x):      return rotr(x,7)  ^ rotr(x,18) ^ (x >> 3)     # расписание
def sig1(x):      return rotr(x,17) ^ rotr(x,19) ^ (x >> 10)    # расписание
def Sig0(x):      return rotr(x,2)  ^ rotr(x,13) ^ rotr(x,22)   # раунд
def Sig1(x):      return rotr(x,6)  ^ rotr(x,11) ^ rotr(x,25)   # раунд
def Ch(e,f,g):    return ((e & f) ^ (~e & g)) & MASK
def Maj(a,b,c):   return (a & b) ^ (a & c) ^ (b & c)

def schedule(W16):
    W = list(W16)
    for i in range(16, 64):
        W.append((sig1(W[i-2]) + W[i-7] + sig0(W[i-15]) + W[i-16]) & MASK)
    return W
```

### Раунд SHA-256

```
T1 = h + Sig1(e) + Ch(e,f,g) + K[r] + W[r]
T2 = Sig0(a) + Maj(a,b,c)
[h=g, g=f, f=e, e=d+T1, d=c, c=b, b=a, a=T1+T2]  (mod 2^32)
```

### Начальные значения (IV)

```python
IV = [0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a,
      0x510e527f, 0x9b05688c, 0x1f83d9ab, 0x5be0cd19]
# e0 = IV[4] = 0x510e527f
```

### Базовые константы (верифицированы)

```
C_IV  = h0 + Sig1(e0) + Ch(e0,f0,g0) + K[0]   = 0xf377ed68
T2_0  = Sig0(a0) + Maj(a0,b0,c0)               = 0x08909ae5
C_e   = d0 + C_IV mod 2^32                     = 0x98c7e2a2  (чётное, bit0=0)
BASE_A1 = C_IV + T2_0 mod 2^32                 = 0xfc08884d  (нечётное, bit0=1)
```

## 3. КРИПТОТОПОЛОГИЧЕСКИЙ ФРЕЙМВОРК

### Определение Δ-графа

**G(f, N)** — граф где вершины — входные значения, рёбра — пары с одинаковой разностью через f. Анализ SHA-256 = изучение структуры Δ-графа через раунды.

### Карта нелинейности SHA-256

| Функция | Тип | Нелинейна? |
|---------|-----|-----------|
| sig0, sig1 | XOR ротаций + сдвига | Нет (линейна над GF(2)) |
| Sig0, Sig1 | XOR ротаций | Нет (линейна над GF(2)) |
| Ch(e,f,g) | (e AND f) XOR (NOT e AND g) | **Да** (зависит от e) |
| Maj(a,b,c) | Мажоритарная | **Да** |
| Сложение mod 2^32 | Арифметическое | **Да** (переносы) |

**Вывод:** Вся нелинейность SHA-256 сосредоточена в Ch, Maj и арифметических переносах.

### Закон лавинного эффекта B(r)

```
B(r) = 256 × (1 − exp(−ln2 × max(0, r−4)))
```

| r | B(r) бит |
|---|----------|
| 5 | 128 |
| 6 | 192 |
| 10 | 252 |
| 15 | 255.9 |

**После 6 раундов изменение распространяется на ~75% всех бит выхода.**

## 4. СТРУКТУРНЫЕ ЛЕММЫ (L1..L7)

### L1 — Линейность ротационных функций [✅ ДОКАЗАНА]

**Формулировка:** Sig0, Sig1, sig0, sig1 **линейны над GF(2)^32**:
```
Sig1(x XOR δ) XOR Sig1(x) = Sig1(δ)   (не зависит от x!)
```

**Следствие L1.1:** При XOR-разности δe:
```
δSig1(e_r) = Sig1(δe_r)   — константа при фиксированном δe_r
```
Это основа для T_SIG1_LINEAR_CONST (П-22) и аналитического построения следов.

### L2 — Трёхсдвиговый регистр (Период 3) [✅ ДОКАЗАНА]

**Формулировка:** В SHA-256:
```
d_r = a_{r-3},   h_r = e_{r-3}   (точно, без нелинейности)
```

**Следствие:** При De_{r-3} = De_{r-2} = De_{r-1} = 0:
```
Δh_r = 0,   ΔSig1(e_r) = 0,   ΔCh(e_r,...) = 0
```
→ Это **основа каскадной техники**: нулевые De-значения «консервируются» через 3 раунда.

### L3 — Рекуррентное уравнение для Δe [✅]

```
Δe_{r+1} = Δd_r + ΔT1_r
         = Da_{r-3} + Δh_r + ΔSig1(e_r) + ΔCh(e_r,...) + ΔW_r
```
**Критическое замечание:** W[r] **не отменяется** в DT1_r из-за арифметических переносов — несмотря на то что ΔW_r кажется простым сдвигом.

### L4 — 3-битный барьер Sig1 [✅ ДОКАЗАНА]

**Формулировка:** `HW(Sig1(2^k)) = 3` для всех k = 0..31.

**Доказательство:** Биты (k−6)%32, (k−11)%32, (k−25)%32 всегда попарно различны, так как разности {5, 14, 19} не делятся на 32. Аналогично `HW(Sig0(2^k)) = 3`.

**Следствие:** Однобитовый дифференциал δe → как минимум 3 активных бита в δSig1(e).

### L5 — Неустранимые биты De₂ [✅ при j=0]

**Формулировка (только для j=0, δW0=1):** HW(De₂) ≥ 4, активные биты: {0, 7, 21, 26} = {0} ∪ Sig1(De₁=1).

**Ограничение:** Только для j=0 (δW0=2^0). При j≠0 набор бит другой.

### L6 — W[r] влияет на De_{r+1} аддитивно [✅]

**Формулировка:** При De_r=0 зависимость De_{r+1} от ΔW_r **линейна**:
```
De_{r+1} = De_{r+1}_nat + ΔW_r   (mod 2^32)
```
Это прямое следствие T_DEk_DECOMPOSITION и основа всей каскадной техники.

### L7 — Доминантный бит De₂ [Вероятностная]

Бит ROTR11 доминирует в De₂ при флипе одного бита δW0 — из-за структуры Sig1(e) = rotr(e,6)⊕rotr(e,11)⊕rotr(e,25), и bit 11 участвует наиболее «нейтрально» относительно переносов.

### Теорема Ch-cross — DCh при нулевом De [✅]

**Формулировка:** При De_r = 0:
```
ΔCh_r(e_r, f_r, g_r) = 0   (точно)
```
**Доказательство:** Ch(e,f,g) = (e AND f) XOR (NOT e AND g). Если e = e' (т.е. De_r=0), то Ch(e',f',g')−Ch(e,f,g) = 0 при f'=f, g'=g (что следует из De_{r-1}=De_{r-2}=0 через L2). ■

### Теорема HW(Da₁) — распределение через trailing ones

**Формулировка:** BASE_A1 = 0xfc08884d — **нечётное** (bit0=1). При δW0=1 (чётный W0):
```
HW(Da₁) ~ Geometric(1/2):  HW=2 (50%), HW=3 (25%), HW=4 (12.5%), ...
```
(Контраст с De₁=1 детерминированно при j=0, поскольку C_e чётное.)

### Теорема 1 — Вероятность Ch [✅]

`P(ΔCh = 0 | Δe = 2^j) = 1/2` (при случайных f,g). Верифицировано: P=0.5005 на 100k образцах.

### T_INVERSE — Обратный раунд SHA-256 [✅ ДОКАЗАНА]

```
T2 = Sig0(b') + Maj(b', c', d')          mod 2^32
T1 = a' − T2                              mod 2^32
d  = e' − T1                              mod 2^32
h  = T1 − Sig1(f') − Ch(f',g',h') − K[r] − W[r]  mod 2^32
```
**Верификация:** 10000 случайных раундов — 0 ошибок. 64 раунда вперёд + 64 назад = identity ✓.

## 6. КАСКАДНАЯ ТЕХНИКА — КЛЮЧЕВОЙ МЕТОД

### Базовый принцип

При `De_k = 0` следующее значение `De_{k+1}` зависит от `ΔW_k` **линейно**:
```
De_{k+1} = De_{k+1}_nat(W) + ΔW_k   (mod 2^32)
```
→ Выбрать `ΔW_k = −De_{k+1}_nat` → `De_{k+1} = 0` **детерминированно**.

### Эволюция

**П-7 (T_CASCADE базовый):** ΔW3 = −De4_nat → De4=0. Каскад k=3..11 → **9 нулей** за 2^22.

**П-10 (T_CASCADE_MAX):** Расширить каскад до k=3..15 → **14 нулей** за 2^22.

**П-13 (T_CASCADE_17 — ПРОРЫВ):**
- **T_DW2_FREEDOM:** ΔW2 = −De3_nat(W0,W1) → De3=0 для **любых** W0, W1 (бесплатно)
- Условие De17=0 требует: Da13 + ΔW16 = 0
- Стоимость: **2^32** (поиск W0, W1, улучшение с 2^54 в 2^22 раз)

### Финальная реализация (П-13)

```python
def cascade_3param(W0, W1, DW0=1):
    """15 нулей De3..De17=0 для ЛЮБЫХ W0, W1."""
    Wn  = [W0, W1, 0] + [0]*13
    DWs = [0]*16
    DWs[0] = DW0

    # Шаг 0: T_DW2_FREEDOM — De3=0 адаптивно (бесплатно)
    Wf_tmp = [(Wn[i] + DWs[i]) & MASK for i in range(16)]
    De3_nat = de(sha_r(schedule(Wn), 3), sha_r(schedule(Wf_tmp), 3), 3)
    DWs[2] = (-De3_nat) & MASK

    # Шаги 1..13: T_CASCADE_MAX — De4..De16=0 детерминированно
    for step in range(13):
        wi = step + 3;  dt = step + 4
        Wfc = [(Wn[i] + DWs[i]) & MASK for i in range(16)]
        DWs[wi] = (-de(sha_r(schedule(Wn), dt), sha_r(schedule(Wfc), dt), dt)) & MASK

    # De17 = 0 с вероятностью 2^(-32)
    Wf = [(Wn[i] + DWs[i]) & MASK for i in range(16)]
    return de(sha_r(schedule(Wn), 17), sha_r(schedule(Wf), 17), 17)
```

## 7. КЛЮЧЕВЫЕ ТЕОРЕМЫ С ДОКАЗАТЕЛЬСТВАМИ

### T_SCHEDULE_DECOUPLING [П-9, ✅]

**Формулировка:** `De_r` полностью определяется `W[0..r-1]`. Слова `W[r..63]` не влияют на `De_r`.

**Следствие:** Для анализа `De_r` при r<16 пространство поиска = 2^(32r), не 2^512.

---

### T_DW2_FREEDOM [П-13, КЛЮЧЕВАЯ ✅]

**Формулировка:**
```
De3 = De3_nat(W0, W1) + ΔW2   (точно, mod 2^32)
```
→ `ΔW2 = −De3_nat` даёт `De3=0` для **любых** (W0, W1).

**Доказательство:** W2 используется в раунде 2 только в T1 как слагаемое:
```
T1_2 = h1 + Sig1(e1) + Ch(e1,f1,g1) + K[2] + W2
e3   = d2 + T1_2   (аддитивно по W2)
```
Поэтому De3 линейна по ΔW2 с коэффициентом +1 (mod 2^32). ■

---

### T_DEk_DECOMPOSITION [П-15, ОБЩАЯ ✅]

**Формулировка:** При `De3..De_{k-1} = 0` для k ≥ 4:
```
De_k = Da_{k-4} + ΔW_{k-1}   (точно, mod 2^32)
```

**Доказательство:**
```
De_k = Δe_k = Δd_{k-1} + ΔT1_{k-1}
Δd_{k-1}        = Da_{k-4}   (L2: 3-сдвиговый регистр)
Δh_{k-1}        = De_{k-4} = 0   (каскад ✓)
ΔSig1(e_{k-1})  = 0              (De_{k-1}=0, L1 ✓)
ΔCh(e_{k-1},..) = 0             (De_{k-1}=0, Ch-cross ✓)
→  De_k = Da_{k-4} + ΔW_{k-1}   ■
```

**Применения:**
- k=17: `De17 = Da13 + ΔW16`
- k=18: `De18 = Da14 + ΔW17`

---

### T_CASCADE_17 [П-13, КЛЮЧЕВАЯ ✅]

**Стоимость De3..De17=0:**
```
1. ΔW2 = −De3_nat(W0,W1)       →  De3=0 бесплатно (∀ W0,W1)
2. ΔW3..ΔW15 = каскад          →  De4..De16=0 детерминированно
3. Поиск (W0,W1): Da13+ΔW16=0  →  2^32
```
**Итого: 15 нулей за 2^32. Улучшение: 2^54 → 2^32 (в 2^22 = 4M раз быстрее).**

---

### T_DE17_UNIFORM [П-13, ✅]

**Формулировка:** После адаптивного ΔW2, De17 распределён **равномерно** по [0, 2^32).

**Верификация:** 2000 пар → 2000 уникальных De17, равномерное покрытие 256/256 байт.
→ P(De17=0) = 2^{−32} подтверждено.

---

### T_DE17_DE18_INDEPENDENCE [П-14, ✅]

**Формулировка:** k17 ≈ 1.87×10^9, k18 ≈ 4.91×10^8, k17 ≠ k18.
→ Условия De17=0 и De18=0 через один параметр ΔW1=β **несовместны** → стоимость 2^64.

---

### T_BARRIER_16 [П-15, ДОКАЗАНА ✅]

**Формулировка:** Стоимость De3..De18=0 (16 нулей) = **2^64**.

**Доказательство:**
```
Все W[0..15] задействованы каскадом De3..De17=0.
W[17] = sig1(W[15]) + W[10] + sig0(W[2]) + W[1] — определён расписанием.
De18=0 ⟺ Da14 + ΔW17 = 0   (T_DE18_DECOMPOSITION)
P(De18=0 | De3..De17=0) ≈ 2^{−32}   (независимо, верифицировано)
→ Итого: 2^32 × 2^32 = 2^{64}   ■
```

---

### T_SAT_CASCADE [П-17, ✅]

Z3 BitVec: k≤16 → SAT за <1с; k=17 → **timeout**. Независимое подтверждение T_BARRIER_16.

---

### T_GENERALIZED_CASCADE [П-19, ✅]

**Формулировка:** Каскад работает для **любого ΔW0 ≠ 0**:
```
De3..De17=0  за 2^32  для произвольного ΔW0
```
P(De17=0) = 2^{−32} не зависит от ΔW0.

## 8. XOR-ДИФФЕРЕНЦИАЛЬНЫЙ АНАЛИЗ (П-20..П-22)

### T_MIXED_DIFF [П-20, ✅]

sig0, sig1 **линейны над XOR**. Расписание при XOR-разностях:
```
δW_r = sig1(δW_{r-2}) XOR δW_{r-7} XOR sig0(δW_{r-15}) XOR δW_{r-16}
```
Детерминировано (P=1). XOR-путь глубины ≥1 из 64 не найден за 200000 попыток.

### T_IV_BIT0 [П-21, ДОКАЗАНА АНАЛИТИЧЕСКИ ✅]

`C_e = 0x98c7e2a2` (bit0=0). При δW0=2^0=1:
```
δe1 = 1   для всех W0 ∈ {чётные}   (P = 1)
```
**Доказательство:** `e1 = C_e + W0 mod 2^32`. bit0(C_e)=0 → добавление 1 всегда даёт carry в бит 0. ■

### T_DOM_DIFF [П-21, КЛЮЧЕВАЯ ✅]

δW0 = 0x8000 (бит 15) — **доминирующий дифференциал**:
```
δe1 = 0x8000   с P = 0.773
```
Минимальный вес расписания: sum_wt=466, только 12/64 активных раундов.

### T_CARRY_ANALYTIC [П-22, ДОКАЗАНА ✅]

При δW0 = 2^k:
```
P(δe1 = 2^k) = C[0..k-1] / 2^k         если bit_k(C_e) = 1
P(δe1 = 2^k) = (2^k − C[0..k-1]) / 2^k  если bit_k(C_e) = 0
```
где C_e = 0x98c7e2a2.

**Верификация:** Подтверждена для k=0,2,4,...,30 (все ✓).

**Для k=15:** P = 0x7fff / 0x8000 ≈ 0.77 (согласуется с T_DOM_DIFF ✓).

### T_SUFFICIENT_R1 [П-22, ✅]

Условие `W0[0..14] ≥ threshold` (= 2^15 − C_e[0..14]):
```
→ δe1 = 0x8000  с P = 1  (детерминировано)
```
**Верификация:** 38554/38554 ✓. Практически: предварительный отбор W0 за ~2 попытки.

### T_SIG1_LINEAR_CONST [П-22, КЛЮЧЕВАЯ ✅]

```
δSig1(e1) = Sig1(δe1)   — НЕ зависит от e1_n!
```
При δe1 = 0x8000:
```
Sig1(0x8000) = rotr(0x8000,6) XOR rotr(0x8000,11) XOR rotr(0x8000,25)
             = 0x00400210   (КОНСТАНТА, P=1.0 для 5000 пар)
```
**Следствие:** Основа для аналитического построения многораундового XOR-следа.

### 2-раундовый след (П-22)

```
δW0 = 0x8000  →  P(δe1 = 0x8000) = 1.0   (T_SUFFICIENT_R1)
δW1 = 0x00408210  →  P(δe2 = 0x8000) ≈ 0.07   (жадный поиск)
```
След быстро ослабевает из-за лавинного эффекта.

## 10. БАРЬЕР 2^64 — ОБОСНОВАНИЕ

### Почему именно 2^64?

1. **Все степени свободы исчерпаны:** W[0..15] = 512 бит. Каскад De3..De17=0 фиксирует все 16 слов.
2. **W[17] не свободный:** `W[17] = sig1(W[15]) + W[10] + sig0(W[2]) + W[1]` — определён расписанием.
3. **De18=0** требует `Da14 + ΔW17 = 0`.
4. **Da14** — независимо от условия De17=0 (другая степень свободы).
5. **P(De18=0 | De3..De17=0) ≈ 2^{−32}** — независимо (верифицировано П-18).

**2^32 × 2^32 = 2^64.**

### Для преодоления барьера нужен принципиально новый подход

- Двухблочные сообщения (MITM через state[32])
- SAT/MILP с дополнительными структурными ограничениями
- Wang-style signed differences + аналитический след (П-23)
- Boomerang distinguisher

## 12. ОТКРЫТЫЕ ВОПРОСЫ И П-23

### Вопрос Q16: 16 нулей за менее 2^64?

**Подход 1 — Wang-style signed differences (П-23):**
- Использовать T_SIG1_LINEAR_CONST: δSig1_r = Sig1(δe_r) = CONST при δe_r = 2^k
- Аналитически строить: δe_r=2^k → выбрать δW_r → δe_{r+1}=2^j
- Цель: 4-раундовый след с P > 2^{−10}
- Конкретно: при δe1=0x8000, Sig1(0x8000) = 0x00400210 — отправная точка

**Подход 2 — Boomerang:**
- E = E_top ∘ E_bot^{−1}, MITM через промежуточное состояние

**Подход 3 — SAT/MILP:**
- CaDiCaL195 с 18-раундовым кодированием + структурные ограничения

**Подход 4 — Двухблочные сообщения:**
- Блок 1: произвольный → state[32]; Блок 2: подобран
- MITM через промежуточное состояние (T_INVERSE уже готов)

**Подход 5 — Покрытие a-регистра:**
- Аналогичный анализ Sig0/Maj для Da-регистра
- Возможна совместная атака (De, Da)

**Подход 6 — Нейросетевой поиск (Gohr 2019 precedent):**
- Нейросеть может найти характеристики, пропущенные вручную
- Применить к SHA-256 аналогично SPECK

### П-23 (следующий шаг)

Signed bit differences + аналитический k-раундовый след:
```
δe1 = 0x8000 (P=1 при T_SUFFICIENT_R1)
δSig1(e1) = 0x00400210 (P=1, T_SIG1_LINEAR_CONST)
δT1_1 = 0x00400210 + δCh(e1,...) + δW1 + δh1
Выбрать δW1: → δe2 = 2^j (следующий однобитовый дифференциал)
```
Цель: аналитическая 4-раундовая цепочка с суммарным весом < 2^{−10}.
