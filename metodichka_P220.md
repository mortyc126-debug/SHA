# П-220: Масштабирование и рекорд min HW(δstate[64])

## Направление A: E[HW] стабилен при N→∞

E[HW(δstate[16N])] = **64.0** для ВСЕХ N=1..20 (50K трейлов).
min HW ≈ 37..43 — стабильно, нет тренда.

**Вывод:** E[HW] константа. Нет убывающей структуры при добавлении блоков.
δstate[16k] имеет одинаковое распределение на каждой границе.

---

## Направление B: Birthday на δstate[32] — закрыто

При birthday δa[32]₁=δa[32]₂: δstate[32] НЕ совпадает (3/3 проверок дали НЕТ).

**Причина:** δb[32]=δa[31] — это δa предыдущего раунда у ЭТОГО трейла,
а не прошлого трейла. При δa[32]_1=δa[32]_2 из-за случайных W[0..31],
δa[31]_1 ≠ δa[31]_2 → δb[32] разный. Ранг δstate[32] = 128 (независимые).
Birthday требует O(2^64). Закрыто.

---

## Направление C (бонус): Новый рекорд!

### Рекорд min HW(δstate[64]) = **30**

```
d0=1, 4-блочный адаптивный трейл (64 раунда)
198M трейлов за 95 секунд (2.1M/с)
HW=33 при 100K, HW=32 при 23M, HW=30 при 160M
```

### Распределение HW(δstate[64]) (1M образцов)

```
E[HW] = 64.00  (симметричен вокруг 64)
min за 1M: HW=37
min за 200M: HW=30
```

Распределение — усечённый Binomial(128, 0.5), E=64, range [0..128].

### Статистический анализ минимума

Для N образцов из Bin(128, 0.5):
```
E[min] ≈ 64 - sqrt(128 × ln(N))
  N=1M:   64 - sqrt(128 × 13.8) ≈ 64 - 42 = 22
  N=200M: 64 - sqrt(128 × 19.1) ≈ 64 - 50 = 14
```

Наш результат HW=30 при 200M >> 14 — значит min убывает медленнее!

**Реальная формула (из данных):**
- HW=36 при N=500K → смещение ~8 от E=64
- HW=30 при N=200M → смещение ~34

Убывание: Δ = 128 - 2 × HW ≈ 2 × sqrt(64 × ln(N) × α) где α < 1.
При α≈0.1: E[min] ≈ 64 - sqrt(6.4 × ln(N))

Для HW=0: N = exp((64/sqrt(6.4))²) = exp(1024) → **недостижимо**.

---

## Честный вывод

4-блочный адаптивный трейл даёт min HW(δstate[64])=30 за 200M пар.
Это впечатляет, но:
- Распределение Bin(128,0.5) с E=64
- Минимум убывает медленно (~sqrt(log N))
- Для HW=0 нужно экспоненциально много трейлов

**min HW(δstate[64])=30** — это новый рекорд для адаптивных трейлов SHA-256,
но не путь к коллизии.

## Следующий шаг П-221

Новый вопрос: можно ли СОВМЕСТИТЬ адаптивный трейл с алгебраической структурой?

**Идея:** при δstate[64]=(δa,δb,δc,δd,0,0,0,0) (адаптивный трейл 4-блочный):
- Нулевые: δe=δf=δg=δh=0 ← всегда
- Ненулевые: δa,δb,δc,δd ← случайные

Если добавить ещё 4 блока адаптации (всего 8 блоков = 128 раундов):
- Аналог: δstate[64] становится новым "IV" для второго набора блоков
- Продолжаем адаптацию → δe[66..128]=0 дополнительно
- Но δstate[128] всё равно Bin(128,0.5) с E=64

Альтернатива: **комбинация с Wang birthday attack**.
Взять 2^16 пар из Wang chain (П-97) и для каждой считать δstate[64].
Если распределение другое → возможно объединение.
