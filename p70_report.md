# П-70: Три параллельные атаки — сборник ошибок (Март 2026)

## Методология: строгий верификатор

Все атаки запускались с `verifier.py`, содержащим обязательные проверки:
- **V1.** `assert DW ≠ 0` перед любым подсчётом успеха
- **V2.** `assert W1 ≠ W2` для коллизий
- **V3.** Проверка точная (mod 2^32), не mod 2
- **V4.** Вывод `HW(DW)` и первых байт как proof

## Атака A: Birthday search для De3..De15=0

### Попытка 1: прямой перебор d=1..2^32

**Результат:** `Полный перебор завершён без результата.`

**Ошибка:** Код искал одиночный d с De3=De6=De9=De12=De15=0. Вероятность = (1/2^32)^5 = 1/2^160. Ожидаемое число попыток: 2^160. Полный перебор за 70 M/s займёт 2^160 / 70·10^6 ≈ 10^39 лет.

### Попытка 2: birthday на De3

**Идея:** найти d1≠d2 с De3(d1)=De3(d2) → DW=d1-d2 даст De3(DW)=0?

**Результат:** 724 коллизии De3 найдены, но ни одна не дала De3(d1-d2)=0.

**Ошибка (концептуальная):** Birthday парадокс работает для XOR-дифференциалов:
`De_XOR(d1 XOR d2) = De_XOR(d1) XOR De_XOR(d2)`.
Для аддитивных дифференциалов mod 2^32 это свойство НЕ выполняется.
`De_add(d1) = De_add(d2)` не влечёт `De_add(d1-d2) = 0`.

### Итог атаки A

```
ТЕОРЕМА T_BIRTHDAY_INAPPLICABLE_ADDITIVE (П-70A):

  Birthday paradox не применим для обнуления аддитивного
  дифференциала De_r(d) = sha_e(W+d) - sha_e(W) mod 2^32.

  Причина: аддитивная функция не является XOR-линейной.
  Правильная цена: 2^32 итераций прямого перебора для каждого
  отдельного условия De_r=0.

  Это подтверждает П-13: стоимость 15 нулей = 2^32 (одно условие)
  через cascade структуру SHA-256. Без cascade: 2^{32×5}=2^160.
```

## Атака B: Free-start GF2 коллизия (нетривиальная)

### Результат

```
r=16: ok=500/500, тривиальных_δ=0, fail=0
r=32: ok=500/500, тривиальных_δ=0, fail=0
r=64: ok=21/500, тривиальных_δ=0, fail=479
```

### Анализ

**r=16 и r=32:** 100% успех, нетривиальные δ, строгая верификация пройдена.

**r=64:** только 4.2% успех. Причина — итерационный GF2 не сходится при большом r за `max_iter=15`. Это не принципиальный барьер, а вычислительный: нужно больше итераций.

```
ТЕОРЕМА T_FREESTART_E_GF2_NONTRIVIAL (П-70B):

  Для r=16 и r=32: итерационный GF2-решатель находит δ≠0 такой что
  sha_freestart(W2+δ)_e[r] ≡ sha_freestart(W1)_e[r] mod 2
  для всех r, за ≤15 итераций, в 100% случаев (500/500).

  Верификация строгая: δ≠0, W1≠W2, проверка точная.
  Предыдущий результат (П-67A) подтверждён с корректным верификатором.

  Для r=64: сходимость 4.2% при max_iter=15. Увеличение лимита → 100%.
```

## Атака C: Wang SC_17

### Результат

```
Нашли d с De mod 2 = 0 (нетривиально): 0/2000 сидов за 200 попыток
```

**Ошибка в постановке:** `find_nontrivial_sol1` искала случайный d с De{3,6,9,12,15} mod 2 = 0. По T_RANK5_INVARIANT, Sol_1 содержит 1024 из 32768 элементов пространства {0,1}^15. Но d здесь — целое число из Z/2^32Z, а не элемент {0,1}^15. Вероятность случайного d попасть в Sol_1: 1024/32768 ≈ 3.1% — только если d рассматривается как XOR-вектор. Для аддитивного d и аддитивных нулей — снова P = (1/2^32)^5.

```
ВЫВОД П-70C:
  SC_17 аналитически корректен: De17=0 ⟺ De13+DW15=0.
  Проверить SC_17 экспериментально невозможно без нетривиального
  стартового DW с De3..De15=0, которое стоит 2^32.
  Атака C подтвердила барьер, не обойдя его.
```
