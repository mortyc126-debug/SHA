# П-224: GF(2) анализ расписания + реальный SHA-256 трейл

## Ключевое открытие: расписание при ΔW[0]=d0, ΔW[1..15]=0

Для разницы двух сообщений (M, M') где M'=M⊕{W[0]+d0}:
```
ΔW[0]  = d0
ΔW[1..15] = 0
ΔW[16] = d0            (копия)
ΔW[17] = 0             (структурный ноль!)
ΔW[18] = sig1(d0)      (2-3 бита)
ΔW[19] = 0
ΔW[20] = sig1(sig1(d0))
...
```
Это детерминированный каскад, известный аналитически.

## Суммарный HW(ΔW[16..63]) для d0=8:

| Раунд | ΔW       | HW |
|-------|----------|----|
| 16 | 0x00000008 | 1 |
| 17 | 0x00000000 | **0** |
| 18 | 0x00050000 | 2 |
| 19 | 0x00000000 | 0 |
| 20 | 0x20000142 | 4 |
| 21 | 0x00000000 | 0 |
| ... | каскад | растёт |

Суммарный HW[16..63] = 514 (для d0=8)
Лучший однобитный d0 = бит 15 → сумма = 466

## Реальный SHA-256: E[HW(δstate[64])] = 128

При ΔW[0]=d0, ΔW[1..15]=0 (реальные сообщения M, M'):
```
E[HW(δstate[64])] = 128.00  ← как у случайных пар!
min HW = 86 за 10M пар
```

Сравнение:
```
Адаптивный трейл:    min=30   (W[0..63] свободны)
Реальный SHA-256:    min=86   (W[16..63]=расписание)
Случайные пары:      min≈88
```

## Фундаментальный результат

**Расписание SHA-256 обеспечивает полный лавинный эффект** даже при sparse входном дифференциале. E[HW]=128=max независимо от структуры ΔW[0..15].

Это не баг — это Feature: SHA-256 специально спроектирован так чтобы расписание обеспечивало диффузию.

## Теорема T_SCHEDULE_DIFFUSION (П-224)

```
Для любого ненулевого ΔW[0..15]:
  E_{W}[HW(δstate[64])] ≈ 128  (полная диффузия)
  
При адаптивном выборе W[16..63]:
  E_{адапт}[HW(δstate[64])] = 64  (половина = структура)
```

Разрыв 64 бита — это "цена расписания" для дифференциального анализа.

## Что дальше?

Расписание как барьер — подтверждено. Пути:

1. **MILP на расписание**: найти ΔW[0..15] который минимизирует sum HW(ΔW[16..63]) аналитически. Но мы видели: E[HW]=128 всегда.

2. **Multi-block**: два блока M1||M2. Второй блок IV' = output первого. Если первый блок создаёт "слабый IV" → второй блок атакуется легче.

3. **Chosen prefix**: атакующий выбирает M так чтобы W[0..15] имели конкретную структуру.

4. **Free-start остаётся**: результаты min HW=30 для сжимающей функции SHA-256 валидны и полезны для академического анализа.
