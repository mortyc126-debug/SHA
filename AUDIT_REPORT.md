## 2. СТРУКТУРНЫЕ ТЕОРЕМЫ (ВЕРНЫ)

### L1 — Линейность ротационных функций
✅ **ВЕРНО.** Σ0, Σ1, σ0, σ1 — линейные над GF(2)³². Следствие L1.1 (ΔΣ1(e,Δe)=Σ1(Δe)) корректно.

### L2 — Двойная цепочка эха (Период 3)
✅ **ВЕРНО.** Δd_r = Δa_{r-3} и Δh_r = Δe_{r-3} **доказано аналитически и верифицировано** для всех r≥3.
Объяснение: d_r = a_{r-3} и h_r = e_{r-3} напрямую следуют из структуры SHA-256 (сдвиговый регистр длины 3).
Число 3 = архитектурная константа SHA-256.

### L4 — 3-битный барьер Σ1
✅ **ВЕРНО.** HW(Σ1(2^k)) = 3 для всех k = 0..31.
Доказательство: биты (k-6)%32, (k-11)%32, (k-25)%32 всегда различны, т.к. разности {5, 14, 19} не делятся на 32.
Аналогично: HW(Σ0(2^k)) = 3 для всех k (разности {11, 20, 9}).

### Теорема 1 — Вероятность Ch
✅ **ВЕРНО.** P(ΔCh = 0 | Δe = 1 бит) = 1/2.
ΔCh[j] = f[j] XOR g[j] — независимо от e[j]. Верифицировано: P=0.5005 на 100k образцах.

### T_INVERSE — Обратный раунд SHA-256
✅ **ВЕРНО.** Формулы:
```
T2 = Sig0(b') + Maj(b', c', d')
T1 = a' − T2  (mod 2³²)
d  = e' − T1  (mod 2³²)
h  = T1 − Sig1(f') − Ch(f', g', h') − K[r] − W[r]  (mod 2³²)
```
Проверено на 10000 случайных раундах — 0 ошибок.

### Закон B(r)
✅ **ВЕРНО.** B(r) = 256×(1−e^{−ln(2)×max(0,r−4)})
Проверено: B(5)=128, B(6)=192, B(10)=252, B(15)=255.9. Все совпадают.

### L2/W_SAT3 — Структура нулевых раундов
✅ **ВЕРНО.** W_SAT3 даёт De3=De6=De9=0. Верифицировано.
✅ **ВЕРНО.** W_SAT4 даёт De3=De6=De9=De12=0. Верифицировано.
✅ **ВЕРНО.** W_SAT_v8 даёт De3=0, cost(16r)=207. Верифицировано.

## 3. НАЙДЕННЫЕ ОШИБКИ

---

### ❌ ОШИБКА 1 (СЕРЬЁЗНАЯ): «HW(De₁) = 1 ВСЕГДА» — обобщение неверно

**Где:** v8, Раздел 25.1; v9, вводная часть (упоминание «carry_xor: De₁ = 1, бит 0, всегда»)

**Заявление в docs:** *«HW(Δe₁) = 1 ALWAYS (carry_xor is always power of 2)»*

**Что на самом деле:**

Для j=0 (флип бита 0 при W[0] чётном): **ВЕРНО**, потому что C_e = 0x98c7e2a2 чётное → bit0(e₁) = 0 → De₁ = 1 всегда.

Для j≠0: **НЕ ГАРАНТИРОВАНО.** De₁ = carry_xor(e₁, j) где e₁ = 0x98c7e2a2 + W[0]. Если bit_j(e₁) = 1 (что возможно при многих W[0]), то HW > 1.

**Верифицированные нарушения при W[0]=0:**
| j | De₁ | HW |
|---|---|---|
| 1 | 0x00000006 | 2 |
| 5 | 0x00000060 | 2 |
| 7 | 0x00000180 | 2 |
| 9 | 0x00000600 | 2 |
| 13 | 0x000fe000 | 7 |
| ... | ... | >1 |

**Правильная формулировка:** HW(De₁) = 1 **гарантировано только для j=0** (что доказывается из bit0(C_e)=0). Для других j зависит от конкретного W[0].

**Последствия:** Все теоремы, опирающиеся на «De₁ = 2^j для любого j», ограничены областью j=0. В частности:
- L5 (биты {0,7,21,26} в De₂) — справедливы только для j=0
- Уравнение DCh и диффузия через Σ1(De₁) — только для j=0

**Где:** methodology_v13.docx, Теорема T_ADD8

**Результат аудита:**

```

0xfbdfff81, 0xfbe00081, 0xfc1fff81, 0xfc200081
```

## 4. УТОЧНЕНИЯ (НЕ ОШИБКИ, НО ТРЕБУЮТ ЯСНОСТИ)

### ⚠️ УТОЧНЕНИЕ 1: HW(Da₁) при j=0 — это НЕ «1 всегда»

**Где:** v8, Раздел 25; v9, Теорема HW(Da₁)

BASE_A1 = 0xfc08884d — **нечётное** (bit0=1). Поэтому при W[0] чётном bit0(a₁)=1, и добавление 1 вызывает перенос:
```
Распределение HW(Da₁):  HW=2 (50%), HW=3 (25%), HW=4 (12.5%), ...  (геом. распределение)
```
v9 это учитывает в «Теореме HW(Da₁) — формула через trailing ones». Но v8 в начале может создавать впечатление что HW=1. Важно: **De₁** (у e) всегда 1 при j=0, а **Da₁** (у a) — нет.

### ⚠️ УТОЧНЕНИЕ 2: Область L5 — биты {0,7,21,26} только для j=0

Биты De₂ = {0, 7, 21, 26} получены из:
- Бит 0: всегда от carry chain (bit0 De₁ = 1 при j=0)
- Биты {7,21,26}: = Σ1(1) = Σ1(De₁) при De₁=2⁰

Для j≠0 неустранимые биты De₂ = {j, (j−6)%32, (j−11)%32, (j−25)%32} — другой набор. Это в документе отмечено, но стоит выделить явно.

### ⚠️ УТОЧНЕНИЕ 3: T_LIFT-1 — формально тавтология

**T_LIFT-1:** De_r = 0 ⟺ lifted_diff_e_r ≡ 0 (mod 2³²)

Это тождественно верно по определению XOR-разности: De_r = e_r(M) XOR e_r(M') = 0 тогда и только тогда, когда e_r(M) = e_r(M') как 32-битные числа, что ⟺ их арифметическая разность ≡ 0 (mod 2³²).

Теорема верна, но её ценность — в связи с lifted-полиномами над Z для аналитического условия. Это стоит явно прояснить: «T_LIFT-1 полезна как мост к аналитическому описанию, не как новое математическое утверждение».

### ⚠️ УТОЧНЕНИЕ 4: T_IMG2 — «ожидание для случайной функции» неявное

**T_IMG2:** 32662 уникальных из 65536 = **78.8% от ожидаемого**

✅ 78.8% верно при условии, что ожидание считается для функции {0,1}^16 → {0,1}^16 (совпадение размеров):
E[unique] = N × (1 − (1−1/N)^N) ≈ 41427. Тогда 32662/41427 = 78.8% ✓

Если бы учитывали полный 32-битный выход, ожидание ≈ 65536 и процент был бы ≈50%.

**Рекомендация:** Добавить: «Ожидание считается при проекции на 16 бит выхода».

### ⚠️ УТОЧНЕНИЕ 5: T_LIFT-2 — ограничен e₁

**T_LIFT-2:** lifted_diff(W₀) = −lifted_diff(W₀ XOR 1)

Верно для lifted_diff_e₁: при W₀ чётном diff = −1, при нечётном diff = +1 (меняются знаком). ✓

Для e₂, e₃: корректность требует отдельной верификации из-за нелинейностей Maj, Ch и переносов.
